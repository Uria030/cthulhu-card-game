- **6 張卡**:跳車相機(asset cost2 reaction) / 跳車刺探(event cost1) / 跳車筆記(asset cost1 中央,報導標記容器,X 標記 → X 抽) / 跳車老編(ally cost3) / 跳車發稿(event cost0,棄堆撿) / 跳車直覺(skill cost0,感知 +2)
- **設計重心**:報導標記累積 → 一次大量抽牌
- **高等卡建議**:升級筆記(上限 5 標記) / 跳車獨家(消耗全標記 = 看光所有遭遇卡)

#### `雜耍`(6 張,反應閃避 + 機動)
- **6 張卡**:雜耍板凳(asset cost0,反應提高敵人 DC) / 雜耍帳棚(asset cost3,同軸 cost-1) / 雜耍棍(asset cost2 sidearm,2 傷 + 同軸觸發 +1) / 雜耍即興(event cost1) / 雜耍配合(ally cost3) / 雜耍空翻(skill cost0,反應 +2 閃避成功時移動)
- **設計重心**:反應閃避 + 走位;cost 0-3 全覆蓋
- **高等卡建議**:升級帳棚(全雜耍卡免費) / 雜耍宗師(每回合 1 次免疫攻擊)

## §3 高等卡寫作硬規則

設計 ★1+ 升級卡 / 簽名卡 / 調查員專屬新卡時,**若要連動既有軸**:

1. **軸值字串完全對齊本表**(例:寫『古籍』不是『古籍商人』也不是『古籍系列』)
2. **DB primary_axis_value 不加書名號**(存「古籍」),只有 effect 內文寫「『古籍』系列卡」才加書名號
3. **新卡 name_zh 仍須含軸值字串**(例:升級古籍書架 → 「古籍祕典書架」「古籍金匱」「古籍大匠書架」,不可叫「祕典書架」)
4. **避免破壞軸內角色分工**:升級版應**強化既有角色**(中央容器升上限 / 武器升傷害 / 加值技能升 +N),不該**重複既有機制**(已有 search event 不該再加一張)
5. **跨軸連動例外**:某些「老兵 × 軍用武器」「探長 × 線索」這類**軸 × 屬性**正交設計,可雙軸共生(本卡 primary_axis 選最強那層 = card_name,另一軸用 effect 文字描述)
