> 本次修正包含兩大部分：**規則系統更新** 與 **介面改良**。

# 第一部分：規則系統更新

## 1.1 狀態系統核心規則新增

在卡片設計器的規則引擎中，需要新增狀態系統的核心規則：

### 核心規則

> **所有狀態皆可堆疊。狀態在經歷完整一回合後，於該回合結束階段減少 1 層。**
> 
> （回合中途獲得的狀態，該回合結束時不減層，須等到下一個回合結束才減少。）

### 狀態分類

狀態分為兩種類型：

1. **數值型狀態** — 層數 = 效果強度
2. **開關型狀態** — 效果固定，層數 = 持續回合數

---

## 1.2 狀態效果定義更新

### 數值型負面狀態

| 狀態 | 代碼 | 效果 |
|------|------|------|
| 中毒 | `poison` | 受到的傷害 +X（X = 層數） |
| 流血 | `bleed` | 回合結束時扣 X 點 HP（X = 層數） |
| 燃燒 | `burning` | 回合開始時扣 X 點 HP。被火屬性攻擊時傷害 +X。獲得燃燒時移除所有潮濕（X = 層數） |
| 冷凍 | `frozen` | 移動花費 2 行動點（開關效果）。被冰屬性攻擊時傷害 +X（X = 層數） |
| 毀滅 | `doom_status` | 回合結束時受到 X 點傷害（X = 層數） |
| 發瘋 | `madness` | 受到的恐懼 +X（X = 層數） |
| 標記 | `marked` | 受到的傷害和恐懼都 +X（X = 層數） |
| 脆弱 | `vulnerable` | 受到的物理傷害 +X（X = 層數） |
| 無力 | `weakness_status` | 近戰傷害降低 X 點（X = 層數） |
| 潮濕 | `wet` | 被電擊屬性攻擊時傷害 +X（X = 層數） |
| 弱化 | `weakened` | 擲骰時丟兩次取差的，**擲骰後減少 1 層** |

### 開關型負面狀態（層數 = 持續回合數）

| 狀態 | 代碼 | 效果 |
|------|------|------|
| 黑暗 | `darkness` | 攻擊命中檢定 -2 |
| 繳械 | `disarm` | 不能用資產進行攻擊 |
| 疲勞 | `fatigue` | 結束階段不能抽牌和獲得資源 |
| 沈默 | `silence` | 無法施法 |

### 數值型正面狀態

| 狀態 | 代碼 | 效果 |
|------|------|------|
| 強化 | `empowered` | 擲骰時丟兩次取好的，**擲骰後減少 1 層** |
| 護甲 | `armor` | 降低 X 點物理傷害（X = 層數） |
| 護盾 | `ward` | 降低 X 點恐懼傷害（X = 層數） |
| 加速 | `haste` | 獲得 +X 行動點（X = 層數） |
| 再生 | `regeneration` | 回合開始時恢復 X 點 HP（X = 層數） |

### 特殊型正面狀態

| 狀態 | 代碼 | 效果 |
|------|------|------|
| 隱蔽 | `stealth` | 不觸發藉機攻擊。攻擊命中時傷害 +X（X = 層數）。**移動後或攻擊後全部移除。** |

---

## 1.3 卡片效果 JSON 結構更新

狀態相關的效果參數，將 `value` 改為 `stacks` 以明確表示層數：

**舊格式：**
```json
{
  "effect_code": "add_status",
  "params": { "status": "burning", "value": 1 }
}
```

**新格式：**
```json
{
  "effect_code": "add_status",
  "params": { "status": "burning", "stacks": 1 }
}
```

# 第二部分：介面改良

## 2.1 AI 生成區新增「一鍵清除」按鈕

### 位置
AI 卡片生成區的文字輸入框

### 需求
在「生成」按鈕旁邊新增「清除」按鈕，點擊後清空輸入框內容

### 使用情境
```
貼上文字 → 生成 → 確定卡片 → 清除 → 再貼上文字 → loop
```

### 實作建議
- 在「生成」按鈕左側新增「清除」按鈕
- 按鈕樣式：次要按鈕（不搶主視覺）
- 按鈕文字：「清除」或使用 ✕ 圖示

---

## 2.2 數值資訊區重構為三用途區塊

### 問題
現有數值欄位缺乏脈絡，無法體現卡片的三種核心用途

### 卡片三用途（來自數值規格 8.2）

| 用途 | 說明 | 卡牌去向 |
|------|------|----------|
| **打出** | 花費資源，佔 1 行動點，卡牌進場生效 | 進場或棄牌堆 |
| **加值** | 檢定時從手牌投入，使用卡面標註的固定加值 | 棄牌堆 |
| **消費** | 使用卡片的消耗性功能（開槍、喝藥等） | 依卡片說明 |

### 需求
將數值相關欄位重新分組為三個區塊，每個區塊有明確的標題

### 區塊一：打出效果
當卡片從手牌打出時的相關數據

| 欄位 | 說明 |
|------|------|
| 費用 (cost) | 打出所需資源 |
| 費用貨幣 (cost_currency) | 資源類型 |
| 裝備欄位 (slot) | 佔用的裝備格位 |
| 使用後去向 (consume_type) | 打出後卡片的去向 |
| 傷害 (damage) | 造成的固定傷害 |
| 恐懼傷害 (horror) | 造成的恐懼傷害 |
| 治療 HP (health_boost) | 恢復的 HP |
| 治療 SAN (sanity_boost) | 恢復的 SAN |

### 區塊二：加值數據
當卡片在檢定時投入作為加值的相關數據

| 欄位 | 說明 |
|------|------|
| 檢定加值 (skill_value) | 投入時提供的固定修正（技能卡 +1~+3，其他卡 0~5） |
| 加值屬性 | 此加值適用的屬性（可多選：力量/敏捷/體質/智力/意志/感知/魅力/萬能） |
| 加值附加效果 | 檢定成功/失敗時的額外效果（選填） |

### 區塊三：消費功能
當使用卡片上的消耗性功能時的相關數據

| 欄位 | 說明 |
|------|------|
| 彈藥 (ammo) | 子彈數量 |
| 使用次數 (uses) | 可使用次數 |
| 武器階層 (weapon_tier) | 武器傷害等級 |
| 戰鬥風格 (combat_style) | 對應的戰鬥風格 |
| 檢定屬性 (check_attribute) | 使用時的檢定屬性 |
| 檢定修正 (check_modifier) | 使用時的檢定加成 |
| 檢定方式 (check_method) | 擲骰 / 混沌袋 |

### 介面呈現
- 每個區塊用金色標題分隔，例如：
  - `▸ 打出效果`
  - `▸ 加值數據`
  - `▸ 消費功能`
- 測試階段先分區塊即可，精細佈局待美術階段調整

## 2.3 描述生成方向修正 + 名稱生成功能

### 問題
1. 效果描述的「重新生成描述」功能做反了，目前是英文 → 中文，應該是**中文 → 英文**
2. 卡片名稱沒有生成功能

### 設計原則
- 主要使用者為繁體中文使用者
- 中文為主，英文為備用
- 所有生成功能都應該是「中文 → 英文」

### 修正項目

#### A. 卡片名稱區
- 在「英文名稱」輸入框旁新增「生成英文」按鈕
- 點擊後：讀取中文名稱 → 呼叫 AI 翻譯 → 填入英文名稱欄位

#### B. 效果描述區
- 修正「重新生成描述」按鈕的邏輯
- 點擊後：讀取中文描述 → 呼叫 AI 翻譯 → 填入英文描述欄位
- 按鈕文字建議改為「生成英文描述」以明確方向

### AI 翻譯 Prompt 建議

```
你是克蘇魯神話卡牌遊戲的翻譯專家。請將以下繁體中文翻譯成英文。
保持卡牌遊戲的專業術語風格，簡潔有力。

中文：{input}

請只回傳英文翻譯，不要加任何解釋。
```
