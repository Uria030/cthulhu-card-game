# Claude Code 指令：卡片設計器改良 v3
## Card Designer Enhancement v3

> **給 Claude Code：** 本次改良整合《卡片價值計算規範 v0.1》與《卡片設計規範 v0.2》的內容，
> 重新設計卡片設計器的工作流程與介面，目標是**大幅提升設計效率與使用體驗**。

---

# 設計理念

## 核心問題
1. 現有介面欄位過多、分散，設計一張卡需要填大量欄位
2. 費用需要手動計算，容易出錯
3. 沒有價值計算輔助，無法即時檢驗平衡
4. AI 生成缺乏價值系統約束，產出常不合規

## 解決方案
1. **類型導向工作流** — 選定卡片類型後，只顯示該類型需要的欄位
2. **價值即時計算** — 自動計算效果總價值與建議費用
3. **平衡檢驗面板** — 即時顯示價值公式、檢驗結果
4. **AI 強化** — 將價值系統寫入 Prompt，讓 AI 產出平衡的卡片

# 第一部分：新工作流程

## 1.1 步驟式設計流程

將設計流程改為**引導式步驟**，減少認知負擔：

```
步驟 1：選擇卡片類型
  ↓ 根據類型顯示對應欄位
步驟 2：填寫基本資訊（名稱、陣營、等級）
  ↓ 
步驟 3：設計效果（使用效果選擇器）
  ↓ 自動計算效果價值
步驟 4：設定費用（0-6）
  ↓ 
步驟 5：系統反推稀有度（根據價值、等級、費用計算）
  ↓ 顯示平衡檢驗結果
步驟 6：填寫風味文字、儲存
```

## 1.2 稀有度反推公式

稀有度不由設計者選擇，而是根據效果價值、等級、費用**反推**：

**事件卡：**
```
稀有度抵扣 = 效果總價值 - 等級抵扣 - 費用
```

**資產卡：**
```
稀有度抵扣 = 總輸出價值 - 留場修正(-1V) - 消耗類型修正 - 等級抵扣 - (費用 × 2)
```

**盟友卡：**
```
稀有度抵扣 = 總價值 - 等級抵扣 - 費用
```

### 抵扣值對照表（向上進位）

| 計算出的抵扣值 | 對應稀有度 |
|----------------|------------|
| 0 以下 | 隨身 |
| 0.1 ~ 1 | 基礎 |
| 1.1 ~ 2 | 標準 |
| 2.1 ~ 3 | 進階 |
| 3.1 ~ 4 | 稀有 |
| 4.1 ~ 5 | 傳奇 |
| > 5 | ⚠️ 超出範圍，需調整設計 |

### 範例

設計一張事件卡：
- 效果總價值：9V
- 等級：LV5（抵扣 -5V）
- 設定費用：2

```
稀有度抵扣 = 9V - 5V - 2 = 2V
→ 對應稀有度：標準（-2V）
```

```
│                                                         │
```

### 計算邏輯

依卡片類型套用不同公式：

**事件卡：**
```
需要稀有度抵扣 = 效果總價值 - 等級抵扣 - 費用
換算比例 = 1:1
```

**資產卡（武器）：**
```
總輸出價值 = 傷害 × 彈藥數
需要稀有度抵扣 = 總輸出價值 - 留場修正(-1V) - 消耗類型修正 - 等級抵扣 - (費用 × 2)
換算比例 = 2:1
```

**資產卡（非武器）：**
```
總輸出價值 = 單次效果價值 × 使用次數
需要稀有度抵扣 = 總輸出價值 - 留場修正(-1V) - 消耗類型修正 - 等級抵扣 - (費用 × 2)
換算比例 = 2:1
```

**盟友卡：**
```
總價值 = HP×0.5V + SAN×0.5V + 留場2V + 能力價值
需要稀有度抵扣 = 總價值 - 等級抵扣 - 費用
換算比例 = 1:1
```

**技能卡：**
```
費用 = 0（固定）
不計算稀有度，改為檢驗價值區間
LV0: 2.5-3V | LV1: 2.5-3V | LV2: 5-6V | LV3: 7-8V | LV4: 8-9V | LV5: 9-10V
```

### 稀有度對照表（向上進位）

| 計算出的抵扣值 | 對應稀有度 |
|----------------|------------|
| ≤ 0 | 隨身（0） |
| 0.1 ~ 1 | 基礎（-1V） |
| 1.1 ~ 2 | 標準（-2V） |
| 2.1 ~ 3 | 進階（-3V） |
| 3.1 ~ 4 | 稀有（-4V） |
| 4.1 ~ 5 | 傳奇（-5V） |
| > 5 | ⚠️ 超出範圍 |

### 警示系統

| 情況 | 顯示 |
|------|------|
| 抵扣值剛好對應整數稀有度 | ✅ 完美平衡 |
| 抵扣值向上進位（如 1.5 → 標準） | ✓ 平衡（稍有餘裕） |
| 抵扣值 > 5 | ❌ 超出範圍：效果過強或費用過低，需調整 |
| 抵扣值 < 0 | ⚠️ 效果偏弱：可增加效果或降低費用 |
| 技能卡價值超出區間 | ⚠️ 價值 X，超出 LV{n} 區間 |

## 2.3 等級與消耗類型選擇

- 0 級 → 0

- 5 級 → -5V

### 消耗類型下拉選單

- 留在場上 → -1V

# 第七部分：AI 生成強化

## 7.1 更新 Gemini Prompt

將價值計算系統寫入 AI Prompt，讓生成的卡片自帶平衡：

```markdown
你是克蘇魯神話卡牌遊戲的卡片設計師。請根據以下規則和使用者需求，生成一張**數值平衡**的卡片。

## 價值計算系統

基礎單位：1V = 1 行動點 = 1 資源 = 抽 1 張牌 = 造成 1 點傷害

### 效果價值表（摘要）
- 直接傷害：1V/點
- 直接恐懼傷害：3V/點
- 單次攻擊傷害加成：2.5V/點
- 恢復 HP/SAN：1.5V/點
- 抽牌：1V/張
- 搜尋牌庫找特定卡：6V
- 移動：1V/格
- 施加燃燒/冷凍/中毒：3V/層
- 施加標記/發瘋/隱蔽：6V/層
- 強化/護甲：3V/層
- 護盾/再生：6V/層
- 快速（不用行動點）：+1V
- 可指定其他調查員：+2V

### 稀有度反推公式

稀有度由系統根據效果價值、等級、費用反推：

**事件卡：**
需要稀有度抵扣 = 效果總價值 - 等級抵扣 - 費用（1:1 換算）

**資產卡：**
需要稀有度抵扣 = 總輸出價值 - 1V - 消耗類型修正 - 等級抵扣 - (費用 × 2)（2:1 換算）

**盟友卡：**
需要稀有度抵扣 = HP×0.5V + SAN×0.5V + 2V + 能力價值 - 等級抵扣 - 費用

**技能卡：**
費用 = 0（固定），不計算稀有度
價值區間：LV0=2.5-3V, LV2=5-6V, LV3=7-8V, LV5=9-10V

### 抵扣值對照表（向上進位）
- ≤ 0：隨身
- 0.1 ~ 1：基礎
- 1.1 ~ 2：標準
- 2.1 ~ 3：進階
- 3.1 ~ 4：稀有
- 4.1 ~ 5：傳奇
- > 5：設計超出範圍，需調整

### 等級抵扣
0級=0, 1級=-1V, 2級=-2V, 3級=-3V, 4級=-4V, 5級=-5V

### 消耗類型修正
留場=-1V, 棄牌=0, 短休息=-1V, 長休息=-2V, 移除=-3V

## 武器傷害參考
傷害 1=隨身, 2=基礎, 3=標準, 4=進階, 5=稀有, 6=傳奇

## 使用者需求
{使用者輸入的描述}

## 輸出要求
1. 先計算效果總價值
2. 設定合理的費用（0-6）
3. 套用反推公式計算需要的稀有度抵扣
4. 向上進位得出稀有度
5. 輸出完整 JSON，包含 `value_calculation` 欄位說明計算過程
```
