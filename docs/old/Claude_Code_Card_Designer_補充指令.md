# Claude Code 補充指令：卡片設計器調整
## Card Designer Patch Instructions

> **給 Claude Code：** 請對已完成的卡片設計器 (MOD-01) 進行以下兩項調整。

---

## 調整一：卡片代碼自動生成系統

### 需求

卡片代碼不再由使用者手動輸入，改為**系統自動生成**，格式如下：

```
[系列][陣營極][風格]-[流水號]
```

### 代碼結構

| 段落 | 說明 | 範例值 |
|------|------|--------|
| 系列 | 卡片所屬的系列/擴充 | C = 核心 (Core)，之後可能有 E1 = 擴充一 等 |
| 陣營極 | 卡片歸屬的陣營極代碼 | E / I / S / N / T / F / J / P / N0（中立） |
| 風格 | 卡片風格代碼 | AH / AC / OH / OC |
| 流水號 | 該組合下的序號，兩位數 | 01, 02, 03... |

### 範例

| 代碼 | 意義 |
|------|------|
| `CEAH-01` | 核心系列 · E 號令 · 直接正面 · 第 1 張 |
| `CEAH-02` | 核心系列 · E 號令 · 直接正面 · 第 2 張 |
| `CNOC-01` | 核心系列 · N 天啟 · 間接負面 · 第 1 張 |
| `CN0AH-01` | 核心系列 · 中立 · 直接正面 · 第 1 張 |
| `CSAC-05` | 核心系列 · S 鐵證 · 直接負面 · 第 5 張 |

### 介面變更

1. 移除代碼的手動輸入框
2. 新增「系列」下拉選單（預設只有「C 核心」，未來可擴充）
3. 代碼根據「系列 + 陣營極 + 風格」三個選擇自動生成
4. 流水號自動計算：查詢資料庫中同一組合的最大流水號 +1
5. 生成的代碼顯示在表單上方，唯讀，不可編輯
6. 代碼在**儲存時才最終確定**（避免中途修改陣營或風格後編號衝突）

### 後端 API 調整

新增一個端點或修改 POST /api/cards：
- 儲存時由後端根據系列 + 陣營 + 風格查詢當前最大流水號
- 自動分配下一個流水號
- 確保在並發寫入時不會產生重複代碼（使用資料庫層級的唯一約束）

## 調整二：消耗品三種回復層級

### 需求

原本的消耗品只有「消耗品 是/否」一個勾選框，現在改為**三種回復層級**：

| 層級 | 名稱 | 說明 | 遊戲效果 |
|------|------|------|----------|
| `long_rest` | 長休息回復 | 使用後進入棄牌堆，長休息時回到牌組 | 每場冒險可反覆使用，但一場戰鬥中打完就沒了 |
| `short_rest` | 短休息回復 | 使用後進入棄牌堆，短休息時回到牌組 | 比長休息更快回來，場景間短暫喘息即可恢復 |
| `removed` | 移除遊戲 | 使用後從遊戲中移除，需要重新購買 | 真正的一次性消耗，用掉就永遠沒了（除非再次獲得） |

### 介面變更

1. 移除原本的「消耗品 (is_consumable)」勾選框
2. 新增「使用後去向」下拉選單，選項：
   - **留在場上**（資產類預設）— 不消耗，持續存在
   - **進入棄牌堆**（事件類預設）— 正常棄牌，洗牌後可再抽到
   - **長休息回復** — 進入棄牌堆，長休息時自動回到牌組
   - **短休息回復** — 進入棄牌堆，短休息時自動回到牌組
   - **移除出遊戲** — 永久移除，需重新獲取
3. 當選擇了有消耗性的選項時，顯示「使用次數」和「彈藥」欄位

### 資料結構變更

將原本的 `is_consumable: boolean` 改為：

```javascript
consume_type: "stay" | "discard" | "long_rest" | "short_rest" | "removed"
```

| 值 | 說明 |
|---|------|
| `stay` | 留在場上（資產預設） |
| `discard` | 進入棄牌堆（事件預設） |
| `long_rest` | 長休息回復 |
| `short_rest` | 短休息回復 |
| `removed` | 移除出遊戲 |

### 資料庫調整

在 `card_definitions` 表中：
- 移除 `is_consumable BOOLEAN` 欄位
- 新增 `consume_type VARCHAR(16) NOT NULL DEFAULT 'discard' CHECK (consume_type IN ('stay', 'discard', 'long_rest', 'short_rest', 'removed'))`

### 預覽卡片更新

在即時預覽中顯示對應的消耗標示：
- 長休息回復 → 顯示「長休息回復」標籤
- 短休息回復 → 顯示「短休息回復」標籤
- 移除出遊戲 → 顯示紅色「永久消耗」標籤

## 調整三：裝備欄位系統重構

### 需求

原本的裝備欄位不正確，需要重新設計。裝備系統分為兩層：

**物理裝備（有格位限制）：** 調查員身上的物理格位有數量上限。

| 欄位代碼 | 中文 | 格位上限 | 說明 |
|----------|------|----------|------|
| `one_hand` | 單手 | 合計 2 格 | 單手武器、工具。雙手武器佔 2 格 |
| `two_hand` | 雙手 | 佔滿 2 格 | 雙手武器、大型工具，佔滿整個手持空間 |
| `head` | 帽子 | 1 格 | 頭部裝備 |
| `body` | 身體 | 1 格 | 護甲、外套 |
| `accessory` | 配件 | 待定 | 飾品、護符 |
| `arcane` | 神秘 | 2 格（待定） | 法器、咒物 |

**非物理卡片（無格位限制）：** 打出後常駐場上，不佔物理裝備格。

| 欄位代碼 | 中文 | 格位上限 | 說明 |
|----------|------|----------|------|
| `talent` | 天賦 | 無限制 | 被動加值，透過天賦樹解鎖 |
| `expertise` | 專長 | 無限制 | 被動加值，透過天賦樹或團隊凝聚力解鎖 |

**不需要裝備欄位的情況：**
- 卡片類別為「盟友」→ 隱藏裝備欄位（盟友不佔格位）
- 卡片類別為「事件」→ 隱藏裝備欄位（一次性效果）
- 卡片類別為「技能」→ 隱藏裝備欄位（檢定加值用）

只有卡片類別為「資產」時才需要選擇裝備欄位。

### 介面變更

1. 裝備欄位下拉選單更新為：單手 / 雙手 / 帽子 / 身體 / 配件 / 神秘 / 天賦 / 專長
2. 只在卡片類別選擇「資產」時顯示裝備欄位
3. 選擇「雙手」時，提示「將佔用 2 個手持格位」

## 調整四：卡片分類系統修正

### 需求

原本把「神啟卡」「簽名卡」「弱點」當作獨立的卡片類別，與「資產/事件/盟友/技能」並列。這是錯誤的。

正確的結構是：

**卡片類別（存在形式）— 必選一個：**
- 資產（打出後留在場上）
- 事件（一次性效果）
- 盟友（打出後留在場上，NPC 夥伴）
- 技能（檢定加值用）

**特殊身份標記 — 獨立勾選，可疊加在任何類別上：**
- 簽名卡（調查員獨有，無法移除）
- 弱點（負面卡，強制納入牌組）
- 神啟卡（詛咒卡，抽到即觸發，無法正常移除）

這意味著一張卡片可以是：
- 簽名 + 資產 → 調查員的專屬武器
- 弱點 + 資產 → 留在場上的持續詛咒
- 弱點 + 事件 → 抽到即發動的一次性災難
- 神啟 + 事件 → 被洗入牌組的一次性恐怖事件
- 神啟 + 資產 → 抽到後留在場上持續折磨你

### 介面變更

1. 「卡片類別」下拉選單只保留四個選項：資產 / 事件 / 盟友 / 技能
2. 新增三個獨立勾選框：
   - ☐ 簽名卡 — 勾選後顯示「歸屬調查員」欄位
   - ☐ 弱點 — 勾選後卡片名稱旁顯示警告標記
   - ☐ 神啟卡 — 勾選後顯示「可否移除」「可否用作加值」「累計觸發致死次數」欄位
3. 這三個勾選框可以同時勾選（雖然罕見，但不排除一張卡同時是簽名 + 弱點的可能）

### 資料結構變更

原本的 `type` 欄位：
```javascript
type: "asset" | "event" | "ally" | "skill" | "weakness" | "revelation" | "signature"
```

改為：
```javascript
type: "asset" | "event" | "ally" | "skill",    // 存在形式（必選一）
is_signature: false,    // 是否為簽名卡（勾選框）
is_weakness: false,     // 是否為弱點（勾選框）
is_revelation: false,   // 是否為神啟卡（勾選框）
```

### 資料庫調整

在 `card_definitions` 表中：
- `card_type` 的 ENUM 只保留 `'asset', 'event', 'skill'`（移除 `'weakness', 'revelation', 'signature'`）
- 注意：保留 `'ally'` 但不在 ENUM 裡的話需要加入，或將盟友視為資產的子類型 — 建議直接加入 `'ally'` 到 ENUM
- 新增三個 BOOLEAN 欄位：
  - `is_signature BOOLEAN NOT NULL DEFAULT FALSE`
  - `is_weakness BOOLEAN NOT NULL DEFAULT FALSE`
  - `is_revelation BOOLEAN NOT NULL DEFAULT FALSE`

### 預覽卡片更新

在即時預覽中：
- 簽名卡 → 顯示金色「簽名」標記
- 弱點 → 顯示紅色「弱點」標記
- 神啟卡 → 顯示紫色「神啟」標記
- 這些標記疊加在類別標籤旁邊
