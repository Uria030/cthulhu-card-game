# 第四部分：後端 API

## 4.1 端點定義

```
# 天賦樹
GET    /api/talent-trees                       — 取得所有（8 棵，含設計進度摘要）
GET    /api/talent-trees/:factionCode          — 取得單棵（含分支 + 全部節點 + 效果）
PUT    /api/talent-trees/:factionCode          — 更新天賦樹基本資訊

# 分支路線
GET    /api/talent-trees/:factionCode/branches          — 取得該樹的 3 條分支
PUT    /api/talent-trees/:factionCode/branches/:index    — 更新單一分支

# 天賦節點
GET    /api/talent-trees/:factionCode/nodes              — 取得該樹全部節點
GET    /api/talent-trees/:factionCode/nodes/:nodeId      — 取得單一節點（含效果列表）
PUT    /api/talent-trees/:factionCode/nodes/:nodeId      — 更新單一節點
POST   /api/talent-trees/:factionCode/nodes              — 新增節點（非標準流程，備用）
DELETE /api/talent-trees/:factionCode/nodes/:nodeId       — 刪除節點（非標準流程，備用）

# 節點效果
PUT    /api/talent-trees/:factionCode/nodes/:nodeId/effects — 批次更新效果（整批覆寫）

# AI 生成
POST   /api/talent-trees/:factionCode/generate-branch    — AI 生成一條完整分支（10 個節點）
POST   /api/talent-trees/:factionCode/generate-node      — AI 生成單一節點效果

# 比較與統計
GET    /api/talent-trees/compare                         — 八棵樹的比較摘要
GET    /api/talent-trees/:factionCode/stats              — 單棵樹的設計統計

# 匯出入
GET    /api/talent-trees/export                          — 匯出全部為 JSON
POST   /api/talent-trees/import                          — 匯入（接受 JSON）
```

# 第八部分：表單互動邏輯

## 8.1 陣營切換

- 點擊左側陣營 → API 請求 `GET /api/talent-trees/:factionCode`
- 載入回傳資料到所有區塊
- 天賦樹視覺化重繪

## 8.2 節點選擇

- 點擊天賦樹圖中的節點 → 展開區塊四（節點詳細編輯）
- 點擊另一個節點 → 切換編輯目標（若有未儲存變更，提示確認）
- 點擊空白處 → 收合區塊四

## 8.3 設計狀態自動判定

### 節點層級

- `complete`：name_zh 非空 + 至少一個效果的 effect_desc_zh 非空
- `draft`：name_zh 非空但無效果描述
- `pending`：使用 Seed Data 的預設名稱，無自訂內容

### 天賦樹層級

- `complete`：全部 32 節點都是 `complete`
- `partial`：至少 1 個節點是 `complete` 或 `draft`
- `pending`：全部節點都是 `pending`

## 8.4 前置需求自動填入

- 主幹節點（Lv1）：無前置
- 主幹節點（Lv2）：前置 = Lv1
- 分支選擇點（Lv3）：前置 = Lv2
- Lv4-12 的分支節點：前置 = 同分支的前一級節點

管理員可以在此基礎上新增額外前置需求（例如跨分支需求）。

## 8.5 屬性提升自動填入

- Lv2 屬性提升：自動填入陣營主屬性
- Lv7/10/11/12 的屬性提升：預設為空，管理員選擇

## 8.6 天賦卡搜尋
