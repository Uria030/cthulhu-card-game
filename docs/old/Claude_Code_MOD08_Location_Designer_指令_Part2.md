### 視覺類型切換邏輯

- **無視覺素材**：預覽區顯示佔位符圖示
- **上傳圖片**：顯示檔案選擇器 + 圖片預覽；儲存圖片到後端並記錄 URL
- **AI 生成 SVG**：顯示 Prompt 輸入框 + 生成按鈕 + SVG 預覽
- **自訂 SVG**：開啟一個 SVG 程式碼編輯器（使用 textarea 即可，不需要完整 IDE）

### SVG 預覽區

- 使用 `<div>` 容器 + `dangerouslySetInnerHTML`（React）或直接插入（原生 JS）
- 預覽區尺寸：固定 600×400 或按比例縮放
- 邊框：`--border` 色
- 若 SVG 載入失敗，顯示錯誤訊息

### AI 生成 SVG 按鈕行為

點擊後：

1. 檢查 Prompt 是否為空 → 若空則提示「請先輸入地點描述」
2. 按鈕變為 loading 狀態
3. 呼叫 `POST /api/admin/locations/:id/ai/generate-svg`
4. 後端回傳 SVG 程式碼 → 前端更新預覽
5. **不自動儲存** — 設計師確認後手動按儲存

# 第八部分：Gemini SVG 生成

## 8.1 生成範圍（測試版）

**測試版僅支援室內俯視平面圖的 SVG 生成。**

- 風格：純黑白線稿 + 文字標籤
- 尺度限制：僅 `room` 和 `block` 級（中小尺度）
- 不支援：城市/跨國級的概念圖、擬真彩色渲染、3D 立體圖

## 8.3 後端呼叫 Gemini 的邏輯

```javascript
// 後端：POST /api/admin/locations/:id/ai/generate-svg

async function generateSVG(req, res) {
  const { id } = req.params;
  const { prompt, scale } = req.body;

  // 1. 取得地點基本資訊
  const location = await db.locations.findById(id);
  if (!location) {
    return res.status(404).json({ error: 'location_not_found' });
  }

  // 2. 驗證尺度（測試版只支援 room/block）
  if (scale && !['room', 'block'].includes(scale)) {
    return res.status(400).json({
      error: 'scale_not_supported',
      message: '測試版僅支援房間級與街區級的 SVG 生成。'
    });
  }

  // 3. 組裝 Prompt
  const fullPrompt = buildSVGPrompt({
    name_zh: location.name_zh,
    scale: scale || location.scale_tag || 'room',
    prompt: prompt
  });

  // 4. 呼叫 Gemini API
  try {
    const response = await callGemini(fullPrompt);

    // 5. 解析回應 — 提取 SVG 程式碼
    let svgCode = response.trim();

    // 若 Gemini 包了 ```svg 或 ```xml，清除掉
    svgCode = svgCode.replace(/^```(?:svg|xml)?\n?/, '').replace(/\n?```$/, '');

    // 6. 驗證基本格式
    if (!svgCode.startsWith('<svg') || !svgCode.endsWith('</svg>')) {
      return res.status(500).json({
        error: 'invalid_svg',
        message: 'Gemini 回傳的內容不是有效的 SVG 格式。',
        raw_response: response
      });
    }

    // 7. 回傳 SVG（不自動儲存）
    return res.json({
      svg_code: svgCode,
      generation_note: `為「${location.name_zh}」生成的 ${scale} 級 SVG`
    });

  } catch (err) {
    return res.status(500).json({
      error: 'gemini_api_error',
      message: err.message
    });
  }
}
```

## 8.4 品質改善迭代機制

第一次生成結果可能不理想。設計師可透過以下方式迭代：

1. **重新生成**：同樣的 Prompt 再按一次生成按鈕，Gemini 會給不同結果
2. **修改 Prompt 再生成**：調整描述中的關鍵詞（例如加上「三個房間」「長形走廊」等具體指示）
3. **手動編輯 SVG**：點「手動編輯 SVG 程式碼」直接修改生成結果
4. **放棄 SVG，改上傳圖片**：切換視覺類型為「上傳圖片」

## 8.5 生成失敗的處理

| 失敗情境 | 處理方式 |
|---------|---------|
| Gemini API 連線失敗 | 顯示錯誤 + 建議稍後再試 |
| Gemini 回傳非 SVG 格式 | 顯示原始回應供除錯，按鈕可重試 |
| SVG 格式無效（瀏覽器無法渲染） | 預覽區顯示錯誤，但保留原始程式碼供手動修改 |
| 生成太慢（超過 30 秒） | 顯示 loading 超時提示，可取消 |
