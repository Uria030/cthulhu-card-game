
## 一、標籤頁 1：鍛造詞條設計器

### 1.1 左欄：詞條列表與分類導航

**上半部：類別導航（固定高度約 160px）**

五個類別以視覺色塊呈現：

```
┌── 類別導航 ──────────────────────┐
│                                  │
│  ■ 全部（23）                    │
│  ├ 🪨 礦物（5）    [堅硬 鋒利]   │
│  ├ 🪵 木材（5）    [結構 效率]   │
│  ├ 🐛 蟲類（3）    [毒素 腐蝕]   │
│  ├ 🐟 魚類（5）    [恢復 適應]   │
│  └ 👁 怪物（5）    [超自然]      │
│                                  │
└─────────────────────────────────┘
```

點擊類別 → 篩選中央欄的詞條列表

**下半部：篩選器 + 搜尋**

```
搜尋：[______________]
階級模式： [全部▾] scaling / fixed / choice
設計狀態： [全部▾] complete / partial / pending
適用卡型： [全部▾] 七種子類型 + all_asset
```

**最下方：新增按鈕**

```
[+ 新增詞條]  [📥 匯入範本]
```

### 1.2 中央欄：詞條編輯主區

中央欄為當前選中詞條的**完整編輯介面**。採用分段式卡片設計：

```
┌── 詞條編輯：利刃 ──────────────────────────────┐
│                                                │
│  ▼ 基本資訊                                     │
│    代碼       [blade_____________] ※ 唯讀      │
│    名稱（中） [利刃______________]              │
│    名稱（英） [Blade_____________]              │
│    類別       [礦物 ▾] ※ 唯讀（建立後不可改）  │
│    階級模式   [scaling ▾]                      │
│    設計狀態   [complete ▾]                     │
│                                                │
│  ▼ 效果敘事                                     │
│    [✨ AI 敘事輔助]                             │
│    中文敘述   [傷害 +X________________]         │
│    英文敘述   [Damage +X______________]         │
│                                                │
│  ▼ 適用卡片子類型                               │
│    ☑ 近戰武器（weapon_melee）                  │
│    ☑ 遠程物理武器（weapon_ranged）             │
│    ☐ 戰鬥法術（weapon_arcane）                 │
│    ☐ 魔法道具（arcane_item）                   │
│    ☐ 一般道具（item）                          │
│    ☐ 消耗品（consumable）                      │
│    ☐ 光源（light_source）                      │
│    ☐ 全資產（all_asset）※ 勾選後其餘變唯讀     │
│                                                │
│  ▼ 階級設計（scaling: 3 階級必填）              │
│    ┌──────────────────────────────┐            │
│    │ 階級  │ V 值 │ 效果敘述       │            │
│    ├──────────────────────────────┤            │
│    │ +1    │ 1.0V │ 傷害 +1       │            │
│    │ +2    │ 2.0V │ 傷害 +2       │            │
│    │ +3    │ 3.0V │ 傷害 +3       │            │
│    └──────────────────────────────┘            │
│    [編輯階級] [重設為建議值]                    │
│                                                │
│  ▼ 備註                                         │
│    [設計備註自由欄位__________________]         │
│                                                │
│  [💾 儲存]  [🗑 刪除]  [📋 複製]                │
│                                                │
└──────────────────────────────────────────────┘
```

**關鍵互動：**

- **`tier_mode` 切換警告**：若從 `scaling` 切到 `fixed`，系統警告「這會移除 2 個階級紀錄，確定嗎？」
- **階級表格行內編輯**：點擊 V 值或效果敘述即可行內編輯
- **重設為建議值**：依類別與詞條類型套用規則書第四章第四節的預設 V 值

### 1.3 右欄：詞條試算與參考

右欄為**唯讀的即時資訊面板**，顯示與當前詞條相關的試算與參考資料。

**「預覽鍛造結果」按鈕**：彈出一個對話框，模擬「把某張具體卡片鍛造上此詞條」後的完整卡面呈現。用於設計者驗證詞條與卡片的視覺搭配。

### 1.5 choice 模式的專用子對話框

當 `tier_mode = 'choice'`（蟲類、元素附魔），階級編輯器需顯示 `choice_payload` 的結構化欄位：

```
╔═══════════════════════════════════════════════════╗
║  編輯選項 — 蟲淬 II                                ║
║                                                   ║
║  【中毒選項】                                       ║
║  標籤：[poison__]  排序：[1]  V 值：[3.0]          ║
║  效果：[命中施加中毒 1 層________]                 ║
║  Payload：                                        ║
║    狀態類型：[中毒（poison）▾]                     ║
║    層數：    [1]                                   ║
║  [🗑 刪除]                                         ║
║                                                   ║
║  【燃燒選項】                                       ║
║  ...                                              ║
║                                                   ║
║  [+ 新增選項]                                      ║
╚═══════════════════════════════════════════════════╝
```

**狀態類型下拉選單**：列出規則書第三章的完整狀態清單（流血、中毒、燃燒、冷凍、潮濕、脆弱、弱化、繳械、疲勞、沈默、發瘋、無力等）。

**元素附魔的 Payload**：類型改為「元素」下拉（火、冰、雷、神秘，不含物理）。

## 五、Gemini AI 敘事輔助

### 5.1 設計原則

沿襲 MOD-11 的「敘事留白」哲學：

| AI 會做 | AI 不會做 |
|---------|----------|
| 詞條的名稱建議 | 指定 V 值 |
| 詞條的效果敘述文字 | 指定 tier 結構 |
| 配方的解鎖敘事 | 選擇產出卡片 |
| 素材的名稱與傳敘 | 決定素材數量 |

### 5.2 模式 A：詞條敘事建議

**觸發：** 詞條編輯器的「✨ AI 敘事輔助」

**輸入：**
```json
{
  "affix_id": "uuid",
  "category_code": "mineral",
  "tier_mode": "scaling",
  "tiers": [
    { "tier_label": "+1", "affix_value": 1.0 },
    { "tier_label": "+2", "affix_value": 2.0 },
    { "tier_label": "+3", "affix_value": 3.0 }
  ],
  "applicable_subtypes": ["weapon_melee", "weapon_ranged"],
  "user_hint": "這個詞條代表將武器打磨到極為鋒利"
}
```

**系統 Prompt：**

```
你是克蘇魯神話卡牌遊戲的鍛造系統文案設計師。

請為一個鍛造詞條生成：
1. 中文名稱（2–4 字，簡潔有力）
2. 英文名稱
3. 中文效果敘述（明確機制描述，避免花俏文字）
4. 英文效果敘述

詞條基本資訊：
- 素材類別：{category_code}（{category_zh}）
- 階級模式：{tier_mode}
- 階級數量：{tier_count}
- 適用卡片：{applicable_subtypes_zh}
- 使用者提示：{user_hint}

【風格要求】
- 1920–30 年代氛圍
- 不輕佻、不戲謔
- 機制敘述使用明確詞彙（如「+X 傷害」「使用時獲得護甲 X 層」）
- 避免空泛詞彙（如「變強」「更厲害」）

輸出格式（JSON）：
{
  "suggested_names": [
    { "name_zh": "利刃", "name_en": "Blade", "reason": "強調鋒利的本質" },
    { "name_zh": "銳化", "name_en": "Sharpen", "reason": "..." },
    { "name_zh": "磨礪", "name_en": "Hone", "reason": "..." }
  ],
  "suggested_descriptions": [
    {
      "description_zh": "傷害 +X",
      "description_en": "Damage +X",
      "tone": "直接機制描述"
    }
  ]
}
```

**前端呈現：** 提供 2–3 個命名選項，設計者挑選後可再手動編輯。

### 5.3 模式 B：配方解鎖敘事

**觸發：** 配方編輯器的「✨ AI 解鎖敘事」按鈕

**輸入：**
```json
{
  "recipe_id": "uuid",
  "recipe_name_zh": "醫療包",
  "output_card": { "name_zh": "急救包", "card_type": "asset" },
  "materials": [
    { "category": "mineral", "level_min": 3, "qty": 2 },
    { "category": "wood",    "level_min": 1, "qty": 1 }

  "unlock_type": "exploration",
  "user_hint": "這應該是在診所場景的藥櫃中獲得"
}
```

**系統 Prompt：**

```
你是克蘇魯神話冒險遊戲的敘事設計師。

請為一個製作配方撰寫解鎖敘事。解鎖敘事是玩家在遊戲中如何取得此配方的故事
情境。

配方資訊：
- 配方名稱：{recipe_name_zh}
- 產出：{output_card_name}
- 解鎖類型：{unlock_type}
- 使用者提示：{user_hint}

【敘事要求】
- 40–80 字中文
- 提供具體場景或情境（不要抽象）
```

```
【命名原則】

  "suggestions": [
```
