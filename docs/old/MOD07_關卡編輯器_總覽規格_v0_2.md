## 一、模組定位

### 1.1 本模組要解決的問題

關卡編輯器是**實際可玩單元的設計工具**。MOD-06 給出骨架,MOD-07 把骨架填滿血肉。它負責的是:

- 設計**主線關卡**:地圖佈局、目標牌堆、議案牌堆、遭遇卡池引用、混沌袋配置、怪物家族池、神話卡池引用
- 設計**支線預設關卡**:機制同主線但結果二元(通過／失敗)
- 設計**支線重返版**:以覆寫方式微調原始支線的參數與敘事
- 設計**隨機地城生成器**:輸出的不是關卡實例,而是一組產生器參數,執行期即時生成
- 定義**場景序列**:關卡內從起始場景切換到下一場景的地圖操作指令

### 1.2 邊界

| 是 MOD-07 的職責 | 不是 MOD-07 的職責 |
|---|---|
| 關卡本體(結構、牌堆、場景序列) | 所屬章節的敘事、分支(交給 MOD-06) |
| 目標牌／議案牌的正反面設計 | 通關後的章節結果判定(交給 MOD-06) |
| 地圖操作指令(`place_tile` 等的編寫) | 地點本身的定義(交給 MOD-08) |
| 關卡起始混沌袋配置 | 戰役整體混沌袋的演變規則(交給 MOD-06) |
| 怪物家族池指派 | 怪物的數值與招式(交給 MOD-03) |
| 神話卡池與遭遇卡池的**引用選擇** | 神話卡、遭遇卡、傳奇行動本身(交給 MOD-10) |
| 關卡進入所需旗標、通關授予旗標 | 旗標字典本身(交給 MOD-06) |

### 1.3 與其他模組的對接

- **MOD-06**:關卡必須綁定所屬章節;所授予／讀取的旗標必須在戰役旗標字典中存在
- **MOD-08**:地點以引用方式使用;關卡只設計「用哪些地點、怎麼擺、怎麼連」
- **MOD-03**:怪物家族以代碼引用;關卡指定家族池,城主執行期挑選實際怪物
- **MOD-10**:神話卡池、遭遇卡池、傳奇行動皆為 MOD-10 資料;關卡僅引用

## 二、功能範圍

### 2.1 關卡管理

- 建立／編輯／刪除關卡
- 關卡類型選擇:主線(`main`)／預設支線(`side`)／支線重返版(`side_return`)／隨機地城(`side_random`)
- 關卡元資料:名稱(中英)、關卡代碼、敘事定位、設計狀態、版本
- 所屬章節(主線必填;支線留空)
- 進入所需旗標(支援旗標組合表達式;留空為無條件)
- 通關授予旗標(主線必填;支線不產生旗標)
- 人數縮放參數(`scaling_rules`)
- 設計完成度檢查

### 2.2 地圖與場景序列編輯器

關卡包含**一個或多個場景**。每個場景是關卡內某一階段的地圖狀態快照,**場景之間不主動切換**——場景切換完全透過目標牌／議案牌背面結算的地圖操作指令實現(即呼叫多次 `place_tile`、`remove_tile`、`connect_tiles` 等)。

- **場景序列**:第一個為起始場景,後續場景僅作為設計師的**地圖狀態備忘錄**與 AI 生成上下文,不具執行期切換動詞
- **場景資料**:場景編號、名稱、敘事描述
- **起始場景配置**(對應執行期實際會渲染的初始地圖):
  - 從地點池(MOD-08)挑選初始地點清單
  - 設定初始連接關係(在地點本身的 `connections` 基礎上,可追加／覆寫)
  - 設定調查員初始位置(指定哪個地點為調查員進場點)
  - 初始光照／火況配置
  - 初始怪物生成(指定哪些地點預先放置哪些家族的怪物)
- **非起始場景**:僅作為設計備忘錄,記載「目標 1 翻面後地圖應該長這樣」;執行期靠目標卡背面指令達成這個狀態
- **地圖預覽**:以圖形方式顯示每個場景的預期地點佈局與連接(SVG 繪製)

### 2.3 地圖操作指令編輯器

**共用的指令編寫工具**,出現在兩個地方:
- 目標牌背面結算
- 議案牌背面結算

支援的指令動詞(取自第三章 §5.2 環境類):

| 指令 | 參數 | 用途 |
|---|---|---|
| `place_tile` | 地點代碼、放置位置 | 放置新地點進場 |
| `remove_tile` | 地點代碼 | 移除地點 |
| `reveal_tile` | 地點代碼 | 翻開地點(揭開隱藏資訊) |
| `flip_card` | 目標 | 翻面卡片(目標／議案／地點) |
| `connect_tiles` | 地點 A、地點 B | 建立兩地點連接 |
| `disconnect_tiles` | 地點 A、地點 B | 斷開兩地點連接 |
| `spawn_enemy` | 家族代碼、位階、地點 | 生成敵人 |
| `place_enemy` | 家族代碼、位階、地點 | 直接放置敵人 |
| `create_light` / `extinguish_light` | 地點 | 光源操作 |
| `create_darkness` / `remove_darkness` | 地點 | 黑暗操作 |
| `create_fire` / `extinguish_fire` | 地點 | 失火操作 |
| `place_clue` | 地點、數量 | 放置線索 |
| `advance_act` / `advance_agenda` | — | 推進目標／議案牌堆 |

每個指令的參數欄位依指令類型動態產生(類似 MOD-01 效果編輯器的動態欄位模式)。

**整組切換場景**的慣用寫法:一組 `remove_tile` + 一組 `place_tile` + 一組 `connect_tiles` 組成完整的「搬場景」指令集。本編輯器不另設場景切換動詞,依慣例寫即可。

### 2.4 目標牌堆編輯器(ACT)

目標牌堆推進玩家端進度。每張目標卡有正反面。

**目標卡正面**:
- 目標卡編號(牌堆中的第幾張)
- 目標卡名稱(中英)
- 任務目標敘事
- 任務類型(從五大任務類型中選:`seal_gate` / `defeat_titan` / `uncover_truth` / `escape` / `endurance`;可混搭)
- 推進條件(條件表達式:花費 X 線索、完成特定行動、擊殺指定敵人等)
- 人數縮放(某些條件值乘以玩家人數)

**目標卡背面**:
- 完成結算敘事
- 授予旗標清單
- 結算獎勵(經驗值、天賦點、資源等立即獎勵)
- **地圖操作指令清單**(觸發場景切換或地圖變化)
- 結算指引代碼(給下一張目標卡或章節結算用)

### 2.5 議案牌堆編輯器(Agenda)

議案牌堆推進敵人端計畫。每張議案卡有正反面。

**議案卡正面**:
- 議案卡編號
- 議案卡名稱(中英)
- 劇情敘事
- **毀滅門檻**(當毀滅標記累積到此數值時翻面)

**議案卡背面**:
- 後果結算敘事
- 授予旗標清單
- 懲罰(扣 HP／SAN、生成敵人、扣線索等)
- **地圖操作指令清單**
- 結算指引代碼

### 2.6 遭遇卡池引用

**遭遇卡本體由 MOD-10 設計**。本模組僅做引用配置:

- 從 MOD-10 既有的遭遇卡庫挑選本關卡適用的遭遇卡清單
- 每張遭遇卡的抽取權重(控制抽取機率)
- 本關卡是否啟用遭遇牌堆(規則書第五章 §2.1 提到「部分關卡可不設」)

### 2.7 混沌袋配置

關卡起始混沌袋的組成。

- 四類標記完整配置:數字標記、情境標記(主題選擇)、神話標記(分佈)、極端標記
- 難度預設套用(從第六章 §12 的「簡單／標準／困難／專家」配置中選一,可微調)
- 情境標記的場景效果指派(從預設選項中為每類情境標記配置具體效果)
- **怪物家族主題對應**:系統依第六章 §13「怪物家族與混沌袋場景效果對應表」推薦情境標記效果
- 混沌袋整體數值預覽(正負期望值、方差)

### 2.8 怪物家族池指派

關卡不指定具體怪物,只指定家族池。

- 主家族選擇(1–2 個,對應戰役主題)
- 副家族選擇(可選,用於製造意外)
- 每家族在本關卡允許的位階範圍(例如主家族最高到精英、副家族只出雜兵)
- 頭目／巨頭指定(若本關卡設計有頭目戰,直接指定具體敵人代碼)
- 城主執行期的選擇權限說明(哪些參數由城主即時決定)

### 2.9 神話卡池引用

**神話卡(含傳奇行動類的神話卡)本體由 MOD-10 設計**。本模組僅做引用配置:

- 從 MOD-10 既有的神話卡庫挑選本關卡適用的神話卡清單
- 每張神話卡的抽取權重
- 神話卡自身的觸發條件不在此配置(屬於神話卡自身屬性,在 MOD-10 定義)

### 2.10 支線重返機制(`side_return`)

重返版關卡**以覆寫方式微調原始支線**,而非完整複製。

- 關聯的原始支線關卡代碼(`return_parent_id`)
- 重返次數階段(第 1 次重返、第 2 次、…;可為每階段設計不同覆寫)
- **Overrides 覆寫結構**(JSON Merge Patch 語意:有定義即覆寫,未定義則沿用原始支線):
  - 關卡元資料覆寫(敘事定位、進入旗標)
  - 目標牌覆寫(依牌序定位;可覆寫正反面敘事、推進條件、獎勵、地圖操作指令)
  - 議案牌覆寫(依牌序定位;可覆寫正反面敘事、毀滅門檻、懲罰、地圖操作指令)
  - 怪物家族池覆寫(包含劇情專屬頭目／巨頭的替換)
  - 混沌袋覆寫(數值與標記配置的調整)
  - 神話卡池覆寫
  - 獎勵規則覆寫
- 執行期讀取流程:先讀原始支線完整配置 → 套用 overrides 合併 → 產出當前重返版的執行配置
- 獎勵規則:**簽名卡只在首次通關給予,後續僅給予經驗值／天賦點／素材／凝聚力**(此規則由執行期處理,設計器不需重複定義)

### 2.11 隨機地城生成器

設計師產出的不是關卡實例,而是**產生器規則**。執行期由後端 API(`/api/random-dungeons/:id/generate`)即時生成。

- **地點池配置**:從 MOD-08 挑選候選地點清單,每個地點含權重
- **地圖拓撲規則**:
  - 地點數量範圍(最小、最大)
  - 連接密度(每地點平均幾條連接)
  - 入口地點規則(從哪類地點標籤中挑)
  - 出口／目標地點規則
  - 拓撲形狀偏好(線性／樹狀／網狀)
- **目標牌堆生成規則**:
  - 目標模板槽位(第幾張目標卡的任務類型)
  - 模板池引用(從哪些目標模板中隨機挑選)
- **議案牌堆生成規則**:同上
- **怪物家族池**:家族選擇 + 每家族的生成密度
- **混沌袋配置**:固定或依難度參數浮動
- **勝利條件類型**:從五大任務類型混搭
- **獎勵規則**:經驗值／天賦點／素材／凝聚力的基礎量 + 難度加乘
- **種子驗證器**:輸入隨機種子,預覽會生成什麼樣的關卡(供設計師驗證產生器規則合理);與執行期生成邏輯共用同一後端 API,確保設計師預覽即是玩家將玩到的

## 三、資料結構概觀

### 3.1 `stages`(關卡)

| 欄位 | 類型 | 說明 |
|---|---|---|
| `id` | UUID | 主鍵 |
| `chapter_id` | UUID | 外鍵(支線留 NULL) |

| `role` | VARCHAR(16) | `primary` / `secondary` |
| `allowed_tiers` | VARCHAR[] | 允許的位階清單 |
| `fixed_boss_ids` | UUID[] | 固定指定的頭目／巨頭 ID |

### 3.9 重返版覆寫結構(`return_overrides` JSONB)

重返版採 **JSON Merge Patch** 語意:有定義即覆寫原始支線對應欄位,未定義則沿用。支援的覆寫路徑:

```json
{
  "stage_metadata": {
    "narrative": "重返版的新敘事"
  },
  "act_cards": {
    "1": {
      "front_narrative": "重返版的新正面敘事",
      "front_advance_condition": { /* 新推進條件 */ },
      "back_rewards": { /* 新獎勵 */ }
    },
    "2": {
      "back_map_operations": [ /* 新地圖指令 */ ]
    }
  },
  "agenda_cards": {
    "1": {
      "front_doom_threshold": 6
    }
  },
  "monster_pool": {
    "primary_family_tier_adjustment": "+1",
    "fixed_boss_replacement": {
      "original_boss_id": "<UUID>",
      "new_boss_id": "<UUID>"
    }
  },
  "chaos_bag": {
    "additions": [{ "op": "add", "marker": "skull", "count": 2 }]
  },
  "mythos_pool": {
    "add_cards": [{ "mythos_card_id": "<UUID>", "weight": 3 }],
    "remove_cards": [{ "mythos_card_id": "<UUID>" }]
  },
  "encounter_pool": {
    "add_cards": [{ "encounter_card_id": "<UUID>", "weight": 2 }]
  }
}
```

> **覆寫鍵的命名**:目標卡／議案卡以 `card_order` 為鍵(字串形式)。若要新增原始支線沒有的卡,以 `"new_1"`、`"new_2"` 為鍵(合併時追加到牌堆末尾)。

## 四、介面結構

### 4.1 整體佈局

沿用 `admin-shared.css` 暗黑哥德風格,三欄式佈局:

- **左側欄**:關卡列表(可依章節／類型篩選)
- **主編輯區**:依當前分頁顯示對應編輯器
- **右側欄**:關卡預覽卡 + 地圖視覺化預覽

### 4.2 分頁結構

| 分頁 | 主編輯區內容 |
|---|---|
| 關卡總覽 | 關卡元資料、類型、所屬章節、進入／通關旗標 |
| 場景序列 | 場景清單、起始場景配置、非起始場景的地圖備忘錄 |
| 目標牌堆 | 目標卡清單、正反面編輯 |
| 議案牌堆 | 議案卡清單、正反面編輯 |
| 遭遇卡池 | 從 MOD-10 遭遇卡庫選取 + 權重 |
| 神話卡池 | 從 MOD-10 神話卡庫選取 + 權重 |
| 混沌袋配置 | 四類標記配置、難度預設、情境標記效果指派 |
| 怪物家族池 | 主／副家族、位階範圍、固定頭目指定 |
| 重返覆寫 | 僅 `side_return` 類型顯示;以原始支線為底的差異編輯器 |
| 隨機地城規則 | 僅 `side_random` 類型顯示;產生器參數配置 + 種子驗證器 |
| 完整性檢查 | 關卡發佈前檢查清單 |

### 4.3 地圖操作指令編輯器(共用元件)

這是本模組最複雜的 UI 元件。以清單形式編輯一組指令序列。

每一條指令:
- 左側選動詞(13 種地圖操作動詞)
- 右側依動詞動態顯示參數欄位
- 支援拖拉排序(指令執行順序重要)
- 支援複製、刪除、停用

參數欄位的動態產生方式與 MOD-01 卡片設計器的效果動態欄相同:依動詞代碼查 `map-operations-options.json` 資料檔決定要渲染哪些欄位。

### 4.4 地點挑選對話框(共用元件)

出現在多處:
- 場景初始地點配置
- 地圖操作指令的地點參數
- 隨機地城的地點池配置
- 重返覆寫中新增的地點

依情境啟用不同模式:
- **指定模式**(主線):選擇一個或多個具體地點
- **權重模式**(隨機地城):選擇地點 + 為每個地點設定權重
- **替換模式**(重返覆寫):選定原地點後指定替換為哪個新地點

### 4.5 地圖視覺化預覽

- 以 SVG 繪製當前場景的地點佈局
- 節點:地點(顯示名稱、調查難度、線索數)
- 邊:連接關係
- 顏色區分:調查員起點、隱藏地點、含怪物的地點
- 可切換顯示不同場景
- 執行預覽:模擬某張目標牌翻面後地圖變化(執行一組指令看結果)

### 4.6 重返覆寫編輯器

特殊的差異式編輯介面:

- 左半部顯示原始支線的當前設定(唯讀)
- 右半部顯示重返版的覆寫值(可編輯)
- 欄位未覆寫時顯示灰色佔位符「沿用原始值」
- 編輯後右半部變為實值,並有「還原為沿用」按鈕
- 支援重返次數分階段設計(第 1 次重返、第 2 次、…各自一組 overrides)

### 4.7 關鍵操作流程

**建立新主線關卡**
1. 選擇所屬章節(從 MOD-06 章節下拉)
2. 填寫關卡元資料
3. 設定進入所需旗標(從戰役旗標字典中選)
4. 進入場景序列分頁,建立起始場景
5. 進入目標牌堆分頁,至少設計 2 張目標卡
6. 進入議案牌堆分頁,至少設計 2 張議案卡
7. 設定混沌袋配置(可選擇難度預設後微調)
8. 指派怪物家族池
9. 從 MOD-10 挑選遭遇卡池與神話卡池
10. 設定通關授予旗標(檢查與戰役字典一致)
11. 完整性檢查通過即可發佈

**建立支線重返版**
1. 選擇關卡類型 `side_return`
2. 指定原始支線關卡
3. 進入重返覆寫分頁,以差異編輯方式微調目標卡、議案卡、怪物、混沌袋、神話卡池等
4. 其餘結構自動沿用原始支線

**建立隨機地城產生器**
1. 選擇關卡類型 `side_random`
2. 填寫基礎元資料(無章節綁定、無進入旗標、無通關旗標)
3. 進入隨機地城規則分頁
4. 配置地點池、拓撲規則、模板池、怪物規則、神話／遭遇卡池規則、獎勵規則
5. 用種子驗證器預覽幾組生成結果,驗證產生器合理
6. 完整性檢查通過即可發佈

## 七、完成度檢查規則

### 7.1 主線關卡可發佈條件

- 元資料齊備(名稱、代碼、類型、所屬章節)
- 進入所需旗標在戰役字典中存在(若設定)
- 通關授予旗標在戰役字典中存在(至少 1 個)
- 至少 1 個場景(起始場景含初始地點與調查員進場點)
- 至少 2 張目標卡
- 至少 2 張議案卡
- 混沌袋配置完整(四類標記均有配置)
- 怪物家族池至少 1 個主家族
- 神話卡池至少 1 張神話卡(引用 MOD-10)
- 人數縮放參數齊備
- 所有地圖操作指令引用的地點代碼在 MOD-08 中存在
- 所有引用的怪物家族代碼在 MOD-03 中存在
- 所有引用的旗標代碼在 MOD-06 戰役字典中存在
- 所有引用的神話卡與遭遇卡在 MOD-10 中存在

### 7.2 支線預設關卡可發佈條件

- 元資料齊備(不含章節綁定)
- **不產生任何旗標**(檢查 `completion_flags` 為空)
- 至少 1 個場景、2 張目標卡、2 張議案卡
- 混沌袋配置完整
- 怪物家族池、神話卡池、人數縮放參數齊備
- 可指定支線專屬簽名卡(選配)

### 7.3 支線重返版可發佈條件

- 已指定 `return_parent_id`(對應的原始支線關卡存在且可發佈)
- 至少一階段的 overrides 有實質變化
- Overrides 中引用的所有外部代碼(地點、怪物、神話卡等)均存在於對應模組

### 7.4 隨機地城產生器可發佈條件

- 地點池至少 5 個候選地點
- 拓撲規則地點數量範圍合理(最小 ≥ 3,最大 ≤ 最小 × 3)
- 目標模板池、議案模板池至少各 1 套模板
- 怪物家族規則至少 1 個家族
- 混沌袋規則完整
- 勝利條件至少 1 種
- 獎勵規則齊備
- 種子驗證器至少執行過 1 次且成功生成
