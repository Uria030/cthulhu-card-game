## 五、MOD-02 天賦樹設計的規格搜尋過程（重要教訓）

### 問題

天賦樹的具體結構（12 級、分支路線、八陣營概覽等）在規則書 v1.0 六章中找不到。規則書只提到天賦樹的存在和定位（五條成長路徑之一），但沒有具體設計章節。第六章待設計清單明確標記「天賦點獲取量與天賦樹結構 — 依賴：陣營設計 — 中優先級」。

### 搜尋經歷

Uria 告知天賦樹設計存在於 GDD05 聊天室。Claude 進行了多次搜尋但回報「找不到」。Uria 反覆要求搜尋共超過三次，Claude 每次都聲稱搜過但沒有。

**根本原因：** `conversation_search` 工具是基於關鍵字匹配的文本搜索，回傳結果有限且會優先匹配其他聊天室（GDD03 的大量內容佔據了搜索結果）。GDD05 的天賦樹設計內容雖然存在，但被其他聊天室的高頻率匹配結果淹沒。

**最終解決：** Uria 直接指示 Claude 閱讀 GDD05 聊天室。透過 `recent_chats` 找到 GDD05 的 URL（`https://claude.ai/chat/a98bc230-3ddb-43bd-bcae-b13da74e4625`，更新於 2026-04-13T06:07:24），並從搜尋結果片段中成功提取了天賦樹的完整設計要點。

**教訓：** 
1. 搜索工具有盲區，搜不到不代表不存在。
2. 應該在第一次被告知時就信任使用者的指向。
3. GDD05 產出了 `成長子系統設計_Growth_Subsystem_v0_1.md`，但此文件在規則書 v1.0 整合時被遺漏。

### GDD05 中確認的天賦樹設計要點

- 創角總點數 18 點（非 21 點），差額透過天賦樹的 5 次屬性提升補回
- 天賦樹 12 級通用節奏：3/6 級質變、5/8 級專精、5 次屬性提升
- 每陣營 3 條分支路線，12 級終極天賦依分支不同
- 八陣營天賦樹概覽含戰鬥熟練傾向、基礎被動、三條分支方向
- 天賦點 vs 經驗值 = 質變 vs 量變
- 卡片等級上限受陣營天賦等級限制
- I/T 都對應智力、N/F 都對應意志
- 天賦卡都是簽名卡
- 12 級 = 超凡入聖，滿等角色能正面對抗巨頭
- 八陣營全滿需 96 天賦點（約 8 個戰役）
