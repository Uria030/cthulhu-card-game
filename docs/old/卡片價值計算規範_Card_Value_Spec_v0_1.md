# 卡片價值計算規範 v0.1
## Card Value Specification v0.1

> **文件用途｜Purpose**
> 本文件定義卡片效果的價值計算方式、費用公式、等級與稀有度抵扣機制。
> 所有卡片設計皆以本文件為數值平衡的依據。

---

## 一、基礎單位｜Base Unit

> **1V = 1 行動點 = 1 資源 = 抽 1 張牌 = 造成 1 點傷害**

此為價值計算的基準單位，所有效果價值皆以此為錨點推導。

## 三、費用公式｜Cost Formula

### 3.1 事件卡

> **費用 = 總價值 - 等級抵扣 - 稀有度抵扣 - 負面副作用**

- 換算比例：**1:1**
- 打牌成本 = 1 行動點 = 1V（卡片效果需 > 1V 才值得打出）

### 3.2 資產卡

> **費用 = （總價值 - 留場修正 - 消耗類型修正 - 等級抵扣 - 稀有度抵扣）÷ 2**

- 換算比例：**2:1**
- 總價值 = 單次效果價值 × 預期使用次數

### 3.3 盟友卡

> **費用 = 總價值 - 等級抵扣 - 稀有度抵扣**

- 換算比例：**1:1**
- 總價值 = HP 價值 + SAN 價值 + 留場價值 + 能力價值

| 項目 | 價值 |
|------|------|
| 盟友 1 HP | 0.5V |
| 盟友 1 SAN | 0.5V |
| 留場價值 | 2V |
| 持續被動（單屬性 +1） | 0.5V |
| 有代價的能力 | 0V |

### 3.4 技能卡

- 費用固定為 **0**（不花資源，代價是佔手牌空間）
- 價值體現在等級，用經驗值購買

## 四、修正值｜Modifiers

### 4.1 等級抵扣

| 等級 | 抵扣 |
|------|------|
| 0 級 | 0 |
| 1 級 | -1V |
| 2 級 | -2V |
| 3 級 | -3V |
| 4 級 | -4V |
| 5 級 | -5V |

升級時卡片可能同時獲得能力提升（價值增加），最終費用 = 新總價值 - 等級抵扣，不一定變便宜。

### 4.2 稀有度抵扣

| 稀有度 | 抵扣 |
|--------|------|
| 隨身 | 0 |
| 基礎 | -1V |
| 標準 | -2V |
| 進階 | -3V |
| 稀有 | -4V |
| 傳奇 | -5V |

稀有度有雙重作用：
1. **抵扣價值** → 降低費用
2. **允許更強效果** → 決定能力價值上限

### 4.3 消耗類型修正（暫定）

| 消耗類型 | 修正 | 說明 |
|----------|------|------|
| 留在場上（stay） | -1V | 打出當下無即時價值 |
| 進入棄牌堆（discard） | 0 | 基準，洗牌後可再抽到 |
| 短休息回復（short_rest） | -1V | 比 discard 受限 |
| 長休息回復（long_rest） | -2V | 比短休息更受限 |
| 移除出遊戲（removed） | -3V | 最嚴格，只能用一次 |

消耗限制用於平衡過強效果：效果越強 → 使用限制越嚴格 → 價值修正越多 → 費用平衡

### 4.4 條件限制

> 條件限制**不影響價值計算**

例如「交戰中才能使用」「HP ≤ 一半才能使用」等條件不調整價值。

### 4.5 負面副作用

> 直接扣除對應效果價值作為代價

例如：
- 「受到 1 點傷害」= -1.5V
- 「放置 1 毀滅標記」= -4V
- 「棄 1 張手牌」= -0.5V
- 「對盟友造成 1 點恐懼」= -0.5V

## 六、能力價值計算原則｜Ability Value Principles

| 能力類型 | 價值計算 |
|----------|----------|
| 有代價的能力 | 0V（代價抵消價值） |
| 無代價的被動 | 依效果價值表計算（不乘次數） |
| 無代價的主動/反應 | 依效果價值表計算 |
| 二選一效果 | 取較高價值的效果 |
| 條件觸發效果（如「成功時」） | 依效果價值表計算（條件不打折） |

> **給未來 Claude 實例的備註｜Note to Future Claude Instances**
>
> 本文件的所有數值均經過與專案原創者 Uria 的逐項討論確認。
> 特別注意以下設計精神：
>
> 1. **1V = 1 行動點 = 1 資源 = 抽 1 張牌 = 造成 1 點傷害** — 這是一切計算的基準。
> 2. **檢定加值是曲線成長** — +1/+2/+3 不是等比增加，高加值價值更高。
> 3. **單次攻擊傷害加成遠高於直接傷害** — 因為疊加在攻擊上效益更大。
> 4. **恢復 > 取消** — 已受傷回復比事前取消更珍貴。
> 5. **有代價的能力 = 0V** — 代價抵消價值，不額外計價。
> 6. **稀有度雙重作用** — 既抵扣價值，也決定能力強度上限。
> 7. **資產卡換算 2:1，事件/盟友卡換算 1:1** — 因為資產可多次使用。
> 8. **條件限制不影響價值** — 「成功時」「交戰中」等條件不打折。
