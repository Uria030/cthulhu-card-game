## 一、專案身份｜Project Identity

**專案類型｜Type：** 克蘇魯神話世界觀・卡牌驅動・牌組構築・回合制・合作冒險・網頁遊戲

**核心精神一句話｜Elevator Pitch：**
你帶著一把手槍、六發子彈和一瓶威士忌走進迷霧，然後在黑暗裡一點一點找到活下去的方法。

**目標平台｜Platform：** 網頁瀏覽器（響應式，行動裝置相容）
**玩家人數｜Players：** 1–4 人合作（從第一天起就納入多人架構設計）
**戰鬥節奏｜Pacing：** 慢節奏桌遊級策略（每場戰鬥 15+ 分鐘）
**美術方向｜Art Direction：** 暗黑哥德奇幻，1920–30 年代偵探黑色電影 × 宇宙恐怖

## 二、八大設計支柱｜Eight Design Pillars

以下八根支柱是本專案的結構基礎。任何設計決策若與支柱矛盾，支柱優先。

### 支柱 1：陣營與構築｜Faction & Deckbuilding
- 調查員創建時綁定一個**主陣營**，完全存取該陣營卡池。
- **副陣營**在冒險開始時鎖定，需透過以下三種路徑之一在冒險中解鎖：
  - **劇情觸發** — 完成特定故事里程碑。
  - **資源累積** — 累積足夠的「禁忌洞察」等特殊貨幣。
  - **代價交換** — 永久犧牲生命上限或理智上限。
- 中立卡可供所有調查員使用。
- **詳細設計：** 見《支柱一詳細設計：陣營與構築》。

### 支柱 2：雙軌檢定｜Dual-Track Skill Checks
- **主要檢定：擲骰 + 修正值。** 玩家感受到成長帶來的數值提升，具備掌控感。
- **特殊檢定：混沌袋抽取。** 保留給宇宙恐怖時刻 — 直面舊日支配者、翻開禁忌典籍、神啟事件。無論多強都無法保證安全。
- **戰鬥中的應用：** 物理攻擊使用擲骰（掌控感），施法類攻擊使用混沌袋（玩火感）。檢定使用的屬性由**戰鬥風格卡**動態決定，而非由武器固定指定。詳見系統 E-2 與 E-3。
- **設計意圖：** 骰子給予掌控感，混沌袋剝奪掌控感。兩者交替使用創造情緒層次。

### 支柱 3：章節制冒險｜Campaign Chapters
- 每個戰役由約 **10 個章節**（場景）組成，劇情連貫。
- 章節間保持完整狀態持續（牌組、生命、理智、劇情旗標、資源、NPC 關係、混沌袋修改）。
- 通關後調查員可**存檔並帶入其他戰役**。

### 支柱 4：三層死亡｜Three-Layer Death
- **第一層 — 創傷侵蝕：** 被擊倒（HP 或 SAN 歸零）獲得創傷，永久降低上限。上限歸零 = 永久死亡。節奏：緩慢磨損。
- **第二層 — 神啟詛咒：** 特殊詛咒卡被洗入牌組，抽到即觸發，無法正常移除。特定高階詛咒可直接刪除角色。節奏：定時炸彈。
- **第三層 — 倒地瀕死：** HP 或 SAN 歸零時進入瀕死狀態。瀕死檢定沿用 D&D 規則（d20, DC 10），成功 3 次站起來，失敗 3 次死亡且上限 -1。隊友可花 1 行動點穩定（自動成功一次），或用治療效果直接解除瀕死。節奏：突發危機。**完整規則見《補充 03》第一章。**
- **永久死亡是真實的。角色死亡 = 資料刪除。** 這是克蘇魯遊戲的核心情感重量。

### 支柱 5：成長子系統｜Growth Subsystems
- 包含：卡片強化、裝備鍛造、道具製作、角色技能樹、陣營天賦。
- **所有系統的產出都是卡片或對卡片的修改** — 卡片是通用單位。
- 成長必須在世界中贏得，不能透過抽象選單獲取。
- **武器卡片的成長路線：** 基礎武器（等級 0）通常只有單一屬性修正。升級後可選擇廣度路線（所有檢定 +1）或深度路線（特定屬性 +2）。負面修正不出現在基礎卡上，只在玩家選擇強化路線時作為取捨出現。

### 支柱 6：動態牌組｜Dynamic Deckbuilding
- 冒險中持續獲得新卡，玩家即時決定是否納入牌組。
- 每一張加入的卡都是有重量的決策。

### 支柱 7：模組化架構｜Modular Architecture
- **調查員與關卡（場景）是兩個獨立模組**，可各自獨立設計與擴充。
- 擴充路線三軌：新調查員包 / 新冒險劇本 / 大型擴充（兩者皆有）。
- 提供**三大工具模組**：**創角器**（玩家 / 社群）、**關卡編輯器**（玩家 / 社群）、**系統管理員後台**（開發者 / 管理員，含數值模擬器群組）。

### 支柱 8：探險驅動構築｜Exploration-Driven Building
- **起始牌組精簡**（約 15–20 張），這不是完整裝備。
- **消耗品真實消耗** — 子彈打完就沒了，藥用了就少一張，牌組會「變薄」。
- **新卡透過探索場景獲得** — 搜索書櫃、翻查屍體、與 NPC 交易。不是戰鬥後的三選一獎勵畫面。
- **成長曲線由關卡設計與玩家選擇共同塑造。**

## 二之二、補充設計系統｜Supplementary Design Systems

以下系統是八大支柱的延伸，在確認後將視規模決定是否升格為正式支柱。

### 系統 A：凝聚力與團隊精神｜Cohesion & Team Spirit

固定成員組隊冒險時，隊伍可累積一種共用資源：**凝聚力（Cohesion）**。

- **團隊精神（Team Spirit）**：隊伍最多可從候選池（目標 32 種，待設計確認數量）中選擇 **7 種**團隊精神。
- 每種團隊精神有 **5 點**可投資，每點花費 1 點凝聚力。
- **採用**一個新的團隊精神項目時，也花費 1 點凝聚力（等於完整點滿一個項目共需 6 點凝聚力）。
- 當一個團隊精神項目完全點滿後，可解鎖**團隊里程碑（Team Milestone）**— 更強大的進階能力，能**直接讓隊伍的打法產生質變**。
- **設計意圖：** 獎勵固定組隊的長期合作，讓「一起走過地獄」的夥伴關係有機械上的實質回報。凝聚力的稀缺性迫使隊伍在廣度（多個團隊精神）與深度（點滿解鎖里程碑）之間抉擇。

### 系統 B：支線關卡與重返機制｜Side Scenarios & Return Mechanic

- 每一關主線完成後，或玩家單人上線時，可挑戰**支線小關卡（Side Scenario）**，由 1–3 個場景組成，可賺取經驗值或其他資源。
- 關卡設有**「重返」（Return）**設計：
  - 第一次遊玩與重返時，**內容會有所不同**（不同的遭遇、不同的敘事細節、可能不同的地圖配置）。
  - **每次重返難度都會提高**，提供持續挑戰與資源獲取的平衡。
- **設計意圖：** 提供主線之外的彈性遊玩空間，讓玩家可以自主選擇練習、賺取資源或單純體驗更多內容。重返機制確保重複遊玩不會感到無聊，同時難度遞增防止成為無風險的資源農場。

### 系統 C：休息機制｜Rest Mechanic

參考 D&D 的長休息 / 短休息概念，納入冒險節奏的調控：

- **短休息（Short Rest）**：回合開始時決定，犧牲整回合行動換取重洗牌庫、回收特定消耗品、觸發陣營專屬效果。**完整規則見《補充 01》第三章。**
- **長休息（Long Rest）**：每關結束後自動進入，獲得 1 點凝聚力，進入整備模式（天賦樹、技能樹、牌組調整等）。**完整規則見《補充 01》1.5 節。**
- **設計意圖：** 創造冒險中的節奏起伏 — 緊張探索後的短暫安寧，讓恐懼感有對比才更有效。同時作為資源管理的戰略節點：何時休息、在哪休息本身就是有意義的決策。短休息的代價是讓 Agenda 多推進一步。

### 系統 D：分支路線與非線性敘事｜Branching Paths & Non-Linear Narrative

- 每一關打完會產生**三種結果**（A / B / C），不同結果導向不同的後續路線。
  - 例如：第九關完成後，依結果可能進入 **10A 或 10B** 兩條不同的路線。
- 關卡結構**不一定是單線式**的：
  - 玩家可能在一個大區域中**自行決定前往哪裡冒險**。
  - 移動有時間成本（例如：花費數小時趕路），影響可用的行動時間或議程推進。
  - 這為場景設計帶來**策略選擇層**：先去危險但高報酬的地點，還是先去安全的地點穩固基礎？
- **設計意圖：** 大幅提升重玩價值與敘事沉浸感。玩家的選擇不只影響一場戰鬥的結果，而是重塑整個戰役後半段的走向。這也讓關卡編輯器需要支援分支邏輯與條件判斷。

### 系統 E：戰鬥系統設計｜Combat System Design

戰鬥是調查員與威脅正面交鋒的核心時刻。本系統融合三個設計來源的精髓，創造有深度、有敘事、有層次的戰鬥體驗。

#### E-1：戰鬥熟練與專精｜Combat Proficiency & Specialization

參考 D&D 的熟練技能概念，但專門應用於戰鬥情境：

- **第一層：戰鬥熟練（Proficiency）** — 對某類攻擊方式的基礎掌握，提供熟練修正。共 **8 種**。
- **第二層：戰鬥專精（Specialization）** — 從熟練中進一步特化的方向，提供更高修正或專精特有效果。共 **30 種**。
- **機制影響：** 發起攻擊時，若調查員擁有對應的熟練或專精，會獲得額外修正加成。不具備相關熟練的攻擊方式仍可使用，但數值上處於劣勢。
- **設計意圖：** 讓每位調查員在戰鬥中有清晰的角色定位與擅長領域，鼓勵團隊分工。同一把獵槍在神射手和學者手上應該有截然不同的效果。

**八種戰鬥熟練與 30 種專精完整列表：**

| 戰鬥熟練 | 專精方向 |
|---------|---------|
| **槍枝射擊**（Shooting） | 步槍、衝鋒槍、雙槍、手槍 |
| **弓術**（Archery） | 獵手、連射、毒箭、無聲射手 |
| **隨身武器**（Sidearm） | 匕首術、護身格擋、鈍擊、街頭格鬥 |
| **軍用武器**（Military） | 雙手武器、防禦架式、雙持、長柄武器 |
| **搏擊**（Brawl） | 酒館鬥毆者、摔角大師、空手道 |
| **施法**（Arcane） | 儀式、咒語、引導、冥想、煉金 |
| **工兵**（Engineer） | 爆破、陷阱、機械 |
| **暗殺**（Assassination） | 無聲處決、伏擊戰術、暗器 |

> **完整代碼定義見《補充 03》第四章。**

#### E-2：戰鬥風格卡與敘事式戰鬥｜Combat Style Cards & Narrative Combat

參考 Mansions of Madness 的戰鬥系統。戰鬥不是單純的「擲骰比數字」，而是**一段有場景描述的微型敘事**，透過**戰鬥風格卡**實現：

- **核心規則：誰發起攻擊，誰抽戰鬥風格卡。** 被攻擊方根據卡面指示做對應的檢定。
- **調查員攻擊時：** 依據武器的攻擊方式（例如「射擊」），從對應的風格卡池中隨機抽一張。卡面呈現敘事描述，並指定本次檢定使用的屬性。同一把手槍可能這次抽到力量檢定（穩穩握住開槍），下次抽到敏捷檢定（快速拔槍射擊），再下次抽到意志檢定（克服恐懼扣板機）。
- **怪物攻擊時：** 怪物從自己的招式池中抽取風格卡，卡面指定調查員要用哪個屬性做防禦檢定，同時標明 DC 和傷害值。調查員擲骰防禦。
- **武器屬性修正只在對應屬性被抽到時生效。** 例如 .45 手槍的「力量 +1」只在抽到力量檢定時有用，抽到敏捷檢定時無加成。
- **風格卡池是公用的** — 有無熟練的差異體現在修正值上，不是卡池內容。
- **調查員與怪物的風格卡是完全獨立的資料結構**，分別在 MOD-05（戰鬥熟練管理）和 MOD-03（敵人設計器）中設計。
- **卡面標籤是動態的** — 顯示調查員當前的熟練等級名稱（第一層顯示熟練名，第二層顯示專精名）。
- **敵人的行動同樣有敘事描述**，而非單純的「敵人對你造成 3 點傷害」。
- **設計意圖：** 戰鬥風格卡讓「萬能角色」不存在 — 再強的射手，抽到意志檢定時一樣心裡發毛。每一次攻擊自帶一段不同的敘事，戰鬥本身就是故事的一部分。這也與 15+ 分鐘的慢節奏策略定位一致 — 每個動作都值得被描述。

> **完整規則見《補充 03》第三章。**

#### E-3：雙軌攻擊解析｜Dual-Track Attack Resolution

將支柱 2（雙軌檢定）的哲學直接應用到戰鬥系統中，創造兩種截然不同的攻擊體驗：

- **物理攻擊 → 擲骰 + 修正值：** 槍械、近戰武器、體術、戰術行動。玩家有掌控感，熟練與專精提供穩定加成，成長有明確的數值回報。**檢定使用的屬性由戰鬥風格卡動態決定**，武器修正只在對應屬性時生效。
- **神秘 / 法術攻擊 → 混沌袋抽取：** 當調查員使用施法類攻擊方式時，不擲骰，改為**抽混沌袋**決定結果。這類攻擊通常更強大，但結果不可預測且可能反噬。
  - 對應 D&D 中高階攻擊「無法閃避，必須豁免」的概念 — 神秘攻擊繞過物理防禦，敵人改為進行豁免（同樣可能以混沌袋或擲骰處理）。
- **設計意圖：** 物理戰鬥給予「我在進步」的可靠感；神秘戰鬥給予「我在玩火」的不安感。選擇學習法術的調查員獲得了更強的火力，但代價是永遠無法完全掌控結果 — 這完美呼應克蘇魯世界觀中「觸碰禁忌力量」的本質。

#### E-4：隱藏資訊與觀察判斷｜Hidden Information & Observation

參考 Monster Hunter 與 D&D 的戰鬥資訊設計哲學：**敵人的數值對玩家是不透明的。**

- **敵人血量不顯示數字。** 玩家不會看到「深潛者 HP: 12/20」。取而代之的是**敘事性的狀態描述** — 「牠看起來毫髮無傷」「牠的動作開始遲緩」「牠拖著斷裂的肢體在地上爬行」「牠發出瀕死的嘶吼」。玩家必須透過觀察來判斷敵人還能撐多久。
- **敵人防禦屬性不直接公開。** 玩家的攻擊只會回饋**結果描述** — 「你的子彈打在牠的甲殼上彈開了」「利刃劃過牠的皮膚但只留下淺淺的痕跡」「火焰讓牠劇烈退縮」。玩家需要**自行觀察並判斷**敵人的防禦特性（物理抗性、元素弱點、部位差異等），進而調整戰術與出招選擇。
- **「感知」屬性的戰鬥價值：** 高感知的調查員或特定卡牌效果可以揭示更多線索 — 例如更精確的狀態描述、弱點提示、行為模式預告。這讓「觀察」本身成為一種有價值的戰鬥行動，而非只有輸出才有意義。
- **設計意圖：** 這是 Monster Hunter 讓戰鬥保持緊張感的核心方法 — 你永遠不能百分之百確定怪物還能撐幾下。搭配敘事式戰鬥（E-2），每一次攻擊的回饋都是一段需要解讀的訊息，而非一個可以精算的數字。這迫使玩家**真正投入戰鬥場景**，而不是盯著血條做數學。同時也強化了團隊溝通的價值 — 「牠對火焰有反應，換火焰彈試試！」

#### E-5：敵人 AI 行為規則｜Enemy AI Behavior

所有檢定都由玩家擲骰。城主（系統 AI）不會擲骰，怪物的行為由系統根據規則自動執行。

- **行動順序：** 依位階從高到低行動（巨頭 → 頭目 → 精英 → 威脅 → 雜兵），同位階依生成順序。
- **基礎行為：** 已與調查員交戰 → 攻擊交戰對象。未交戰 → 朝目標移動一格，進入有調查員的板塊則交戰。
- **偏好系統：** 每隻怪物可配置追蹤偏好（最近、血量最低、理智最低、線索最多、仇恨、屬性最低、隨機），由關卡設計師在怪物定義中配置。
- **尋路：** 最短路徑前進，多條等距路徑隨機選一條。
- **攻擊次數：** 雜兵和威脅 1 次，精英以上可透過詞綴或設計增加。
- **怪物攻擊解析：** 從招式池抽戰鬥風格卡，調查員做防禦檢定。

> **完整規則見《補充 03》第二章。**

## 三、設計紅線｜Design Red Lines

以下是絕對不可違反的設計禁區。若任何設計方案觸及紅線，必須立即停止並重新思考。

1. **不得出現無限循環構築。** 這是從 Slay the Spire 學到的教訓。行動經濟有上限、重複效果有遞減、資源費用防止免費循環。
2. **不得讓成長消滅挑戰 — 但威脅有層級。** 克蘇魯神話的恐怖來自於層級分明的存在階序：
   - **不可戰勝的存在：** 外神與大部分舊日支配者（如猶格‧索托斯、阿撒托斯、奈亞拉托提普）不是用來被「打敗」的。規則應明確寫出「你無法對祂造成傷害」。玩家能做的是**絆倒祂、拖慢祂、把祂引去別的地方、封印裂隙** — 而這通常需要有人犧牲。這是克蘇魯的核心敘事：人類的勝利不是殺死神，而是「活著逃出來」或「阻止祂完全降臨」。
   - **可被武力擊敗的存在：** 較低階的神話生物（星之精、深潛者、修格斯、暗黑幼崽等）、邪教徒、人類敵人是可以被調查員用武力正面擊敗的。隨著調查員成長，面對這些威脅會變得更有信心 — 這是成長感的合理回報。
   - **設計原則：** 力量曲線的天花板由敵人的存在層級決定，而非由數值無限攀升決定。再強的調查員在面對混沌袋與高階存在時仍然脆弱，但他們對低階威脅的從容應對正是成長的證明。
3. **不得讓卡片獲取脫離敘事。** 沒有抽象的「獎勵畫面」。每張卡的獲得都有故事脈絡。
4. **不得弱化死亡的重量。** 永久死亡是真實的。不設「付費復活」「花金幣取消死亡」等機制。
5. **不得讓多人模式成為事後補丁。** 1–4 人合作從架構層開始設計。
6. **不得破壞模組獨立性。** 調查員與場景的耦合僅限於明確定義的介面。

## 四、核心情感設計｜Emotional Design Targets

克蘇魯遊戲的靈魂不在機制，在情感。以下是每個系統設計時應追求的情感體驗：

| 情境 | 目標情感 |
|------|----------|
| 探索未知地點 | 好奇與不安並存 — 「我想知道裡面有什麼，但我怕打開之後無法承受」 |
| 獲得新卡片 | 如獲至寶的珍惜 — 每一把新武器、每一瓶新藥都意味著多一次活命的機會 |
| 消耗品用盡 | 真實的焦慮 — 「我只剩兩發子彈，但還有三個房間沒搜」 |
| 面對混沌袋 | 無助與臣服 — 「無論我準備得多好，命運不由我」 |
| 隊友倒地 | 急迫的抉擇 — 「我救他就會暴露自己，不救他他就死了」 |
| 累積創傷 | 溫水煮青蛙的恐懼 — 「我還活著，但我的上限越來越低」 |
| 角色永久死亡 | 真實的失落與敬意 — 這個角色的故事結束了，但他的犧牲有意義 |
| 副陣營解鎖 | 禁忌的誘惑 — 「我不應該碰這個力量，但我需要它」 |
| 團隊精神解鎖里程碑 | 戰友的羈絆 — 「我們一起走過地獄，這份默契是真實的」 |
| 重返已知關卡 | 熟悉中的陌生 — 「我記得這個地方，但有些東西變了……而且更危險了」 |
| 休息時刻 | 暴風雨前的寧靜 — 「趁現在還能喘口氣，整理好自己」 |
| 分支路線抉擇 | 不可逆的重量 — 「走這條路就回不了頭了，但另一條路的代價更大」 |
| 用熟練武器戰鬥 | 職人的自信 — 「這把獵槍在我手上，每一發都有意義」 |
| 用不熟練的方式掙扎 | 絕望的即興 — 「我不會用劍，但我沒有別的選擇了」 |
| 施放神秘法術 | 玩火的代價 — 「力量從我體內湧出，但我不確定我還能控制多久」 |
| 閱讀戰鬥敘事描述 | 身歷其境 — 不是「你造成 3 點傷害」，而是「你的銀彈穿透了深潛者的鱗甲，牠發出尖銳的嘶鳴」 |
| 觀察敵人狀態描述 | 獵人的直覺 — 「牠的動作變慢了……再一擊應該就夠了，但我不確定」 |
| 發現敵人弱點 | 破局的興奮 — 「火焰有用！快，把火油瓶傳給我！」 |
| 戰鬥風格卡抽到不擅長的屬性 | 命運的玩笑 — 「該死，偏偏這時候要靠意志力……」 |
| 瀕死檢定 | 命懸一線 — 「求你了，10 以上就好……」 |

## 五、技術設計原則｜Technical Design Principles

1. **卡片是通用單位。** 武器是卡片、道具是卡片、技能是卡片、詛咒是卡片。所有系統都透過卡片介面互動。
2. **狀態持續是核心。** 章節間、戰役間的狀態保持必須可靠且完整。這不是 roguelike。
3. **模組邊界清晰。** 調查員模組與場景模組透過定義明確的 API 介面溝通，不得直接耦合內部狀態。
4. **人數縮放內建。** 敵人數值、線索需求、遭遇效果都必須包含人數縮放參數。
5. **反作弊考量。** 永久死亡機制意味著資料完整性至關重要。關鍵狀態必須在伺服器端驗證。
6. **編輯器優先設計。** 創角器和關卡編輯器不是事後功能。內部開發工具應以最終成為玩家工具為目標設計。
7. **系統管理員模組。** 除了面向玩家的創角器與關卡編輯器之外，需建置**系統管理員後台（Admin Module）**，內含數值模擬器群組：骰子機率分析器、Boss 戰鬥模擬器、創角點數分配模擬器、敵人階層設計器、傷害曲線與 HP/SAN 推算器等。這些工具是數值設計的永久基礎設施，用於設計新調查員、新敵人、新武器、新關卡時即時驗證平衡性。
8. **圖示優先。** 遊戲中的屬性、戰鬥風格、元素、狀態效果、動作類型等資訊使用圖案表示（非文字）。開發測試階段以有顏色的文字代替，正式圖案設計為後續美術任務。

## 六、設計差異化定位｜Competitive Differentiation

本遊戲不是任何現有遊戲的複製品。以下是與主要參照對象的關鍵差異：

| 面向 | Arkham Horror LCG | Slay the Spire | 本專案 |
|------|-------------------|-----------------|--------|
| 起始牌組 | 厚（30+ 張預構築） | 無（從零開始） | 精簡（15–20 張，帶著最低限度出門） |
| 卡片獲取 | 花經驗值購買 | 戰鬥後隨機獎勵 | 探索場景中有機發現 |
| 消耗品 | 部分消耗 | 無真實消耗 | 完全真實消耗（子彈打完就沒了） |
| 構築時機 | 關卡之間 | 戰鬥之間 | 冒險進行中即時 |
| 死亡後果 | 獲得創傷但可繼續 | 重新開始一局 | 角色資料永久刪除 |
| 檢定機制 | 混沌袋 | 無（純卡牌） | 雙軌（擲骰 + 混沌袋） |
| 戰鬥解析 | 固定屬性檢定 | 無（卡牌效果） | 戰鬥風格卡動態決定檢定屬性 |
| 平台 | 實體卡牌 | PC / 主機 | 網頁瀏覽器 |
