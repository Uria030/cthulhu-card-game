## 一、設計哲學｜Design Philosophy

### 1.1 核心定位

支線是**加速成長管道** — 提供主線之外的彈性遊玩空間，讓玩家自主選擇練等、賺取資源或體驗更多內容。支線不影響主線劇情走向。

### 1.2 與主線的關係

| 面向 | 主線 | 支線 |
|------|------|------|
| 劇情影響 | 產生旗標，影響後續章節 | 不產生任何旗標 |
| 結果判定 | 多分支（A/B/C） | 二元（通過/失敗） |
| 場景結構 | ACT + Agenda + 旗標 | ACT + Agenda，無旗標 |
| 角色風險 | 瀕死/死亡照常 | 瀕死/死亡照常 |
| 關卡失敗懲罰 | 影響旗標與後續劇情 | 無懲罰，白跑一趟 |

### 1.3 設計意圖

- **練等管道：** 玩家覺得主線太難時，可以先去支線強化自己。
- **防止無風險農場：** 角色瀕死與死亡照常運作，即使是支線也有真實風險。
- **填充遊玩空間：** 讓玩家在主線章節之間有事做，單人上線也有內容可玩。

## 二、支線類型｜Side Scenario Types

### 2.1 兩種支線

| 類型 | 代碼 | 說明 |
|------|------|------|
| **預設小關卡（A）** | `side` | 預先設計好的場景，有劇情、有敘事 |
| **隨機地城（B）** | `side_random` | 每次進入時重新生成的場景 |

### 2.2 預設小關卡（A）

| 項目 | 規格 |
|------|------|
| 場景結構 | 使用 ACT + Agenda 結構 |
| 結果判定 | ACT 先完成 = 通過，Agenda 先跑完 = 失敗 |
| 旗標系統 | **不使用**，不產生任何主線旗標 |
| 關卡規模 | 可以是單關，也可以多關成組（3 關以上皆有可能） |
| 多關成組 | 多關成組時，各關之間僅以「通過/失敗」串接，不使用旗標流轉 |
| 解鎖條件 | 彈性設計：無條件開放 / 特定條件門檻 |
| 重返機制 | 可重返，每次難度遞增 |

### 2.3 隨機地城（B）

| 項目 | 規格 |
|------|------|
| 地圖結構 | 待定，場景編輯器階段設計 |
| 地點數量 | 待定，場景編輯器階段設計 |
| 勝利條件 | 待定，場景編輯器階段設計（不限定單一種） |
| 重返機制 | 不存在重返概念，每次都是重新生成的新局 |
| 創建規則 | 待定，場景編輯器階段設計 |

## 三、進入條件｜Access Conditions

### 3.1 進入時機

玩家在**遊戲大廳**即可選擇進入支線，包含：

- **首次進入遊戲**，進入遊戲大廳後
- **每個章節結束**，回到遊戲大廳後

### 3.2 大廳選擇

玩家從遊戲大廳出發時，可自由選擇：

1. 進行某一個主線章節
2. 進入預設小關卡（A）
3. 進入隨機地城（B）

### 3.3 解鎖條件

預設小關卡（A）的解鎖條件由關卡設計師定義，可為以下任一種：

| 解鎖類型 | 說明 |
|---------|------|
| 無條件 | 遊戲開始即可進入 |
| 主線進度 | 完成指定主線章節後解鎖 |
| 特定條件 | 滿足特定條件（擁有特定物品、調查員等級達標等） |

隨機地城（B）的解鎖條件待定。

## 四、獎勵結構｜Reward Structure

### 4.1 通關獎勵

支線通關獎勵與主線相同，是主線獎勵的**加速版本**：

| 獎勵類型 | 說明 |
|---------|------|
| 經驗值 | 調查員成長資源 |
| 天賦點 | 天賦樹投資資源 |
| 素材 | 鍛造與製作用素材 |
| 凝聚力 | 團隊精神投資資源 |

### 4.2 支線專屬簽名卡

每個預設小關卡（A）可設計**支線專屬簽名卡** — 僅首次通關時獲得的獨特卡片。

| 項目 | 規格 |
|------|------|
| 獲得條件 | 首次通關限定 |
| 重返時 | 不再給予 |
| 設計定位 | 收藏誘因 + 構築選項 |

### 4.3 隨機地城（B）的獎勵

隨機地城每次都是新局，獎勵**不遞減**。每次通關都給予標準獎勵（經驗值、天賦點、素材、凝聚力），無專屬簽名卡。

## 五、重返機制｜Return Mechanic

### 5.1 適用範圍

| 類型 | 重返 |
|------|------|
| 預設小關卡（A） | ✅ 可重返 |
| 隨機地城（B） | ❌ 不適用（每次重新生成） |

### 5.2 重返規則

| 項目 | 規格 |
|------|------|
| 難度 | 每次重返難度遞增 |
| 內容 | 重返時內容可能有所不同（不同的遭遇、敘事細節、地圖配置） |
| 獎勵 | 遞減 — 簽名卡僅首次給予，後續重返只給經驗值、天賦點、素材、凝聚力 |
| 難度遞增機制 | 具體機制待數值規格定義 |

### 5.3 重返次數

重返次數無上限。難度持續遞增，自然形成玩家自行判斷的停止點。

## 六、失敗規則｜Failure Rules

### 6.1 關卡層級

| 情境 | 後果 |
|------|------|
| ACT 未完成，Agenda 先跑完 | **無懲罰**，不獲得通關獎勵 |
| 可重新挑戰 | 是，無限制 |

### 6.2 角色層級

| 情境 | 後果 |
|------|------|
| 調查員 HP 或 SAN 歸零 | 進入瀕死狀態，照常執行瀕死檢定 |
| 瀕死檢定三次失敗 | 死亡，歸零項上限 -1 |
| 上限歸零 | **永久死亡**，角色資料刪除 |

> **設計意圖：** 支線「輕鬆」是指獎勵好拿、不影響主線。但角色的命是真的 — 永久死亡不因支線而豁免。這確保玩家不會拿高等角色去支線無腦刷，因為意外永遠存在。非常克蘇魯。

## 七、對資料庫結構的影響｜Database Schema Impact

### 7.1 scenarios 表補充

`scenario_type` ENUM 需新增值：

```sql
CREATE TYPE scenario_type AS ENUM (
  'main',           -- 主線章節
  'side',           -- 預設小關卡
  'side_return',    -- 預設小關卡重返版本
  'side_random'     -- 隨機地城（新增）
);
```

### 7.2 支線專屬簽名卡欄位

支線場景定義需新增：

| 欄位 | 類型 | 說明 |
|------|------|------|
| `reward_signature_cards` | JSONB | 首次通關時獲得的專屬簽名卡列表 |

### 7.3 既有欄位確認

以下欄位在資料庫結構設計中已預留，確認用途：

| 欄位 | 用途 |
|------|------|
| `return_count` | 追蹤已重返次數（運行時遞增） |
| `difficulty_scaling` | 重返時的難度縮放參數 |

## 八、對場景編輯器的影響｜Scenario Editor Impact

場景編輯器需支援以下功能：

| 功能 | 說明 |
|------|------|
| 支線類型選擇 | 建立場景時選擇 `side` / `side_random` |
| 解鎖條件設定 | 設定支線的解鎖條件（無條件 / 主線進度 / 特定條件） |
| 獎勵設定 | 設定通關獎勵（經驗值、天賦點、素材、凝聚力數量） |
| 簽名卡設定 | 指定首次通關獎勵的專屬簽名卡 |
| 重返版本設計 | 為同一小關卡設計不同的重返版本 |
| 多關成組 | 將多個小關卡串接為一條支線劇情線 |
| 隨機地城生成規則 | 待定 |

> **給未來 Claude 實例的備註｜Note to Future Claude Instances**
>
> 本文件的所有設計均經過與專案原創者 Uria 的逐項討論確認。
> 特別注意以下設計精神：
>
> 1. **支線不使用旗標系統** — 結果只分通過/失敗，不影響主線。
> 2. **支線使用 ACT + Agenda 結構** — 同一套場景引擎，只是結果判定簡化。
> 3. **角色瀕死/死亡照常** — 永久死亡不因支線而豁免，這是紅線。
> 4. **支線專屬簽名卡僅首次通關給予** — 重返時獎勵遞減。
> 5. **隨機地城不存在重返概念** — 每次重新生成，獎勵不遞減。
> 6. **遊戲大廳是出發點** — 包含首次進入遊戲和每個章節結束後。
> 7. **隨機地城的詳細設計留待場景編輯器階段** — 地圖結構、地點數量、勝利條件、創建規則均為待定。
