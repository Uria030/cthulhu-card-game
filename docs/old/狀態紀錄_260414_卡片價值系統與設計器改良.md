## 三、卡片價值計算系統（核心成果）

### 3.1 基礎單位

> **1V = 1 行動點 = 1 資源 = 抽 1 張牌 = 造成 1 點傷害**

### 3.2 效果價值表（9 大類）

完整建立了以下類別的價值定義：

1. 傷害與恢復類（直接傷害 1V、恐懼傷害 3V、恢復 HP/SAN 1.5V 等）
2. 卡牌操作類（抽牌 1V、搜尋牌庫 6V、回收 1.5V 等）
3. 資源類（資源 1V、彈藥/充能 0.5V）
4. 移動類（移動 1V、跳躍 2V、傳送 3V）
5. 檢定修正類（曲線成長：+1=0.5V、+2=1.5V、+3=3V、+4=5V、+5=7.5V）
6. 負面狀態（中毒 3V、流血 2V、燃燒 3V、標記 6V、發瘋 6V 等）
7. 正面狀態（強化 3V、護甲 3V、護盾 6V、加速 4V、再生 6V、隱蔽 6V）
8. 戰鬥控制類（絆倒 2V、脫離 1V、斬殺 3V+）
9. 環境互動類（發現線索 2V、移除毀滅 4V、封印次元門 4V+）

### 3.3 費用公式（依卡片類型）

| 卡片類型 | 費用公式 | 換算比例 |
|----------|----------|----------|
| 事件卡 | 效果價值 - 等級抵扣 - 稀有度抵扣 | 1:1 |
| 資產卡 | (總輸出價值 - 留場修正 - 消耗修正 - 等級抵扣 - 稀有度抵扣) ÷ 2 | 2:1 |
| 盟友卡 | HP×0.5 + SAN×0.5 + 留場2V + 能力價值 - 等級抵扣 - 稀有度抵扣 | 1:1 |
| 技能卡 | 固定 0，改用價值區間檢驗 | — |

### 3.4 抵扣機制

**等級抵扣：**
- 0 級 = 0、1 級 = -1V、2 級 = -2V ... 5 級 = -5V

**稀有度抵扣（系統反推）：**
- 隨身 = 0、基礎 = -1V、標準 = -2V、進階 = -3V、稀有 = -4V、傳奇 = -5V

**消耗類型修正：**
- 留場 = -1V、棄牌 = 0、短休息 = -1V、長休息 = -2V、移除 = -3V

### 3.5 重要設計決策

1. **稀有度是反推的，不是選擇的**
   - 設計者決定：效果、等級、費用
   - 系統計算：需要的稀有度抵扣 → 向上進位得出稀有度

2. **向上進位規則**
   - 1.1 ~ 2 算標準（-2V）
   - 2.1 ~ 3 算進階（-3V）
   - 以此類推

3. **有代價的能力 = 0V**
   - 代價與效果互相抵消

4. **武器彈藥打完進棄牌堆**
   - 不再使用休息回復機制

## 五、卡片設計器改良（v3 指令）

### 核心改進

1. **稀有度反推流程**
   ```
   設計效果 → 選等級 → 設費用 → 系統反推稀有度
   ```

2. **三合一區塊介面**
   - ▸ 打出效果（費用、消耗類型、裝備欄位）
   - ▸ 加值數據（加值屬性、數值、成功/失敗時附加效果）
   - ▸ 消費功能（彈藥、傷害、戰鬥風格、檢定）

3. **效果選擇器（視覺化）**
   - 下拉選單選擇效果類別 → 效果類型 → 數值
   - 即時顯示每個效果的價值

4. **平衡計算面板**
   - 即時計算效果總價值、等級抵扣、需要的稀有度抵扣
   - 顯示反推出的稀有度與平衡狀態

5. **AI 生成強化**
   - Prompt 寫入價值計算系統
   - 回傳包含 `value_calculation` 欄位

## 九、給下一個 Claude 實例的備註

1. **價值計算系統是核心**
   - 所有卡片設計都應該用《卡片價值計算規範 v0.1》的公式驗算
   - 1V = 1 行動點 = 1 資源 = 抽 1 張牌 = 造成 1 點傷害

2. **稀有度是反推的**
   - 設計者決定效果、等級、費用
   - 系統計算稀有度抵扣 → 向上進位 → 得出稀有度
   - 這與一般卡牌遊戲「先選稀有度再設計」的思路相反

3. **三合一區塊**
   - 每張卡片有三種用途：打出、加值、消費
   - 介面依此分為三個可收合區塊

4. **技能卡特殊處理**
   - 費用固定為 0
   - 改用價值區間檢驗（LV0=2.5-3V、LV2=5-6V、LV3=7-8V、LV5=9-10V）

5. **武器彈藥打完進棄牌堆**
   - 這是最新的設計決定
   - 不再使用休息回復機制
