## §4 覆蓋規則（V 值公式更新）

**適用檔案**：`03_rulebook_ch3.md`（第三章 · 卡片系統總綱）

規則書 v1.0 第三章本文中的 V 值計算公式為**待更新文字**，**以下列新版為準**：

- **所有卡片類型統一為 1:1**（含資產卡；原 2:1 作廢）
- **事件卡、盟友卡、技能卡**亦為 1:1；技能卡費用固定為 0
- **等級抵扣新階梯**：LV1 -0.5V / LV2 -1V / LV3 -2V / LV4 -3V / LV5 -4V（**非**舊版 -1V/-2V/-3V/-4V/-5V）
- 驗證器、Prompt、程式碼以 1:1 為準
- 若看到第三章文字中的 2:1 公式，**視為待更新文字，不要採用**

**權威依據**：
- `s04_card_value_spec.md` 卡片價值計算規範 v1.1 修正案（§2 專項）
- `docs/04_SystemPrompt模板庫_卡片設計篇.md`（bridge 用的 Prompt，已用 1:1）
- 記憶 `project_v_value_formula.md`（Claude Code 本機記憶，跨 session 持續有效）

## §5 給 claude.ai 使用者的指示

這份索引只是地圖。**實際規則文字在各 `.md` 檔案裡**。使用流程：

1. **上傳這份索引** → 讓 claude.ai 理解規則書結構
2. **依需求額外上傳 `packages/client/public/rulebook/` 下對應的檔案**：
   - 要討論**卡片數值/機制** → 上傳 `03_rulebook_ch3.md` + `s04_card_value_spec.md`
   - 要討論**怪物/敵人** → 上傳 `s01_monster_family.md` + `06_rulebook_ch6.md`
   - 要討論**場景/關卡** → 上傳 `05_rulebook_ch5.md`
   - 要討論**譯名/命名** → 上傳 `s02_mythos_terms.md`
   - 全規則討論 → 上傳六章全部（3,320 行，約 50k tokens，Claude 3.7/Opus 4 輕鬆）
3. **避免上傳 `h*` 歷史版本**——除非你要溯源「為什麼從 v0.3 改成 v1.0」
4. **記得告知 claude.ai §4 覆蓋規則**——否則它會讀到第三章的舊 2:1 公式
