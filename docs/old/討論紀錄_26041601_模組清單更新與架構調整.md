## 一、討論背景

在完成 MOD-03 敵人設計器指令文件（三份）後，Uria 提出現有的模組清單不足以涵蓋所有設計需求，需要進行以下調整：

1. 新增城主設計器
2. 新增調查員設計器
3. 場景編輯器（原 MOD-07）內容過大，需拆分
4. 混沌袋設計器（原 MOD-08）應整合進關卡編輯器

---

## 二、場景系統拆分

### 問題

原架構中 MOD-06 戰役管理 + MOD-07 場景編輯器只有兩個模組，但遊戲的場景結構有四個層級（戰役 → 章節 → 關卡 → 場景），內容過於龐大。

### 決議：拆分為三個獨立設計器

#### 戰役敘事設計器（MOD-06）

管「劇本結構」— 最上層。

- 戰役有幾個章節、章節之間的分支邏輯
- 旗標系統、劇情選項、結果如何影響後續路線
- 點進某一關 → 跳轉到關卡編輯器

#### 關卡編輯器（MOD-07）

管「單關的完整配置」— 中間層。

- 從地點庫挑選地點、排列連通關係
- 配置混沌袋（原 MOD-08 混沌袋設計器整合至此）
- 開放哪些怪物家族池（呼叫 MOD-03 的家族池，不指定具體怪物）
- ACT/Agenda 牌堆設計
- 勝利/失敗條件與結果判定
- 遭遇牌堆配置

#### 地點設計器（MOD-08）

管「可重複使用的空間積木」— 底層。

- 每個地點獨立設計：調查難度、線索數、隱藏調查點、環境狀態
- 地點本身不知道自己會被哪些關卡使用
- 關卡編輯器從地點庫挑選地點進行排列組合

### 三者的呼叫關係

```
戰役敘事設計器（MOD-06）
  │ 定義章節結構、分支邏輯
  │ 點進某一關 →
  ▼
關卡編輯器（MOD-07）
  │ 從地點庫挑選地點 →
  ▼
地點設計器（MOD-08）
  │ 提供可重複使用的地點積木
```

## 三、混沌袋設計器整合

### 決議

原 MOD-08 混沌袋設計器**取消獨立模組**，整合進關卡編輯器（MOD-07）作為其中一個編輯區塊。

### 原因

混沌袋的配置跟著關卡走 — 每一關有自己的混沌袋內容。混沌袋沒有脫離關卡獨立存在的意義，不需要獨立的「模板庫」。

---

## 四、新增城主設計器（MOD-10）

### 定位

設計城主 AI 的「牌庫」。城主不是自由發揮，而是有一副手牌、有行動點預算的對手。本質上是「城主版的卡片設計器」。

### 管理兩種卡片

#### 神話卡（Mythos Cards）

- 城主的「手牌」，類似調查員打出卡片
- 有行動點花費（越強大越貴）
- 有發動時機限制：
  - 調查員階段響應（反應型）
  - 敵人階段使用
  - 皆可
- 效果範例：召喚怪物、改變環境、施放全場效果、推進議程等

#### 遭遇卡（Encounter Cards）

- 不同地點風格的神秘事件
- 結構：情境描述 → 選項 A/B → 檢定屬性 + DC → 成功/失敗效果
- 調查員做出選擇後進行檢定
- 相對簡單

## 五、新增調查員設計器（MOD-11）

### 定位

設計玩家可選的調查員角色。

### 管理內容

- 四極組合歸屬（MBTI 四維度選擇 → 決定哪些玩家可選此調查員）
- 基礎屬性配點（七大屬性，總點數 18）
- 專屬簽名卡設計（2–3 張，調查員獨有）
- 專屬弱點卡設計（1 張，強制納入牌組）
- 預組起始牌組（從歸屬的四個極卡池 + 中立卡池中挑選）
- 角色敘事（稱號、背景故事、能力描述、立繪）
- 初始戰鬥熟練配置

## 八、追加討論：四項待決定事項的決議

### 8.1 城主行動點系統

**決議：**

- 城主每回合獲得行動點，公式為：**基礎點數（由關卡難度 1–5 決定）+ 人數加成**
- 關卡難度在關卡編輯器（MOD-07）中設定
- **可跨回合累積** — 城主可以選擇前幾回合按兵不動，蓄力後某回合集中爆發
- 神話卡有行動點花費，越強大的神話卡花費越高

**設計意圖：** 城主不是自由發揮的 AI，而是有資源預算限制的對手。蓄力機制符合克蘇魯的壓迫感設計 — 玩家會感受到「暴風雨前的寧靜」。

> **注意：** 此為本次討論中新產生的設計規則，目前不存在於規則書任何章節。需要在後續回寫進規則書第二章（敵人階段）或獨立成為補充文件。具體的基礎點數對照表和人數加成公式待數值設計時確定。

### 8.2 遭遇卡歸屬

**決議：**

- 遭遇卡在**城主設計器（MOD-10）**中設計
- 遭遇卡帶有「地點風格標籤」，用於分類
- 關卡編輯器（MOD-07）配置遭遇牌堆時，根據該關使用的地點風格自動篩選可用的遭遇卡
- 城主設計器本質上就是「城主版的卡片設計器」— 管理神話卡和遭遇卡兩種卡片

### 8.3 六個待寫模組的撰寫順序

**決議：** 依賴關係排序如下：

| 順序 | 模組 | 理由 |
|------|------|------|
| 1 | MOD-08 地點設計器 | 最底層積木，無依賴 |
| 2 | MOD-10 城主設計器 | 神話卡/遭遇卡無依賴，獨立設計 |
| 3 | MOD-11 調查員設計器 | 依賴 MOD-01 卡片庫（簽名卡/弱點）和 MOD-05 戰鬥熟練 |
| 4 | MOD-09 鍛造與製作管理 | 依賴卡片系統但相對獨立 |
| 5 | MOD-06 戰役敘事設計器 | 定義章節結構，下轄關卡 |
| 6 | MOD-07 關卡編輯器 | 依賴最多：地點庫、怪物家族池、混沌袋、城主牌庫、ACT/Agenda |

### 8.4 SIM 模組調整

- **SIM-04 敵人階層設計器** → 移除（功能已被 MOD-03 敵人設計器的家族總覽面板覆蓋）
- **SIM-04 位置替換為「規則書閱覽器」** — 讓管理員在設計過程中隨時查閱規則書作為參考
