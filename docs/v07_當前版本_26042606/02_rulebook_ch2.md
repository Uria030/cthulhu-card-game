## Chapter 2: Game Rules

> **文件用途｜Purpose**
> 本章定義遊戲的完整規則系統，包含遊戲大循環、回合結構、檢定系統、行動類型、
> 戰鬥規則、瀕死與救援、混沌袋與法術、戰鬥風格卡、敵人 AI、環境系統、調查系統等。
>
> 本章是第一章（核心設計原則）的規則實作層。所有卡片設計、場景設計、程式實作皆以本章為規則權威。

## 一、遊戲大循環｜Game Loop

### 1.1 主線流程

```
遊戲大廳
  ↓
選擇主線章節 / 支線關卡 / 隨機地城
  ↓
劇情演示 + 劇情選項（主線限定）
  ↓
個人秘密任務選擇
  ↓
場景遊玩模式（回合制）
  ↓
遊玩結束 → 結果判定
  ↓
長休息 + 整備模式
  ↓
回到遊戲大廳 → 選擇下一步
  ↓
（循環直到戰役結束）
```

### 1.2 遊戲大廳

- 戰役的起點與終點。
- 玩家在此自由選擇：進行主線章節、進入預設小關卡、進入隨機地城。
- 其他玩家可在大廳中加入隊伍、觀看、參與聊天。
- 社交功能為後期版本功能，核心功能（選關、整備）優先。

### 1.3 劇情演示

- 進入主線章節後，首先播放劇情演示。
- 劇情中會出現**劇情選項**供玩家選擇，影響後續劇情走向與關卡內容。

### 1.4 個人秘密任務

- 每位玩家在進入場景前獲得一個**個人秘密小任務**。
- **其他隊友看不到**此任務的內容。
- 過關本身給予天賦點，**達成個人秘密任務額外獎勵天賦點**，依任務難度分級。
- **設計意圖：** 創造微妙的團隊張力 — 你可能為了完成秘密任務而做出隊友看起來「奇怪」的行動。完美呼應克蘇魯世界觀中「每個人都有自己的秘密與動機」。

### 1.5 結果判定

- 主線場景結束後判定結果，可能有 **A / B / C 等多種結果**（最少 2 種，最多 5 種分歧）。
- **設計傾向讓玩家能往下推進** — 不同結果影響後續劇情路線和獎勵，而非是否能繼續遊戲。
- 支線場景只分**通過/失敗**，不產生旗標。

### 1.6 長休息與整備模式

- 每個章節結束後進入**長休息**，獲得 **1 點凝聚力**。
- 整備模式中玩家可進行以下操作：
  - 花費**天賦點** → 天賦樹
  - 花費**經驗值** → 從卡片庫購買卡片副本進牌組,或對牌組內卡片購買強化(書籍/遺跡升級版走 transform 路徑,不在此項)
  - 花費**凝聚力** → 團隊精神
  - **鍛造**裝備（需已解鎖鍛造團隊精神）
  - **製作**道具（需已解鎖製作團隊精神）
  - **購買已完成閱讀/研究的書籍/遺跡升級版**
  - **調整下一關要帶的牌組**
  - **換別的調查員**加入團隊

## 二、回合結構｜Turn Structure

### 2.1 回合流程

```
回合開始
  ↓
短休息決定（每位調查員各自決定）
  ├ 選擇短休息 → 執行短休息效果 → 本回合結束（不進入調查員階段）
  └ 不休息 → 進入調查員階段
  ↓
調查員階段（3 行動點，自由順序）
  └ 城主可能在此階段中插入傳奇行動
  ↓
敵人階段（城主行動）
  ├ 控制怪物移動與攻擊
  ├ 翻開一張神話卡（立即效果 + 持續效果）
  └ 若有議程牌堆 → 放置毀滅標記 → 到達門檻時翻面結算
  ↓
回合結束階段
  ├ 每位調查員抽 1 張卡
  ├ 每位調查員獲得 1 資源
  ├ 所有橫置卡片轉正
  └ 手牌超過上限（8 張）須棄牌至上限
  ↓
下一回合
```

### 2.2 調查員階段細則

- 每位調查員擁有 **3 行動點**。
- 調查員之間**不需要固定輪流** — 可自由溝通協調行動順序。
- 每位調查員可一口氣用完自己的 3 行動點。
- **城主傳奇行動**可在調查員行動期間穿插觸發，觸發條件在場景中預先設計。

### 2.3 敵人階段細則

- 城主（系統 AI）控制所有怪物的移動與攻擊。
- 怪物行為依據其詞綴與位階決定（見第八節）。
- 翻開一張**神話卡**，有立即效果與持續效果。
- 若場上有議程牌堆，放置一個毀滅標記。當毀滅標記達到卡面門檻時，翻面結算背面效果。

### 2.4 回合結束階段

- 每位調查員從自己的牌庫**抽 1 張卡**。
- 每位調查員**獲得 1 資源**。
- 所有**橫置卡片轉正**。
- 若手牌超過上限（8 張），須棄牌至上限。

## 三、短休息｜Short Rest

### 3.1 核心規則

- 在回合開始時，每位調查員各自決定是否進行短休息。
- 選擇短休息的調查員：
  - **重洗牌庫**（將棄牌堆洗回牌庫）。
  - **短休息消耗品回歸牌庫**（標記為「短休息可回收」的消耗品卡片回歸牌庫）。
  - 觸發**陣營專屬短休息效果**（見 3.2）。
  - **犧牲整回合行動** — 不進入調查員階段。
- 可透過團隊精神「短休息強化」解鎖短休息中的額外行動。

### 3.2 陣營短休息效果（方向草案，待正式設計）

| 陣營極 | 中文 | 短休息效果方向 |
|--------|------|---------------|
| E | 號令 | 隊友也獲得某種小好處 |
| I | 深淵 | 看牌庫頂數張重新排列 |
| S | 鐵證 | 回收一件已消耗的裝備 |
| N | 天啟 | 窺探混沌袋 |
| T | 解析 | 額外獲得資源 |
| F | 聖燼 | 回復少量 SAN |
| J | 鐵壁 | 獲得臨時護甲 |
| P | 流影 | 從棄牌堆撈一張卡回手牌 |

### 3.3 牌庫耗盡規則

- 當調查員的牌庫為空且需要抽牌時：**不自動重洗牌庫。**
- 每抽一張卡（從空牌庫），調查員受到 **1 點恐懼傷害**。
- **設計意圖：** 迫使玩家在「繼續硬撐」與「花一整回合短休息重洗牌庫」之間做出有重量的抉擇。議程牌堆上的毀滅標記每回合都在增加。

## 四、檢定系統｜Skill Check System

### 4.1 主要檢定（擲骰）

| 項目 | 規格 |
|------|------|
| 骰子 | d20，均勻分佈，每面 5% 機率 |
| 公式 | d20 + 屬性修正 + 熟練/專精修正 + 裝備/卡牌加成 |
| 判定 | 結果 ≥ DC = 成功 |
| 屬性修正 | **屬性值 = 修正值（1:1）** |

### 4.2 修正值來源

| 層級 | 來源 | 說明 |
|------|------|------|
| 屬性修正 | 屬性值（1:1） | 屬性 3 = +3 |
| 熟練/專精 | 角色成長 | 戰鬥熟練/專精提供的修正 |
| 裝備/卡牌 | 牌組構築 | 武器的 `attribute_modifiers`、場上卡片效果 |
| 加值（Commit） | 手牌投入 | 檢定前從手牌投入的屬性圖示 |

### 4.3 三層修正值疊加（參考曲線）

| 層級 | 初期（Ch.1） | 中期（Ch.5） | 後期（Ch.8） | 巔峰（Ch.10） |
|------|-------------|-------------|-------------|--------------|
| 屬性修正 | +3 | +5 | +7 | +10 |
| 熟練/專精 | +0 | +1 | +2 | +3 |
| 裝備/卡牌 | +1 | +2 | +3 | +4 |
| **合計** | **+4** | **+8** | **+12** | **+17** |

> **注意：** 此為屬性修正 1:1 後的更新數值。敵人 DC 需配合調整以維持設計意圖的命中率曲線，具體調整見《數值規格文件》。

### 4.4 特殊檢定結果

| 擲骰結果 | 效果 | 適用範圍 |
|----------|------|---------|
| 自然 20 | 爆擊 — 2 倍傷害 | 戰鬥攻擊 |
| 自然 20 | 直接站起來 | 瀕死檢定 |
| 自然 1 | 有交戰隊友 → 打到隊友（造成武器傷害） | 戰鬥攻擊 |
| 自然 1 | 無交戰隊友 → 單純未命中 | 戰鬥攻擊 |
| 自然 1 | 算 2 次失敗 | 瀕死檢定 |

> 自然 1 造成的額外懲罰（如武器掉落）不寫在基礎規則中，由卡片效果引入。

### 4.5 特殊檢定（混沌袋）

施法類攻擊不擲骰，改為抽混沌袋決定副作用。完整規則見第五節。

## 五、混沌袋與法術系統｜Chaos Bag & Spell System

### 5.1 設計哲學

| | 物理攻擊（d20 軌） | 法術攻擊（混沌袋軌） |
|--|-------------------|-------------------|
| 不確定性 | 「能不能打到？」 | 「代價多大？」 |
| 命中 | 需要擲骰達到 DC | **一定命中，傷害一定造成** |
| 代價 | 消耗使用次數 | 抽混沌袋承受副作用 |
| 成長感 | 修正值提升，命中率上升 | 副作用管理更熟練，但命運不可控 |
| 情感 | 「我在進步」的可靠感 | 「我在玩火」的不安感 |

### 5.2 混沌袋標記分類

| 分類 | 標記 | 說明 |
|------|------|------|
| 數字標記 | +1、0、-1、-2、-3、-4 | 純數值修正 |
| 情境標記 | 骷髏、邪教徒、石版、遠古邪物 | 由場景定義數值與場景效果（負面為主） |
| 神話標記 | 怪物、毀滅、次元門、線索、頭條 | 由場景定義數值，場景效果固定（正面到負面光譜） |
| 極端標記 | 觸手（自動最壞）、遠古印記（好運） | 固定規則，各 1 個，不可移除 |
| 動態標記 | 祝福（+1，再抽）、詛咒（-1，再抽） | 由卡片效果加入或移除，抽後移出袋 |

### 5.3 法術防禦值

每隻怪物有一個**法術防禦值（Spell Defense, 0–9）**，決定施法副作用的嚴重程度。法術傷害本身不受法術防禦值影響 — 傷害永遠造成。法術防禦值不公開，玩家需透過嘗試觀察推測。

| 位階 | 建議法術防禦值 |
|------|-------------|
| 雜兵 | 0–1 |
| 威脅 | 1–3 |
| 精英 | 3–5 |
| 頭目 | 5–7 |
| 巨頭 | 7–9 |

### 5.4 情境標記（雙重效果）

每個情境標記同時產生數值效果和場景效果，由場景設計師從預設選項中配置：

**骷髏（Skull）— 死亡、傷害、肉體侵蝕：**
死亡之觸（HP 傷害）、創傷侵蝕（HP 上限 -1）、血祭（流血狀態）、生命流逝（無力狀態）、生命代價（失去盟友或資產）

**邪教徒（Cultist）— 混沌、議程推進、增援：**
末日推進（放置毀滅標記）、信徒回應（生成敵人）、儀式共鳴（怪物回血）、暴露（失去隱蔽）、黑暗儀式（板塊進入黑暗）

**石版（Tablet）— 精神侵蝕、理智崩潰：**
禁忌知識（SAN 傷害）、瘋狂低語（發瘋狀態）、記憶崩解（隨機棄手牌）、精神枯竭（疲勞狀態）、不應知曉之事（神啟卡洗入牌庫）

**遠古邪物（Elder Thing）— 環境異變、時空扭曲：**
裂隙擴張（開啟次元門）、時空扭曲（隨機傳送）、異界滲透（放置鬧鬼）、空間斷裂（斷開地點連接）、異界之火（板塊失火）、虛空寒流（冷凍狀態）

### 5.5 神話標記（固定場景效果）

| 標記 | 性質 | 數值傾向 | 場景效果（固定） |
|------|------|---------|---------------|
| 線索 | 正面 | 0 至 -1 | 施法者所在地點生成 1 個線索 |
| 頭條 | 中性 | -1 至 -2 | 抽取 1 張遭遇卡，立即結算 |
| 怪物 | 負面 | -2 至 -3 | 在施法者所在或相鄰地點生成 1 隻敵人 |
| 毀滅 | 負面 | -2 至 -4 | 議程牌堆放置 1 個毀滅標記 |
| 次元門 | 重度負面 | -3 至 -5 | 在施法者所在地點開啟次元門 |

> 神話標記的場景效果**無視法術防禦值判定**，一律發動。法術防禦值只影響數值部分。

### 5.6 極端標記

**觸手：** 自動視為最低，副作用全額發動，同時觸發所有情境標記中最嚴重的場景效果，施法者受到 1 點恐懼傷害。

**遠古印記：** 自動視為最高，副作用完全取消，觸發調查員卡上的個人效果，施法者恢復 1 點 SAN。

### 5.7 副作用減輕規則

| 比較結果 | 副作用處理 |
|---------|----------|
| 數值 ≥ 法術防禦值 | 場景效果取消，數值效果取消 |
| 數值 < 法術防禦值，差值 1–2 | 場景效果取消，施法者受 1 點 SAN 傷害 |
| 數值 < 法術防禦值，差值 3–4 | 場景效果全額發動 |
| 數值 < 法術防禦值，差值 5+ | 場景效果全額發動 + 施法者額外受 1 點 SAN 傷害 |

觸手無視法術防禦值，遠古印記無視法術防禦值。

### 5.8 法術攻擊完整結算流程

```
施法者宣告施放法術 → 確認費用已支付
  ↓
法術效果一定生效 — 造成卡面定義的傷害/效果
  ↓
抽混沌袋 1 顆標記
  ↓
判定標記類型並記錄數值與場景效果
  ↓
將最終數值與目標怪物的法術防禦值比較
  ↓
依 5.7 規則處理副作用
  ↓
結算場景效果（若有）
  ↓
法術結算完成
```

### 5.9 六大法術類型

| 代碼 | 中文 | 核心功能 | 副作用傾向 |
|------|------|---------|----------|
| `combat` | 戰鬥與毀滅 | 造成傷害 | 骷髏（肉體反噬） |
| `investigation` | 調查與預言 | 獲取資訊、發現線索 | 石版（精神侵蝕） |
| `protection` | 防護與迴避 | 保護、閃避 | 邪教徒（引起注意） |
| `spacetime` | 時空與位面操控 | 封印、傳送、空間操控 | 遠古邪物（環境異變） |
| `summoning` | 召喚與束縛 | 召喚援軍、控制敵人 | 邪教徒（黑暗勢力回應） |
| `healing` | 治癒與淨化 | 恢復、淨化 | 骷髏（以己之血換人之命） |

### 5.10 已確認規則

| 項目 | 決議 |
|------|------|
| 多次施法累積 | 每次獨立抽混沌袋，無疲勞機制 |
| 隊友影響 | 副作用可能波及同地點隊友（視場景效果設計） |
| 抽取數量 | 固定抽 1 顆。抽多顆能力由成長系統解鎖 |
| 法術防禦值公開性 | 不公開，符合隱藏資訊設計 |

## 六、行動類型｜Action Types

### 6.1 基礎行動

調查員在調查員階段中，每花費 1 行動點可執行以下行動之一：

| 行動 | 代碼 | 效果 |
|------|------|------|
| 拿一點資源 | `gain_resource` | 獲得 1 資源 |
| 抽一張卡 | `draw_card` | 從牌庫抽 1 張到手牌 |
| 打出卡片 | `play_card` | 從手牌打出 1 張卡，需支付費用 |
| 攻擊 | `attack` | 對同地點的敵人發動攻擊 |
| 移動 | `move` | 移動到相鄰地點 |
| 調查 | `investigate` | 在當前地點搜索線索 |
| 嘲諷 | `taunt` | 將同地點未交戰的敵人拉入與你交戰 |
| 閃避 | `evade` | 嘗試脫離與敵人的交戰 |
| 執行卡片行動 | `use_card` | 使用場上資產卡標示的行動能力 |
| 消費手牌 | `consume` | 從手牌直接棄掉，觸發消費效果 |

### 6.2 移動細則

- **普通移動：** 花費 1 行動點，移動到相鄰地點。
- **障礙物移動：** 花費 2 行動點（例如翻牆、涉水、破門等）。
- 障礙物由地點之間的連接關係定義，在場景設計中設定。

### 6.3 場上資產卡的三種功能類型

| 功能類型 | 說明 | 範例 |
|---------|------|------|
| 新行動 | 增加一種新的行動，佔 1 行動點 | .45 手槍 — 花費 1 使用次數進行攻擊 |
| 反應 | 不佔行動點，特定條件滿足時觸發 | 護身符 — 受到恐懼傷害時抵消 1 點 |
| 資源承載 | 卡片上放置使用次數標記 | 彈藥、充能、證據等 |

卡面必須明確標示：行動類型、觸發條件（反應類）、費用、效果、使用限制。

## 七、戰鬥規則｜Combat Rules

### 7.1 攻擊範圍

- 同地點的所有敵人都可以攻擊，不限於與自己交戰中的敵人。

### 7.2 交戰狀態

- 怪物依據自身偏好選擇交戰對象。
- **交戰限制：** 與怪物交戰中的調查員，只能執行攻擊和閃避。若執行其他行動，觸發**藉機攻擊**。
- **藉機攻擊後果：** 同時受到敵人的物理傷害和恐懼傷害（雙重懲罰）。

### 7.3 嘲諷

- 將同地點中**未與你交戰的敵人**改為與你交戰。
- 坦克型角色的核心行為 — 主動把威脅拉到自己身上。

### 7.4 閃避

| 結果 | 效果 |
|------|------|
| 閃避成功 | 敵人被絆倒 + 調查員脫離交戰 |
| 閃避失敗 | 調查員受到敵人物理傷害一次 + 脫離交戰 |

閃避無論成敗都會脫離交戰，但失敗的代價是挨一下。

### 7.5 攻擊結算

| 擲骰結果 | 效果 |
|----------|------|
| 命中（≥ DC） | 造成武器卡面固定傷害 |
| 自然 20 | 爆擊 — 2 倍傷害 |
| 自然 1（有交戰隊友） | 打到隊友（造成武器傷害） |
| 自然 1（無交戰隊友） | 單純未命中 |
| 未命中（< DC） | 未造成傷害 |

### 7.6 恐懼檢定

- **觸發時機：** 第一次看到怪物時（進入怪物所在地點，或怪物進入你的地點）。
- **檢定方式：** 意志（Willpower）檢定。
- **失敗後果：** 受到怪物的恐懼傷害，恐懼值印在怪物卡上。

### 7.7 恐懼半徑

每個怪物有兩個獨立參數：

| 參數 | 說明 |
|------|------|
| 恐懼半徑 | 在幾個地點遠就能觸發恐懼檢定 |
| 恐懼值 | 恐懼檢定失敗時造成的 SAN 傷害 |

兩者獨立設計：遠處嚎叫的怪物可能恐懼半徑 5 但恐懼值 1；近身的深潛者恐懼半徑 1 但恐懼值 3。巨頭與舊日支配者全場恐懼、每回合檢定。

### 7.8 隱藏資訊設計

**敵人的數值對玩家不透明：**

- 敵人血量不顯示數字，以敘事性狀態描述取代（「牠看起來毫髮無傷」→「牠拖著斷裂的肢體在地上爬行」）。
- 敵人防禦屬性不直接公開，攻擊只回饋結果描述（「子彈打在甲殼上彈開了」→ 推測物理抗性）。
- 高感知調查員或特定卡牌效果可揭示更多資訊。

## 八、戰鬥風格卡系統｜Combat Style Card System

### 8.1 核心概念

**誰發起攻擊，誰抽戰鬥風格卡。** 被攻擊方根據卡面指示做對應的檢定。每一次攻擊不是單純的「擲骰比數字」，而是抽出一張風格卡，呈現一段敘事，並決定本次檢定的屬性。

### 8.2 調查員攻擊流程

```
調查員選擇攻擊方式（由武器的 combat_style 決定）
  ↓
系統從該攻擊方式的風格卡池中隨機抽 1 張
  ↓
呈現敘事描述
  ↓
卡面指定本次檢定屬性
  ↓
調查員擲 d20 + 屬性修正 + 熟練/專精修正 + 武器修正（僅對應屬性生效）
  ↓
結果 ≥ 怪物 DC → 命中，造成武器卡面固定傷害
結果 < 怪物 DC → 未命中
自然 20 → 爆擊（2 倍傷害）
自然 1 → 大失敗
```

### 8.3 怪物攻擊流程

```
怪物從招式池中隨機抽 1 張怪物戰鬥風格卡
  ↓
呈現攻擊敘事
  ↓
卡面指定：防禦檢定屬性 + DC + 傷害值
  ↓
調查員擲 d20 + 屬性修正 + 卡片修正
  ↓
結果 ≥ DC → 防禦成功
結果 < DC → 承受卡面傷害
```

### 8.4 神秘攻擊例外

施法類攻擊（`combat_style: arcane`）不擲骰，改為抽混沌袋（見第五節）。

### 8.5 調查員風格卡結構

| 欄位 | 說明 |
|------|------|
| 歸屬攻擊方式 | 第一層戰鬥風格（8 種之一） |
| 檢定屬性 | 本次檢定使用的屬性 |
| 攻擊敘事 | 攻擊前的描述文字 |
| 成功敘事 | 命中時的描述文字 |
| 失敗敘事 | 未命中時的描述文字 |

- DC 不在風格卡上 — 來自目標怪物。
- 傷害不在風格卡上 — 來自武器卡。
- 風格卡只決定「用什麼屬性檢定」和「敘事怎麼說」。

### 8.6 怪物風格卡結構

| 欄位 | 說明 |
|------|------|
| 歸屬怪物 | 屬於哪隻或哪類怪物 |
| 攻擊方式名稱 | 招式名稱 |
| 防禦檢定屬性 | 調查員需要用哪個屬性防禦 |
| DC | 防禦檢定目標值 |
| 傷害值 | 防禦失敗時造成的傷害 |
| 傷害類型 | 物理傷害 / 恐懼傷害 / 兩者皆有 |
| 元素屬性 | 物理 / 火 / 冰 / 雷 / 神秘 |
| 攻擊/成功/失敗敘事 | 三段敘事文字 |

調查員風格卡與怪物風格卡是**完全獨立的資料結構**。

### 8.7 武器修正值與風格卡的關係

武器卡上的屬性修正（`attribute_modifiers`）**只在對應屬性的風格卡被抽到時生效**。

| 武器等級 | 修正結構 | 範例 |
|---------|---------|------|
| 0 級 | 單一屬性 +1 | `{ "strength": 1 }` |
| 1–2 級 | 單一屬性 +1 至 +2 | `{ "agility": 2 }` |
| 3 級 | 單一 +2 或雙屬性各 +1 | `{ "strength": 1, "agility": 1 }` |
| 4–5 級 | 可出現 `"all": 1` 或單一 +3 | `{ "all": 1 }` |

負面修正不出現在基礎卡上，只在玩家強化路線中作為取捨。

### 8.8 戰鬥熟練與風格卡池的關係

- 風格卡池是**公用的** — 有無熟練都從同一卡池抽卡。
- 差別在修正值：有熟練 = 額外加成，有專精 = 更高加成或特有效果。
- 卡面標籤是動態的 — 顯示調查員當前的熟練/專精等級名稱。

### 8.9 八種戰鬥風格與 30 種專精

| 戰鬥風格 | 代碼 | 專精方向 |
|---------|------|---------|
| 槍枝射擊 | `shooting` | 步槍、衝鋒槍、雙槍、手槍 |
| 弓術 | `archery` | 獵手、連射、毒箭、無聲射手 |
| 隨身武器 | `sidearm` | 匕首術、護身格擋、鈍擊、街頭格鬥 |
| 軍用武器 | `military` | 雙手武器、防禦架式、雙持、長柄武器 |
| 搏擊 | `brawl` | 酒館鬥毆者、摔角大師、空手道 |
| 施法 | `arcane` | 儀式、咒語、引導、冥想、煉金 |
| 工兵 | `engineer` | 爆破、陷阱、機械 |
| 暗殺 | `assassin` | 無聲處決、伏擊戰術、暗器 |

> 完整專精代碼列表見《數值規格文件》。

## 九、瀕死與救援系統｜Downed & Rescue System

### 9.1 觸發條件

HP 歸零或 SAN 歸零時，調查員進入瀕死狀態。HP 和 SAN 是獨立的，可能只有一項歸零，也可能兩項同時歸零。

### 9.2 瀕死狀態

- 瀕死調查員留在地點上，算躺在地上。
- 與其交戰中的怪物維持交戰關係。
- 瀕死調查員不能執行任何行動。
- 回合開始時以瀕死檢定取代正常行動。

### 9.3 瀕死檢定

| 項目 | 規則 |
|------|------|
| 檢定方式 | 擲 d20，DC 10 |
| 成功累積 3 次 | 自行穩定，站起來，歸零的數值恢復至 1 點 |
| 失敗累積 3 次 | 死亡，且該項歸零的數值上限永久 -1（創傷侵蝕） |
| 自然 20 | 直接站起來，歸零值恢復至 1 |
| 自然 1 | 算 2 次失敗 |
| 被怪物攻擊 | 每次命中算 1 次瀕死檢定失敗 |

### 9.4 HP 與 SAN 同時歸零

- 只做 1 組瀕死檢定。
- 站起來時 HP 和 SAN 都恢復 1 點。
- 死亡時 HP 和 SAN 上限都 -1。

### 9.5 隊友救援方式

| 方式 | 效果 | 是否需要檢定 |
|------|------|-------------|
| 治療 | 對瀕死調查員使用任何量的治療效果，立即脫離瀕死 | 否 |
| 穩定 | 花費 1 行動點，讓瀕死者的檢定計數增加 1 次成功 | 否，自動成功 |

### 9.6 永久死亡銜接

- 瀕死檢定失敗 3 次 → 死亡 → 歸零項上限 -1。
- 若上限 -1 後變為 0 → **永久死亡**，角色資料刪除。

## 十、敵人 AI 行為規則｜Enemy AI Behavior

### 10.1 設計前提

所有檢定都由玩家擲骰。城主是 AI，不會擲骰。

### 10.2 行動順序

敵人階段中，多隻怪物的行動順序依位階從高到低：

```
巨頭（Titan）→ 頭目（Boss）→ 精英（Elite）→ 威脅（Threat）→ 雜兵（Minion）
```

同位階依生成順序行動。

### 10.3 基礎行為模式

```
已與調查員交戰？
  ├ 是 → 攻擊交戰對象
  └ 否 → 朝目標移動 1 格
           └ 移動後若進入有調查員的地點 → 進入交戰
```

### 10.4 攻擊次數

| 位階 | 基礎攻擊次數 |
|------|-------------|
| 雜兵、威脅 | 1 次 |
| 精英 | 1 次（可透過詞綴增加） |
| 頭目 | 可透過設計增加至 2–3 次 |
| 巨頭 | 可透過設計增加至 2–3 次 |

### 10.5 偏好系統

| 偏好代碼 | 中文 | 說明 |
|---------|------|------|
| `nearest` | 最近 | 追蹤最近的調查員 |
| `lowest_hp` | 血量最低 | 追蹤當前 HP 最低的調查員 |
| `lowest_san` | 理智最低 | 追蹤當前 SAN 最低的調查員 |
| `most_clues` | 線索最多 | 追蹤持有最多線索的調查員 |
| `last_attacker` | 仇恨 | 追蹤上回合攻擊過牠的調查員 |
| `lowest_attr` | 屬性最低 | 追蹤指定屬性最低的調查員 |
| `random` | 隨機 | 隨機選擇 |

### 10.6 移動尋路

- 最短路徑前進，每回合 1 格（快速詞綴 2 格）。
- 多條等距路徑時隨機選一條。

## 十二、環境系統｜Environment System

### 12.1 光照系統

| 狀態 | 條件 | 效果 |
|------|------|------|
| 白天 | 所有地點全亮 | 無特殊效果 |
| 夜間 | 只有光源物件照明範圍內有視線 | 無視線時攻擊減值、閃避加值 |
| 黑暗 | 城主傳奇行動強制施放 | 覆蓋區域失去視線 |
| 失火 | 光源掉落或特定事件引發 | 提供光照（解除黑暗），但造成火焰傷害 |

- 失火與黑暗可在同一地點並存。
- 光源以光源物件為中心計算照明範圍，不以調查員為中心。
- 部分怪物不靠視覺，不受光照狀態影響。

### 12.2 地點環境狀態追蹤

每個地點需追蹤以下狀態：線索數量、隱藏調查點（是否已揭露）、光照狀態、失火狀態、場上的光源物件、場上的怪物與交戰關係、鬧鬼附著。

## 十三、調查系統｜Investigation System

### 13.1 地點調查屬性

每個地點具有以下調查相關屬性：

| 屬性 | 公開性 | 說明 |
|------|--------|------|
| 線索數量 | 明示 | 此地點有幾個線索可蒐集 |
| 調查難度 DC | 明示 | 進行調查檢定需達到的目標值 |
| 隱藏調查點 | 暗藏 | 各有自己的感知門檻 |

### 13.2 調查流程

1. 調查員進入地點時，系統自動比對調查員的**感知值**與隱藏調查點的門檻。
2. **感知達標** → 揭露隱藏調查點（「你注意到這裡有什麼不對勁」）。
3. **感知未達標** → 看不到隱藏調查點。

### 13.3 調查結果層次

| 情況 | 結果 |
|------|------|
| 一般調查成功 | 獲得 1 個線索 |
| 一般調查成功且觸發發現隱藏調查點 | 獲得線索 + 揭露隱藏調查點 |
| 直接對已知隱藏調查點調查成功 | 更豐厚的獎勵（稀有卡片、特殊情報、劇情解鎖等） |

### 13.4 感知的價值

- 高感知調查員：進入地點就能發現隱藏調查點。
- 低感知調查員：需多花行動點碰運氣（透過一般調查成功來觸發發現）。

## 十四、任務類型與過關機制｜Objective Types & Victory Conditions

### 14.1 五大任務類型

| 類型 | 中文 | 代碼 | 目標 |
|------|------|------|------|
| 類型一 | 封印次元門 | `seal_gate` | 花費線索 + 通過檢定關閉次元門 |
| 類型二 | 擊敗巨頭 | `defeat_titan` | 將巨頭 HP 降至 0 |
| 類型三 | 發覺真相 | `uncover_truth` | 找到隱藏線索或累積指定數量線索 |
| 類型四 | 逃脫 | `escape` | 所有調查員到達指定地點 |
| 類型五 | 撐到時間到 | `endurance` | 存活指定回合或等待特定條件 |

場景設計師可在同一場景中混搭多種任務類型。

### 14.2 過關判定

- 主線：多分支結果（最少 2 種，最多 5 種），由目標牌堆進度 × 議程牌堆進度 × 旗標組合判定。
- 支線：二元結果（通過/失敗），目標牌堆先完成 = 通過，議程牌堆先跑完 = 失敗。
- 設計傾向讓玩家能往下推進 — 不同結果影響後續路線，而非是否能繼續。
