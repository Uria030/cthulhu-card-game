# 核心設計原則 補充 01：遊戲規則與回合結構 v0.1
## Core Design Principles Supplement 01: Game Rules & Turn Structure v0.1

> **文件用途｜Purpose**
> 本文件定義遊戲的完整規則系統，包含遊戲大循環、回合結構、行動類型、戰鬥規則、
> 環境系統、調查系統等。所有卡片設計、關卡設計、程式實作皆以本文件為規則權威。
>
> 本文件從《核心設計原則》展開，是支柱 2（雙軌檢定）、支柱 3（章節制冒險）、
> 支柱 8（探險驅動構築）以及補充系統 E（戰鬥系統設計）的規則實作層定義。

## 一、遊戲大循環｜Game Loop

調查員的完整遊戲流程如下，每一段戰役（Campaign）循環一次：

```
遊戲大廳（城鎮）
  ↓
選擇關卡
  ↓
劇情演示 + 劇情選項
  ↓
個人秘密任務選擇
  ↓
場景遊玩模式（回合制：探索、戰鬥、調查）
  ↓
遊玩結束 → 結果判定（A / B / C 三種結果）
  ↓
長休息 + 整備模式
  ↓
選擇下一關
  ↓
（循環直到戰役劇情全破）
  ↓
回到遊戲大廳
```

### 1.1 遊戲大廳（城鎮）

- 戰役的起點與終點。
- 玩家在此選擇要進行的關卡。
- 其他玩家可在大廳中加入隊伍、觀看、參與聊天。
- 社交功能（聊天、觀戰、等待加入）為後期版本功能，核心功能（選關、整備）優先。

### 1.2 劇情演示

- 進入關卡後，首先播放劇情演示。
- 劇情中會出現**劇情選項**供玩家選擇，影響後續劇情走向與關卡內容。

### 1.3 個人秘密任務

- 每位玩家在進入場景前會獲得一個**個人秘密小任務**。
- **其他隊友看不到**此任務的內容。
- 過關本身會給予天賦點，但**達成個人秘密任務會額外獎勵天賦點**，依照任務難度分級。
- **設計意圖：** 創造微妙的團隊張力 — 你可能為了完成秘密任務而做出隊友看起來「奇怪」的行動，但你不能解釋原因。完美呼應克蘇魯世界觀中「每個人都有自己的秘密與動機」。

### 1.4 結果判定

- 場景遊玩結束後判定結果，可能有 **A / B / C 三種結果**。
- **設計傾向讓玩家能往下推進** — 過關判定不會太難，無論如何都希望玩家可以進入下一章節。
- 不同結果會影響後續劇情路線（參見《核心設計原則》系統 D：分支路線）。

### 1.5 長休息與整備模式

- 每關結束後進入**長休息**，獲得 **1 點凝聚力**。
- 整備模式中玩家可以進行以下操作：
  - 花費**天賦點** → 天賦樹
  - 花費**經驗點** → 技能樹買卡
  - 花費**凝聚力** → 團隊精神
  - **強化卡片**
  - **調整下一關要帶的牌組**
  - **換別的調查員**加入團隊（同一玩家切換角色，或邀請其他玩家）

## 二、回合結構｜Turn Structure

場景遊玩模式中的每一回合（Round）分為以下階段：

```
回合開始
  ↓
短休息決定（每位調查員各自決定）
  ├ 選擇短休息 → 執行短休息效果 → 該調查員本回合結束（不進入行動階段）
  └ 不休息 → 進入調查員階段
  ↓
調查員階段（3 行動點，自由順序，非固定輪流）
  └ 城主可能在此階段中插入傳奇行動
  ↓
敵人階段（城主行動）
  ├ 控制怪物移動與攻擊
  ├ 翻開一張神話卡（立即效果 + 持續效果）
  └ 若有 Agenda → 放置毀滅標記 → 到達門檻時翻開結算
  ↓
回合結束階段
  ├ 每位調查員抽 1 張卡
  ├ 每位調查員獲得 1 資源
  └ 所有橫置卡片轉正（重置一回合一次的能力）
  ↓
下一回合
```

### 2.1 調查員階段細則

- 每位調查員擁有 **3 行動點**。
- 調查員之間**不需要固定輪流** — 可以自由溝通協調行動順序。
- 每位調查員可以一口氣用完自己的 3 行動點。
- 同一調查員的行動順序也是自由的（不強制按特定順序使用行動點）。
- **城主傳奇行動**可在調查員行動期間穿插觸發，觸發條件在關卡中預先設計。調查員看到的呈現方式是「有神秘的事情發生了」。

### 2.2 敵人階段細則

- 城主（系統 AI）控制所有怪物的**移動與攻擊**。
- 怪物的行為依據其**關鍵字（詞綴）**與**位階**決定。
- 翻開一張**神話卡**：
  - 神話卡有**立即效果**與**持續效果**。
  - 若場上有 Agenda 牌堆，放置一個**毀滅標記（Doom Token）**到 Agenda 上。
  - 當 Agenda 上的毀滅標記達到卡面門檻時，翻開 Agenda 結算背面效果。

### 2.3 回合結束階段

- 每位調查員**抽 1 張卡**（從自己的牌庫）。
- 每位調查員**獲得 1 資源**。
- 所有**橫置（Exhausted）的卡片轉正** — 重置一回合限用一次的能力。
- 若手牌超過上限（8 張），須棄牌至上限。

## 三、短休息｜Short Rest

### 3.1 核心規則

- 在回合開始時，每位調查員**各自決定**是否進行短休息。
- 選擇短休息的調查員：
  - **重洗牌庫**（將棄牌堆洗回牌庫）。
  - **短休息消耗品回歸牌庫**（標記為「短休息可回收」的消耗品卡片回歸牌庫）。
  - 觸發**陣營專屬短休息效果**（見 3.2）。
  - **犧牲整回合行動** — 不進入調查員階段，不執行任何行動。
- 可透過特定能力解鎖短休息中的額外行動（待設計）。

### 3.2 陣營短休息效果（方向草案，待正式設計）

| 陣營極 | 中文 | 短休息效果方向 |
|--------|------|---------------|
| E | 號令 | 隊友也獲得某種小好處 |
| I | 深淵 | 看牌庫頂數張重新排列 |
| S | 鐵證 | 回收一件已消耗的裝備 |
| N | 天啟 | 窺探混沌袋 |
| T | 解析 | 額外獲得資源 |
| F | 聖燼 | 回復少量 SAN |
| J | 鐵壁 | 獲得臨時護甲 |
| P | 流影 | 從棄牌堆撈一張卡回手牌 |

> **待設計：** 各陣營短休息效果的具體數值與觸發條件。

### 3.3 牌庫耗盡規則

- 當調查員的牌庫為空，且需要抽牌時：
  - **不自動重洗牌庫。**
  - 每抽一張卡（從空牌庫），調查員**受到 1 點恐懼傷害（SAN 損失）**。
  - 牌庫耗盡代表調查員的心智資源枯竭，繼續硬撐會侵蝕理智。
- **設計意圖：** 迫使玩家在「繼續硬撐」與「花一整回合短休息重洗牌庫」之間做出有重量的抉擇。Agenda 上的毀滅標記每回合都在增加，多休息一回合就是讓敵人計畫多推進一步。

## 四、行動類型｜Action Types

調查員在調查員階段中，每花費 **1 行動點**可執行以下行動之一：

| 行動 | 效果 | 備註 |
|------|------|------|
| **拿一點資源** | 獲得 1 資源 | 基礎經濟行動 |
| **抽一張卡** | 從牌庫抽 1 張到手牌 | 回合結束階段之外的額外抽牌 |
| **打牌** | 從手牌打出 1 張卡到場上 | 需支付卡面費用，卡片進場 |
| **攻擊** | 對同地圖板塊的敵人發動攻擊 | 無武器時為空手攻擊 |
| **移動** | 移動到相鄰地圖板塊 | 普通移動花 1 行動點 |
| **調查** | 在當前地圖板塊搜索線索 | 進行調查檢定 |
| **嘲諷** | 將同板塊未與你交戰的敵人拉入與你交戰 | 坦克行為，保護隊友 |
| **閃避** | 嘗試脫離與敵人的交戰 | 成功/失敗見戰鬥規則 |
| **執行卡片行動** | 使用場上資產卡標示的行動能力 | 例如開槍（消耗子彈）、施法等 |

### 4.1 移動細則

- **普通移動：** 花費 1 行動點，移動到相鄰地圖板塊。
- **障礙物移動：** 花費 2 行動點（例如翻牆、涉水、破門等）。
- 障礙物由**地圖板塊之間的連接關係**定義，在關卡設計中設定。

### 4.2 場上資產卡的三種功能類型

場上的資產卡（已打出到場上的卡片）可以提供以下三種功能：

| 功能類型 | 說明 | 範例 |
|---------|------|------|
| **新行動** | 增加一種新的行動讓調查員執行，佔 1 行動點 | .45 手槍 — 花費 1 子彈進行攻擊 |
| **反應** | 不佔行動點，當特定條件滿足時自動觸發 | 「如果……就……」型效果 |
| **資源承載** | 卡片上放置資源標記，依設計有不同用途 | 彈藥、使用次數、充能等 |

**卡面必須明確標示的資訊：**

- **行動類型** — 這是新行動（佔 1 行動點）、反應（不佔行動點，條件觸發）、還是被動（持續生效）
- **觸發條件** — 反應類必須寫清「如果…」的條件
- **費用** — 消耗什麼（子彈、卡上資源、HP、SAN、棄手牌等）
- **效果** — 做了之後發生什麼
- **使用限制** — 一回合一次（需橫置）、無限制、整場遊戲一次等

## 五、戰鬥規則｜Combat Rules

### 5.1 攻擊範圍

- **同地圖板塊的所有敵人都可以攻擊**，不限於與自己交戰中的敵人。
- 不同的**戰鬥專精**影響攻擊方式的效果，空手未必比拿武器弱。

### 5.2 交戰狀態（Engaged）

- 怪物會依據自身的**偏好**選擇交戰對象。城主依照偏好控制怪物前進方向與交戰目標。
- **交戰的限制效果：** 與怪物交戰中的調查員，**只能執行攻擊和閃避**。若執行其他行動，會觸發**藉機攻擊（Attack of Opportunity）**。
- **藉機攻擊後果：** 同時受到敵人的**物理傷害和恐懼傷害**（雙重懲罰）。

### 5.3 嘲諷

- 將同地圖板塊中**未與你交戰的敵人**，改為與你交戰。
- 是坦克型角色的核心行為 — 主動把威脅拉到自己身上，讓隊友脫離交戰限制。

### 5.4 閃避

- 嘗試脫離與敵人的交戰。
- **閃避成功：** 敵人被絆倒 + 調查員脫離交戰。
- **閃避失敗：** 調查員受到敵人**物理傷害一次** + 脫離交戰。
- 閃避無論成敗都會脫離交戰，但失敗的代價是挨一下。

### 5.5 攻擊結算

| 擲骰結果 | 效果 |
|----------|------|
| 命中（≥ DC） | 造成卡面固定傷害 |
| **自然 20（大成功）** | **爆擊 — 2 倍傷害** |
| **自然 1（大失敗）** | **有交戰隊友 → 打到隊友（造成武器傷害）** |
| 自然 1（無交戰隊友） | 單純未命中，透過敘事文字呈現失敗場景 |
| 未命中（< DC） | 未造成傷害 |

> **備註：** 自然 1 造成的「武器掉落」等額外懲罰不寫在基礎規則中，由卡片效果（遭遇卡、環境效果等）引入，保持基礎規則簡潔。

### 5.6 恐懼檢定

- **觸發時機：** 第一次看到怪物時（進入怪物所在板塊，或怪物進入你的板塊）。
- **檢定方式：** 意志（Willpower）檢定。
- **失敗後果：** 受到怪物的**恐懼傷害（SAN 損失）**，恐懼值印在怪物卡上。
- 第一次被怪物傷害後，你就知道該怪物的傷害值和恐懼值 — 符合 E-4 隱藏資訊設計。

### 5.7 恐懼半徑

- 每個怪物有兩個獨立的恐懼參數：
  - **恐懼半徑** — 在幾個板塊遠就能觸發恐懼檢定。
  - **恐懼值** — 恐懼檢定失敗時造成的 SAN 傷害。
- 這兩個參數獨立設計：
  - 遠處嚎叫的怪物可能恐懼半徑 5 但恐懼值只有 1。
  - 近身的深潛者可能恐懼半徑 1 但恐懼值 3。
  - 巨頭與舊日支配者 — **全場恐懼、每回合檢定**。

## 六、怪物詞綴系統｜Monster Keywords

每個怪物可擁有一個或多個詞綴（Keyword），決定該怪物的特殊能力。某些詞綴**只會出現在特定位階之上**（例如「遠古邪神」級別的詞綴）。

### 6.1 移動類

| 詞綴 | 效果 |
|------|------|
| **快速（Swift）** | 每回合移動 2 格（正常為 1 格） |
| **飛行（Flying）** | 不在地圖上移動。從地圖上移除，下回合放置在**敏捷最低**的調查員板塊，與其交戰 |

### 6.2 交戰類

| 詞綴 | 效果 |
|------|------|
| **獵手（Hunter）** | 移動後若進入交戰，**立刻額外攻擊一次** |
| **巨大（Massive）** | 與板塊中**所有調查員**交戰（無法只讓一個人扛） |
| **冷漠（Apathetic）** | 不主動與調查員交戰（中立生物或需特定條件才敵對） |

### 6.3 死亡效果類

| 詞綴 | 效果 |
|------|------|
| **壓垮（Crush）（數字）** | 被擊敗後，同板塊調查員做閃避檢定，失敗受到（數字）點物理傷害 |
| **詛咒（Curse）（數字）** | 壓垮的恐懼版本，失敗受到（數字）點 SAN 傷害 |
| **鬧鬼（Haunting）** | 死亡後**附著在地圖板塊上**。此板塊有調查員調查檢定失敗時，此怪復活 |

### 6.4 防禦類

| 詞綴 | 效果 |
|------|------|
| **物理抗性（Physical Resistance）（數字）** | 減免（數字）點物理傷害 |
| **物理免疫（Physical Immunity）** | 不受物理傷害 |
| **法術抗性（Arcane Resistance）（數字）** | 減免（數字）點法術傷害 |
| **法術免疫（Arcane Immunity）** | 不受法術傷害 |

### 6.5 特殊類

| 詞綴 | 效果 |
|------|------|
| **群體（Swarm）（數字）** | 此怪有（數字）個分身一起行動 |

> **待擴充：** 以上為初版詞綴列表，未來將隨敵人設計持續增加新詞綴。

## 七、光照系統｜Lighting System

### 7.1 基本規則

| 狀態 | 條件 | 效果 |
|------|------|------|
| **白天** | 所有板塊全亮 | 無特殊效果 |
| **夜間** | 只有光源物件照明範圍內的板塊有視線 | 無視線時攻擊減值、閃避加值 |
| **黑暗** | 城主傳奇行動強制施放 | 覆蓋區域失去視線 |
| **失火** | 光源掉落地面或特定事件引發 | 提供光照（解除黑暗），但造成火焰傷害 |

### 7.2 光照與黑暗可並存

- 失火與黑暗可以在同一板塊並存 — 火災解除黑暗提供光照，但同時帶來火焰傷害的危險。
- 玩家可主動開燈或點火來恢復視線，但需承擔火災風險。

### 7.3 光源物件

- 光源以**光源物件為中心**計算照明範圍，不以調查員為中心。
- 不同光源有不同的**照明半徑與持續時間**（例如火把、手電筒、油燈）。
- 光源物件掉落地面可能引發火災。
- 「光源」本身是一種重要的**消耗品卡片類別**。

### 7.4 無視線效果

- **攻擊檢定減值** — 看不清目標，打不準。
- **閃避檢定加值** — 黑暗中更容易躲藏。
- **部分怪物不受影響** — 不靠視覺的怪物（靠聲音、震動、超感知等）不受光照狀態影響。

### 7.5 地圖板塊環境狀態追蹤

每個地圖板塊需要追蹤以下狀態：
- 線索數量（剩餘）
- 隱藏調查點（是否已揭露）
- 光照狀態（亮 / 暗 / 黑暗）
- 失火狀態
- 場上的光源物件
- 場上的怪物與其交戰關係
- 鬧鬼附著（若有）

## 八、調查系統｜Investigation System

### 8.1 地圖板塊資訊

每個地圖板塊具有以下調查相關屬性：
- **線索數量**（明示）— 此板塊有幾個線索可蒐集。
- **調查難度 DC**（明示）— 進行調查檢定需要達到的目標值。
- **隱藏調查點**（暗藏）— 一個或多個，各有自己的**感知門檻**。

### 8.2 調查流程

1. 調查員進入地圖板塊時，系統自動比對調查員的**感知值**與隱藏調查點的門檻。
2. **感知達標** → 揭露隱藏調查點的存在（「你注意到這裡有什麼不對勁」）。
3. **感知未達標** → 看不到隱藏調查點。

### 8.3 調查結果層次

| 情況 | 結果 |
|------|------|
| 一般調查成功 | 獲得 1 個線索 |
| 一般調查成功且觸發發現隱藏調查點 | 獲得線索 + 揭露隱藏調查點 |
| 直接對已知隱藏調查點調查成功 | 更豐厚的獎勵（稀有卡片、特殊情報、劇情解鎖等） |

### 8.4 感知的價值

- 高感知調查員：**看到別人看不到的機會** — 進入板塊就能發現隱藏調查點。
- 低感知調查員：不會完全錯過隱藏調查點，但需要**多花行動點碰運氣**（透過一般調查成功來觸發發現）。

## 九、任務類型與過關機制｜Objective Types & Victory Conditions

### 9.1 關卡結構

- 一個關卡（Chapter）由 **1–5 個場景**組成。
- 每個場景可以使用不同的任務類型，也可以**混搭多種任務類型**。
- 過關判定的 A/B/C 結果分級邏輯**由場景編輯器定義** — 關卡設計師自行設定每個結果的觸發條件與後續分支走向，保持最大設計彈性。

### 9.2 五大任務類型

#### 類型一：封印次元門｜Seal the Gate

- **目標：** 處理場景中四散的次元門。
- **封印流程：**
  - 花費線索（數量 × 玩家人數 P）。
  - 進行**智力檢定或攻擊檢定**（可以使用武器）。
  - 成功 → 關閉該次元門。
- **特殊道具：** 使用**遠古印記（Elder Sign）**卡片可直接成功關閉次元門，無需檢定。
- **設計意圖：** 同時考驗資源管理（線索蒐集）和行動能力（檢定），讓調查型和戰鬥型角色都能參與封印。

#### 類型二：擊敗巨頭｜Defeat the Titan

- **目標：** 擊敗場景中指定的巨頭級敵人。
- **基本形式：** 將巨頭 HP 降至 0。
- **進階設計（讓非戰鬥角色也有參與感）：**
  - 花費線索可對巨頭造成傷害。
  - 在找到隱藏線索前，無法對巨頭造成傷害（需先調查揭露弱點）。
  - 需累積指定數量的線索才能解除巨頭的護盾或無敵狀態。
- **設計意圖：** 避免「只有戰鬥角色有事做」的問題。調查員蒐集線索、揭露弱點，戰鬥員輸出傷害，團隊分工合作。

#### 類型三：發覺真相｜Uncover the Truth

- **目標：** 找到隱藏的線索，或累積指定數量的線索。
- **形式：**
  - 找到特定的隱藏調查點（需感知門檻或多次調查）。
  - 在場景中累積 N 個線索（N 可能依玩家人數縮放）。
- **設計意圖：** 純調查型任務，感知和智力屬性的舞台。適合偵探、學者類型的場景。

#### 類型四：逃脫｜Escape

- **目標：** 所有（或指定數量的）調查員到達指定地圖板塊。
- **設計意圖：** 強調移動、走位、路線規劃。通常搭配追兵壓力（怪物追趕）或環境威脅（火災蔓延、建築崩塌），讓「跑」本身成為有戰術深度的行為。

#### 類型五：時間到｜Endurance

- **目標：** 滿足特定時間條件。
- **形式：**
  - 存活 N 回合。
  - 經過 N 回合後觸發特定事件。
  - 觸發某個機制後時間到（例如完成儀式、等待救援抵達）。
- **設計意圖：** 防守型任務，考驗資源管理和生存能力。每一回合的 Agenda 推進都是壓力，玩家需要在「撐住」和「不被消耗殆盡」之間取得平衡。

### 9.3 任務類型混搭

場景設計師可在同一場景中組合多種任務類型，創造更複雜的挑戰：

- 「封印三個次元門，同時存活到第八回合」（類型一 + 類型五）
- 「找到巨頭的弱點線索，然後擊敗巨頭」（類型三 → 類型二）
- 「擊敗巨頭後，逃出正在崩塌的神殿」（類型二 → 類型四）
- 「在十回合內找到真相並封印次元門」（類型五 + 類型三 + 類型一）

### 9.4 過關判定

- 場景結束時判定結果，可能有 **A / B / C 三種結果**。
- **判定條件由場景編輯器定義** — 例如：
  - A：所有次元門封印 + 無人陣亡
  - B：至少封印一半次元門
  - C：Agenda 跑完或全滅，但劇情仍可推進
- **設計傾向讓玩家能往下推進** — 過關判定不會太難，不同結果影響的是後續劇情路線和獎勵，而非是否能繼續遊戲。

## 十、卡片類別補充｜Card Type Supplement

### 9.1 盟友卡（Ally）數值

盟友卡打出後留在場上，如同一個小隊友。盟友具有獨立的生存值：

| 屬性 | 說明 |
|------|------|
| **盟友生命值（Ally HP）** | 盟友能承受的物理傷害量 |
| **盟友理智值（Ally SAN）** | 盟友能承受的恐懼傷害量 |

- 任一項歸零時，盟友被**摧毀**，從場上移除進入棄牌堆。
- 調查員可以將受到的傷害或恐懼**分配到盟友身上** — 讓盟友替你承受傷害。
- **設計意圖：** 盟友是有限的保護資源。讓你的線人替你擋一刀、讓你的助手承受恐懼，在克蘇魯的世界觀下充滿情感重量 — 你的夥伴為你犧牲了。

### 9.2 對 card_definitions 資料表的補充

以下欄位需新增至 `card_definitions`：

| 欄位 | 類型 | 說明 |
|------|------|------|
| `ally_hp` | INTEGER | 盟友生命值（僅盟友類卡片使用，其他卡片為 NULL） |
| `ally_san` | INTEGER | 盟友理智值（僅盟友類卡片使用，其他卡片為 NULL） |

> 特別注意以下設計精神：

> 6. **感知是探索的核心屬性** — 在戰鬥中觀察敵人（E-4），在調查中發現隱藏點。
> 7. **個人秘密任務創造隊內張力** — 每個人都有自己的秘密，這就是克蘇魯。
