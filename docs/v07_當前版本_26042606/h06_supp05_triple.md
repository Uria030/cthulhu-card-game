# 核心設計原則 補充 05：三合一卡片用途系統 v0.1
## Core Design Principles Supplement 05: Triple-Use Card System v0.1

> **文件用途｜Purpose**
> 本文件定義每張卡片的三種互斥用途：打出（Play）、加值（Commit）、消費（Consume）。
> 三種用途各自獨立計算價值，互不影響，共同構成本遊戲「一張卡片三種選擇」的核心手牌決策。

> 本文件從《核心設計原則》的牌組構築精神展開，

>
> **依賴文件：**

> - 《卡片設計規範 v0.2》— 各類卡片規格

## 一、設計哲學｜Design Philosophy

### 1.1 為什麼要三合一？

Arkham Horror LCG 的牌庫過於厚重，原因之一是許多消耗性道具被設計成獨立的卡片。本遊戲的解決方案是：**每張卡片同時承載三種用途**，讓玩家手中的每一張牌都有多重選擇，減少「死牌」的出現，同時壓縮牌庫厚度。

### 1.2 三種用途互斥

每張手牌在使用的瞬間只能選擇一種用途：

| 用途 | 動作 | 結果 |
|------|------|------|
| **打出（Play）** | 花費資源與行動點，將卡片放到場上或發動一次性效果 | 依卡片類型留場或進棄牌堆 |
| **加值（Commit）** | 檢定前從手牌投入，提供屬性圖示加值 | 自動進棄牌堆 |
| **消費（Consume）** | 花費 1 行動點，從手牌直接棄掉，觸發一次性輔助效果 | 自動進棄牌堆 |

選擇其中一種，就放棄了其他兩種。這是手牌管理的核心決策。

### 1.3 價值獨立原則

三種用途各自獨立計算價值，互不影響：

- **打出效果的價值** → 決定費用與稀有度（既有的價值計算體系）
- **加值的價值** → 屬性圖示價值 + 附贈效果價值（技能卡用等級區間檢驗）
- **消費效果的價值** → 獨立計算，受稀有度或等級上限約束

一張卡片上配了很強的屬性圖示，不會讓它的打出費用變貴；反之亦然。

## 二、打出（Play）｜打出效果

> **完整規範請參照《卡片價值計算規範 v0.1》與《卡片設計規範 v0.2》。**

打出效果是卡片設計的主體，也是稀有度反推的唯一依據。本文件不重複已有規範，僅在此確認打出效果在三合一體系中的定位：

- 打出效果決定卡片的費用
- 打出效果決定卡片的稀有度（透過反推）
- 稀有度進而影響消費效果的價值上限

## 三、加值（Commit）｜屬性圖示系統

### 3.1 核心機制

每張卡片擁有**屬性圖示**，代表將這張卡從手牌投入（commit）到檢定時能提供的屬性修正。

| 項目 | 規格 |
|------|------|
| 觸發時機 | 檢定前，從手牌投入 |
| 行動點花費 | **不花費行動點** |
| 使用後去向 | 自動進棄牌堆 |
| 機會成本 | 放棄了打出或消費這張卡 |

### 3.2 屬性圖示規格

| 項目 | 規格 |
|------|------|
| 可標註的屬性 | 七大屬性（力量、敏捷、體質、智力、意志、感知、魅力） |
| 每個屬性的加值範圍 | +0 至 +5（極端情況可至 +6） |
| 一張卡片可標註的屬性數量 | 1 個以上，無硬性上限 |
| 萬能加值 | 可設計為「任意屬性 +X」的萬能圖示 |

### 3.3 屬性圖示的價值計算

屬性圖示的價值直接套用《卡片價值計算規範 v0.1》第 2.5 節的檢定修正曲線：

**單屬性加值：**

| 加值 | 價值 |
|------|------|
| +1 | 0.5V |
| +2 | 1.5V |
| +3 | 3V |
| +4 | 5V |
| +5 | 7.5V |

**萬能加值：**

| 加值 | 價值 |
|------|------|
| +1 | 1V |
| +2 | 3V |
| +3 | 6V |

多個屬性圖示的價值為各圖示價值之和。例如「力量 +1、敏捷 +1」= 0.5V + 0.5V = 1V。

### 3.4 技能卡的加值特權

技能卡除了屬性圖示之外，還可以附帶**成功/失敗時的附贈效果**。這是技能卡獨有的機制，其他類型的卡片 commit 時只有純數字修正。

| 項目 | 技能卡 | 非技能卡 |
|------|--------|---------|
| 屬性圖示 | ✓ | ✓ |
| 附贈效果（成功時） | ✓ | ✗ |
| 附贈效果（失敗時） | ✓ | ✗ |

### 3.5 加值價值的檢驗規則

**技能卡：**
- 屬性圖示價值 + 附贈效果價值 → 必須落在等級價值區間內

| 等級 | 價值區間 |
|------|----------|
| LV0 | 2.5V - 3V |
| LV1 | 2.5V - 3V |
| LV2 | 5V - 6V |
| LV3 | 7V - 8V |
| LV4 | 8V - 9V |
| LV5 | 9V - 10V |

**非技能卡：**
- 系統計算屬性圖示總價值，與對應稀有度的技能卡價值區間做比對
- **僅作為提示，不強制約束**
- 設計者根據卡片主題自然分配屬性圖示（手槍印敏捷、魔法書印意志等）

### 3.6 加值不影響打出費用

> **屬性圖示的價值不計入打出效果的費用計算。**

一張卡片印了多強的屬性圖示，都不會讓它的打出費用變貴。理由是使用加值時，玩家已經放棄了打出這張卡的機會，這本身就是足夠的代價。

## 四、消費（Consume）｜一次性輔助效果

### 4.1 核心機制

消費功能是卡片的第三種用途：不打出、不 commit，而是從手牌直接棄掉，換取一個即時的輔助效果。

| 項目 | 規格 |
|------|------|
| 觸發方式 | 從手牌直接棄掉 |
| 行動點花費 | **必須花費 1 行動點** |
| 使用後去向 | 自動進棄牌堆 |
| 機會成本 | 1 行動點 + 放棄了打出或 commit 這張卡 |

### 4.2 消費的固定成本

| 成本項目 | 價值 |
|----------|------|
| 1 行動點 | 1V |
| 棄掉 1 張手牌 | 0.5V |
| **固定成本合計** | **1.5V** |

消費效果必須 > 1.5V 才值得設計，否則玩家永遠不會選擇消費。

### 4.3 合法效果範圍

消費效果**不能是基本動作**（移動、攻擊、調查），方向限定為輔助性效果。

七種合法效果方向：

| 效果方向 | 說明 | 價值參考 |
|----------|------|----------|
| 獲得資源 | 獲得指定類型的資源 | 1V / 資源 |
| 補充彈藥/充能 | 為場上的武器或法術補充消耗 | 0.5V / 彈藥或充能 |
| 回復 HP | 恢復調查員的生命值 | 1.5V / HP |
| 回復 SAN | 恢復調查員的理智值 | 1.5V / SAN |
| 抽牌 | 從牌庫抽取卡片 | 1V / 張 |
| 獲得正面狀態 | 獲得強化、護甲、護盾等 | 依狀態價值表 |
| 移除負面狀態 | 移除中毒、流血、燃燒等 | 依狀態價值表 |
| 取消傷害/恐懼 | 在受傷前取消即將到來的傷害或恐懼 | 0.5V / 點 |

> **不合法的消費效果：** 移動、攻擊、調查、閃避、嘲諷 — 這些是花 1 行動點本來就能做的基本動作。

### 4.4 消費效果的價值上限

消費效果的價值受到卡片稀有度或等級的約束：

**非技能卡 — 由稀有度決定上限：**

| 稀有度 | 消費效果價值上限 |
|--------|----------------|
| 隨身 | 2.5V - 3V |
| 基礎 | 2.5V - 3V |
| 標準 | 5V - 6V |
| 進階 | 7V - 8V |
| 稀有 | 8V - 9V |
| 傳奇 | 9V - 10V |

**技能卡 — 由等級決定上限：**

| 等級 | 消費效果價值上限 |
|------|----------------|
| LV0 | 2.5V - 3V |
| LV1 | 2.5V - 3V |
| LV2 | 5V - 6V |
| LV3 | 7V - 8V |
| LV4 | 8V - 9V |
| LV5 | 9V - 10V |

### 4.5 消費效果不影響稀有度反推

> **稀有度反推只看打出效果，消費效果不計入。**

這意味著：想要強消費效果，必須同時有足夠強的打出效果來撐起稀有度。消費效果不能單獨把稀有度拉高。

### 4.6 消費效果不強制匹配

設計者可以自由選擇低於上限的消費效果強度。一張傳奇卡的消費效果可以只有 2.5V，這完全合理 — 設計重心在打出效果上，消費只是一個小退路。

也可以不設計消費效果（消費價值 = 0），代表這張卡片只有打出和加值兩種用途。

## 五、卡片設計流程（三合一完整版）｜Design Workflow

### 5.1 非技能卡設計流程

```
步驟 1：設計打出效果
  ↓ 決定卡片類型、效果、等級、費用
步驟 2：系統反推稀有度
  ↓ 純看打出效果的價值
步驟 3：設計屬性圖示（Commit）
  ↓ 選擇屬性與數量，系統提示比對（不強制）
步驟 4：設計消費效果（Consume）
  ↓ 從七種合法效果中選擇
  ↓ 系統檢驗：是否 ≤ 稀有度對應的上限
  ↓ 系統檢驗：是否 > 1.5V（否則提示不值得設計）
完成
```

### 5.2 技能卡設計流程

```
步驟 1：選擇等級
  ↓
步驟 2：設計屬性圖示 + 附贈效果（Commit）
  ↓ 圖示價值 + 附贈效果價值 → 檢驗是否落在等級價值區間
步驟 3：設計消費效果（Consume）
  ↓ 從七種合法效果中選擇
  ↓ 系統檢驗：是否 ≤ 等級對應的價值上限
  ↓ 系統檢驗：是否 > 1.5V（否則提示不值得設計）
完成
```

## 六、計算範例｜Calculation Examples

### 範例一：.45 自動手槍（標準武器，三合一視角）

**打出效果：**
- 類型：資產（武器）
- 傷害 3 × 彈藥 3 = 9V → 經公式計算 → 費用 3、等級 0、稀有度：標準

**屬性圖示（Commit）：**
- 敏捷 +1 = 0.5V
- 系統提示：0.5V，遠低於標準稀有度對應的 5-6V → 合理（非技能卡不強制）

**消費效果（Consume）：**
- 消費：獲得 2 資源 = 2V
- 稀有度上限：標準 = 5-6V → 2V ≤ 5V ✓
- 是否 > 1.5V → 2V > 1.5V ✓
- 合法

### 範例二：Vicious Blow LV0（技能卡，三合一視角）

**打出效果：** 無（技能卡費用固定為 0）

**屬性圖示 + 附贈效果（Commit）：**
- 屬性圖示：力量 +1 = 0.5V
- 附贈效果：成功時 +1 傷害 = 2.5V
- 總價值 = 3V → LV0 區間 2.5-3V ✓

**消費效果（Consume）：**
- 消費：獲得 2 資源 = 2V
- 等級上限：LV0 = 2.5-3V → 2V ≤ 2.5V ✓
- 是否 > 1.5V → 2V > 1.5V ✓
- 合法

### 範例三：高等級傳奇法術（三合一視角）

**打出效果：**
- 類型：資產（法術）
- 打出效果很強 → 經公式計算 → 費用 5、等級 5、稀有度：傳奇

**屬性圖示（Commit）：**
- 意志 +2、智力 +1 = 1.5V + 0.5V = 2V
- 系統提示：2V，低於傳奇對應的 9-10V → 合理

**消費效果（Consume）：**
- 消費：回復 3 SAN = 4.5V
- 稀有度上限：傳奇 = 9-10V → 4.5V ≤ 9V ✓
- 是否 > 1.5V → 4.5V > 1.5V ✓
- 合法

>
> 本文件定義的三合一系統是本遊戲區別於 Arkham Horror LCG 的核心創新之一。
> 特別注意以下設計精神：
>
> 1. **三種用途互斥** — 玩家使用卡片時只能選一種，這是手牌決策的核心張力。
> 2. **三種用途各自獨立計算價值** — 加值和消費不計入打出效果的費用計算。
> 3. **稀有度反推只看打出效果** — 消費效果不能單獨把稀有度拉高。

> 5. **消費必須花行動點** — 不是免費動作。
> 6. **消費效果不能是基本動作** — 移動、攻擊、調查不能作為消費效果。
> 7. **屬性圖示由卡片主題自然決定** — 手槍印敏捷、魔法書印意志，是主題驅動而非數值驅動。
> 8. **資料結構上必須區分場上能力的檢定修正與 commit 屬性圖示** — 這是兩個完全不同的概念，不能共用同一組 JSON 欄位。
