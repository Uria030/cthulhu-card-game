## 一、設計哲學｜Design Philosophy

### 1.1 核心定位

凝聚力與團隊精神系統的目的是**獎勵固定組隊的長期合作**，讓「一起走過地獄」的夥伴關係有機械上的實質回報。

### 1.2 三層設計意圖

| 層次 | 機制 | 體驗 |
|------|------|------|
| 資源稀缺 | 凝聚力獲取速度固定且緩慢 | 每一點都是慎重決策 |
| 廣度 vs 深度 | 最多選 7 種，但點滿需 6 點 | 隊伍必須在多元化和專精化之間抉擇 |
| 功能解鎖 | 鍛造、製作等核心功能鎖在團隊精神後面 | 成長節奏由團隊合作驅動 |

### 1.3 成長節奏

- **前期：** 資源有限，只能解鎖最急需的功能（例如鍛造或製作）
- **中期：** 開始展開第二、第三個團隊精神方向，隊伍風格逐漸成形
- **後期：** 點滿特定項目解鎖里程碑，隊伍打法產生質變

## 二、凝聚力（Cohesion）｜Cohesion Resource

### 2.1 基礎規則

| 項目 | 規格 |
|------|------|
| 資源性質 | 隊伍共用資源，非個人資源 |
| 獲取量 | 固定，不按玩家人數縮放 |
| 獲取來源 | 長休息（每次 +1）、支線通關 |
| 用途 | 投資團隊精神（採用 + 點數升級） |

### 2.2 隊長制

| 項目 | 規格 |
|------|------|
| 單人遊戲 | 玩家自己是隊長 |
| 多人遊戲 | 創立遊戲大廳的玩家是隊長 |
| 隊長職責 | 團隊紀錄存在隊長的本地端存檔 |

### 2.3 團隊紀錄管理

| 項目 | 規格 |
|------|------|
| 紀錄內容 | 團隊精神的投資狀態（已採用項目、各項目點數） |
| 紀錄儲存 | 隊長本地端，可存多筆 |
| 紀錄觸發 | 系統根據當前玩家組合自動匹配對應紀錄 |
| 舊紀錄保留 | 是，原班人馬回來可讀回 |

### 2.4 玩家變動規則

| 情境 | 處理方式 |
|------|---------|
| 玩家換調查員（死亡換人或主動換人） | **不影響**團隊紀錄 |
| 隊伍成員變動未超過一半 | 繼續使用當前紀錄 |
| 隊伍成員變動超過一半 | 開啟新的團隊紀錄 |

**範例：**

原始隊伍 ABCD 使用紀錄 1：
- ABCD → ABCE：未超過一半，繼續紀錄 1
- ABCE → ASCE：未超過一半，繼續紀錄 1
- ASCE → AXSE：超過一半（B、C、D 中已有 3 人不同），開啟紀錄 2
- 若 ABCD 重新聚齊：讀回紀錄 1

> **核心判定原則：** 凝聚力綁定的是**玩家組合**而非**角色組合**。是現實中一起打的那群人在累積默契，不是虛擬角色之間的羈絆。

## 三、團隊精神（Team Spirit）｜Team Spirit System

### 3.1 基礎規則

| 項目 | 規格 |
|------|------|
| 候選池 | 32 種 |
| 隊伍最多選擇 | 7 種 |
| 每種深度 | 5 點 |
| 每點花費 | 1 凝聚力 |
| 採用新項目花費 | 1 凝聚力 |
| 點滿一項總花費 | 6 凝聚力（1 採用 + 5 點數） |
| 團隊里程碑 | 點滿 5 點後解鎖，質變級進階能力 |

### 3.2 投資時機

團隊精神的採用與點數投資在**長休息時**進行。

## 五、書籍子系統｜Book Subsystem

### 5.1 書籍卡定義

書籍是一種**資產卡**，除了正常的打出效果外，附帶「閱讀」長期投資行動。

| 項目 | 規格 |
|------|------|
| 卡片類型 | 資產卡 |
| 正常效果 | 與其他資產卡相同（被動能力、行動能力等） |
| 附帶行動 | 「閱讀書籍」— 需花費行動點，多次執行 |

### 5.2 閱讀機制

| 項目 | 規格 |
|------|------|
| 閱讀方式（A） | 純累積次數 — 花費行動點，每次累積 1 次閱讀進度 |
| 閱讀方式（B） | 累積次數 + 智慧檢定 — 花費行動點並進行智慧檢定，成功才累積進度 |
| 所需次數 | 由卡片定義，通常較高（需投入大量行動點） |
| 進度保留 | **跨章節保留** |
| 完成標記 | 達成所需次數後，記錄「已了解此書的知識」 |

### 5.3 書籍分級

| 分級 | 說明 | 解鎖需求 |
|------|------|---------|
| 普通書籍 | 日記、筆記、報紙剪報、一般文獻 | 無，直接可閱讀 |
| 神話典籍 | 《死靈之書》《妖蛆之秘密》等禁忌文獻 | 需團隊精神「古文解讀」達到對應深度 |

### 5.4 古文解讀與書籍難度的對應

「古文解讀」團隊精神的 5 點深度對應書籍的難度等級：

| 深度 | 可解讀的書籍難度 | 代表書籍範例 |
|------|----------------|------------|
| 1 點 | 初階神話文獻 | 入門級翻譯文本、邪教筆記 |
| 2 點 | 中階神話文獻 | 《波納佩教典》《格拉基啟示錄》等 |
| 3 點 | 高階神話文獻 | 《伊波恩之書》《無名祭祀書》等 |
| 4 點 | 頂級神話文獻 | 《屍食教典儀》《妖蛆之秘密》等 |
| 5 點 | 終極禁忌典籍 | 《死靈之書》 |

> 以上為示意性分級，具體書籍歸屬由卡片設計時確定。

### 5.5 升級版取得

| 項目 | 規格 |
|------|------|
| 取得時機 | 長休息時 |
| 取得條件 | 已完成該書的閱讀 |
| 取得方式 | 花費經驗值購買 |
| 升級版內容 | **事先不公開**，購買時才揭曉 |
| 升級版形式 | 可能是替換原卡的全新卡片，也可能是在原卡上附加能力 |

### 5.6 書籍研究團隊精神的效果方向

「書籍研究」團隊精神強化閱讀效率，5 點深度可涵蓋：

- 減少書籍所需閱讀次數
- 降低智慧檢定的 DC
- 閱讀過程中就獲得部分好處（而非要全部讀完才有回報）
- 具體效果數值待設計

## 六、遺跡子系統｜Relic Subsystem

### 6.1 遺跡卡定義

遺跡是一種**資產卡**，與書籍平行，差別在於載體是實物而非文字。

| 項目 | 規格 |
|------|------|
| 卡片類型 | 資產卡 |
| 載體 | 石板、挖掘的古物、雕像、首飾等實物 |
| 正常效果 | 與其他資產卡相同 |
| 附帶行動 | 「研究遺跡」— 需花費行動點，多次執行 |

### 6.2 研究機制

與書籍閱讀機制完全平行：

| 項目 | 規格 |
|------|------|
| 研究方式（A） | 純累積次數 |
| 研究方式（B） | 累積次數 + 智慧檢定 |
| 進度保留 | 跨章節保留 |
| 完成標記 | 達成所需次數後，記錄「已了解此遺跡的秘密」 |

### 6.3 遺跡分級

| 分級 | 說明 | 解鎖需求 |
|------|------|---------|
| 普通遺物 | 一般古物、裝飾品 | 無，直接可研究 |
| 神話遺跡 | 帶有超自然力量的古代遺物 | 需團隊精神「遺跡解讀」達到對應深度 |

### 6.4 遺跡解讀與遺跡難度的對應

「遺跡解讀」團隊精神的 5 點深度對應遺跡的難度等級，結構與古文解讀相同。

### 6.5 升級版取得

與書籍升級版規則完全相同 — 長休息時購買，內容事先不公開。

### 6.6 書籍與遺跡的差異定位

| 面向 | 書籍 | 遺跡 |
|------|------|------|
| 知識路線 | 文字知識 — 閱讀、翻譯、理解 | 考古實物 — 觸摸、拼接、感應 |
| 解鎖管道 | 知識與神話類團隊精神 | 調查與資訊類團隊精神 |
| 主題調性 | 學者在燈下翻書 | 考古學家在遺址挖掘 |
| 設計意圖 | 兩條平行的知識獲取管道，讓不同風格的隊伍各有歸屬 |

```sql
  captain_user_id   UUID NOT NULL REFERENCES users(id),
  member_user_ids   UUID[] NOT NULL,              -- 當前成員的 user ID 列表
  spirit_investments JSONB NOT NULL DEFAULT '{}',  -- 各團隊精神的投資狀態
  cohesion_total    INTEGER NOT NULL DEFAULT 0,    -- 累計獲得的凝聚力
  cohesion_spent    INTEGER NOT NULL DEFAULT 0,    -- 已花費的凝聚力
  created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at        TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

> **給未來 Claude 實例的備註｜Note to Future Claude Instances**
>
> 本文件的所有設計均經過與專案原創者 Uria 的逐項討論確認。
> 特別注意以下設計精神：
>
> 1. **凝聚力綁定玩家組合，不綁定角色** — 換調查員不影響團隊紀錄。
> 2. **凝聚力獲取量固定** — 不按人數縮放，單人和四人獲取速度相同。
> 3. **鍛造和製作鎖在團隊精神後面** — 這是刻意的，創造前期的成長節奏。
> 4. **古文解讀和遺跡解讀分屬不同團隊精神類別** — 古文解讀在知識與神話類，遺跡解讀在調查與資訊類。
> 5. **書籍升級版事先不公開** — 玩家花大量行動閱讀，回到營地才揭曉學到了什麼。非常克蘇魯。
> 6. **書籍和遺跡是平行系統** — 機制相同，載體和主題不同，解鎖管道不同。
> 7. **怪物學按家族獨立** — 每個家族各一種團隊精神，未來新增家族時自然擴展。
> 8. **異常狀態專精按主題分組** — 不是一個團隊精神全吃到，而是分成腐蝕/寒霜/壓制/瓦解四組攻擊面 + 一組防禦面（淨化）。
> 9. **32 種團隊精神的具體 5 點深度效果尚未設計** — 這是後續工作。
> 10. **團隊紀錄的「超過一半變動」判定基於原始隊伍** — 範例：ABCD → AXSE，與原始 ABCD 比較，B/C/D 中有 3 人不同，超過一半。
