## 一、規範目的

本規範是**本專案繁體中文卡片敘述的統一標準**,非 Arkham 翻譯指南。

解決的三個問題:

1. **AI 生成不一致** —— 不同生成批次選用不同同義詞(消除 / 取消 / 忽略)
2. **設計者手寫偏差** —— 個別設計者對句型、標點、自指方式的習慣不同
3. **玩家理解成本** —— 術語混亂導致玩家需反覆查詢 FAQ

## 二、圖示與顏色代碼(測試版)

測試版尚未有美術圖案,所有動作類型、屬性、狀態一律以**有顏色的文字**代替。正式版美術完成後會統一替換為圖示。

### 2.1 行動類型標示(取代觸發符號)

| 本專案代碼 | 中文標示 | 顯示顏色 | 說明 |
|---|---|---|---|
| `action` | **「行動」** | 紅色 `#C62828` | 佔 1 行動點的主動能力 |
| `free_action` | **「免費行動」** | 藍色 `#1565C0` | 不佔行動點的主動能力 |
| `reaction` | **「反應」** | 綠色 `#2E7D32` | 條件觸發的反應能力 |
| `passive` | **「被動」** | 灰色 `#424242` | 持續生效的規則改寫 |
| `forced` | **「強制」** | 橙色 `#E65100` | 條件滿足必須觸發 |

**卡面呈現格式**:
```
【行動】費用:效果內容。
【免費行動】費用:效果內容。
【反應】觸發時機:效果內容。
【被動】效果內容。
【強制】觸發時機:效果內容。
```

**範例**:
- 【行動】花費 1 彈藥:發動攻擊。
- 【被動】你位於已揭示地點時,力量 +1。
- 【強制】在本卡離場時:對你造成 1 點傷害。

### 2.1b 三合一用途標示(與行動類型並列的另一維度)

三合一用途系統(支柱一 v0.2 §4)定義卡片的三種互斥用途。用途標示與行動類型標示**屬不同維度**:行動類型標的是「這個能力佔用什麼行動資源」,用途標的是「這張卡被以什麼方式投入遊戲」。兩者可以在同一張卡上並存(如一張技能卡可同時有【加值】與【消費】兩塊能力)。

| 本專案代碼 | 中文標示 | 顯示顏色 | 說明 |
|---|---|---|---|
| `play` | **「打出」** | (無專用色,用一般文字) | 從手牌正常打出(此標示通常省略) |
| `commit` | **「加值」** | 藍綠色 `#00796B` | 投入檢定的屬性加值用途 |
| `consume` | **「消費」** | 紫色 `#6A1B9A` | 棄置換效果的消費用途 |

**卡面呈現格式**:
```
【加值】在你進行 [屬性] 檢定時投入:效果內容。
【消費】花費 [成本]:效果內容。
```

**範例**:
- 【加值】在你進行意志檢定時投入。這次檢定 +2 意志。
- 【消費】花費 1 行動點:治癒 1 點恐懼。

**設計守則**:一張卡若同時設計了「加值」與「消費」兩種用途,兩塊能力在卡面**獨佔各自一行**,先寫【加值】後寫【消費】(按照三合一用途順序)。

## 四、取消 vs 忽略 vs 預防

Arkham 中譯將 Cancel 翻為「取消」或「忽略」造成混淆,本專案嚴格區分三個動詞:

| 動詞 | 代碼 | 語意 | 適用情境 |
|---|---|---|---|
| **取消** | `cancel` | 事件**不發生**,完全無效果 | 用反應類能力完全否定敵人動作 |
| **忽略** | `ignore` | 事件**發生但不影響你** | 規則豁免(如「忽略藉機攻擊」) |
| **預防** | `prevent` | 事件**不觸發**(上游阻斷) | 減免即將發生的傷害/恐懼 |

**範例**:
- 取消該次攻擊。(攻擊不發生)
- 你可以忽略藉機攻擊。(敵人會攻擊,但你不受影響)
- 預防所有即將對你造成的傷害。(傷害還未造成時阻斷)

## 六、主詞與自指規範

### 6.1 主詞統一使用「你」

卡面敘述主詞**一律用「你」**,指調查員本人。禁用:

- 調查員(過於書面)
- 玩家(破壞代入感)
- 使用者(技術感)
- 我(主詞錯位)

**範例對照**:
- 正確:「你獲得 1 資源。」
- 錯誤:「調查員獲得 1 資源。」、「該玩家獲得 1 資源。」

### 6.2 卡片自指的兩種方式

Arkham 樣本顯示有五種自指方式,本專案**統一為兩種**:

| 情境 | 使用方式 | 範例 |
|---|---|---|
| 通用自指 | **此卡** | 「將此卡移出遊戲。」 |
| 需要與同名卡區分 | **本 [卡名]** | 「治癒本急救箱的所有傷害。」 |

禁用:
- 重複卡名(「將急救箱移出遊戲,接著急救箱獲得...」)—— 冗贅
- 本卡牌 —— 與「此卡」語意重複
- 此、此處 —— 過於模糊

**選擇規則**:
- 牌組允許同名卡多張存在,且敘述需區分「場上這張」與「手牌另一張」時 → 用「本 [卡名]」
- 其他一律使用「此卡」

### 6.3 目標對象的指代

**第一次提及用完整描述,後續指代用「該 X」或「所選 X」**:

```
選擇 [修飾語][目標類別]:對該 [目標類別] 造成 X 點傷害。
```

**範例**:
- 正確:「選擇你所在地點的一名敵人:對該敵人造成 2 點傷害。」
- 錯誤:「對你所在地點的一名敵人造成 2 點傷害。對你所在地點的一名敵人再造成 1 點傷害。」(重複冗長)

### 6.4 盟友卡的性別代詞

盟友卡第一次提及用全名,後續用**他 / 她 / 牠 / 其**:

- 人類男性 → 他
- 人類女性 → 她
- 動物或神話生物 → 牠
- 群體、組織、無性別 → 其

**範例**:
- 「消耗約翰·史密斯並對他造成 1 點傷害。」
- 「消耗特工支援並對其造成 1 點傷害。」

## 七、卡名命名規範

### 7.1 卡名長度

- **2–8 個字**為正常範圍
- 超過 8 字需有設計理由(如簽名卡的特定典故)
- 事件卡允許使用標點符號強調語氣(如驚嘆號)

### 7.2 卡名結構

| 結構 | 適用類別 | 範例 |
|---|---|---|
| 純名詞 | 資產、事件 | 急救箱、醫療書籍 |
| 形容詞 + 名詞 | 資產 | 陳舊的鑰匙圈 |
| 人名 | 盟友 | 瑪麗·霍華德 |
| 動詞句 | 事件 | 我要走了! |
| 典故名 | 簽名卡、傳奇物品 | 伊波恩之書 |

### 7.3 副標題使用時機

副標題**非必填**,使用時機:

- **簽名卡**:氛圍句或典故(「讓暴風雨來得更猛烈些吧」)
- **盟友卡**:身份說明(「歐巴尼恩司機」、「資深獵手」)
- **雙面卡**:區分正反面狀態(「心神合一 / 運功」)
- **其他卡片**:原則上不使用副標題

副標題應**簡短有力**,字數控制在 2–8 字。

### 7.4 卡名禁用事項

- 禁用純英文(測試版中文字段除外)
- 禁用連串省略號、連串標點
- 禁用帶攻擊性、歧視性、政治性的用詞
- 禁用與克蘇魯神話世界觀衝突的現代流行語

## 八、卡片結構規範

### 8.1 效果區的排列順序

卡面效果區從上到下的順序**固定為**:

```
1. 關鍵字(如「快速」、「多重」、「卓越」、「永久」)
2. 使用次數聲明(如「使用 (4 彈藥)」)
3. 打出限制句(如「只有在... 時才能打出」)
4. 被動能力(無符號)
5. 主動能力(【行動】/【免費行動】/【反應】/【強制】)
6. 風味文字(視情況)
```

**範例**:
```
快速。使用 (3 充能)。
只有在你所在地點有敵人時才能打出。
你的意志 +1。
【反應】在你受到傷害時:花費 1 充能,取消該傷害。
```

### 8.2 使用次數聲明的標準格式

```
使用 (N [資源名稱])。
```

- 括號使用**半形**(英文括號)
- 數字與資源名稱間**無空格**
- 結尾接**全形句號**(。)

**範例**:
- 使用 (4 彈藥)。
- 使用 (3 充能)。
- 使用 (0 證據)。

### 8.3 關鍵字聲明的標準格式

多個關鍵字用**全形句號** 分隔,排在效果區最上方:

```
快速。多重。使用 (2 充能)。
```

### 8.4 能力區塊的換行

每個獨立能力**獨佔一行**,不同能力之間不留空行:

```
【行動】花費 1 彈藥:攻擊。本次攻擊 +1 傷害。
【免費行動】消耗此卡:抽 1 張牌。
```

被動能力與主動能力混合時,**被動先寫,主動後寫**:

```
你的力量 +1。
【行動】花費 1 彈藥:攻擊。
```

## 一、效果語言六要素結構

每個卡片效果由規則書第三章 §5 定義的六要素組成。書寫順序**固定為**:

```
【觸發時機】條件限制:費用 → 目標 → 效果 → 持續時間。
```

**完整範例**:

```
【反應】在你受到傷害時,若你位於已揭示地點:花費 1 充能,預防對你造成的所有傷害,直到本回合結束。
```

拆解:
- **觸發時機**:在你受到傷害時
- **條件限制**:若你位於已揭示地點
- **費用**:花費 1 充能
- **目標**:對你(隱含)
- **效果**:預防所有傷害
- **持續時間**:直到本回合結束

## 二、成本句型

### 2.1 基本成本句型

冒號作為成本與效果的分隔符:

```
【能力類型】[成本]:[效果]。
```

**範例**:
- 【行動】花費 1 彈藥:攻擊。
- 【免費行動】消耗此卡:抽 1 張牌。

### 2.2 無額外成本的主動能力

冒號前**留空**:

```
【行動】:[效果]。
```

**範例**:
- 【行動】:進行調查。
- 【行動】:攻擊。本次攻擊你 +1 力量。

### 2.3 複合成本的連接

兩個以上的成本用「**並**」連接:

```
[成本 A]並[成本 B]:[效果]。
```

**範例**:
- 【行動】消耗此卡並花費 1 證據:發現 1 線索。
- 【免費行動】消耗此卡並對此卡造成 1 點傷害:對敵人造成 1 點傷害。

### 2.4 成本類型的標準用詞

規則書第三章 §5.4 費用類型的卡面標準用詞:

| 代碼 | 卡面標準寫法 |
|---|---|
| `resource` | 花費 N 資源 |
| `forbidden_insight` | 花費 N 禁忌洞察 |
| `faith` | 花費 N 信仰 |
| `elder_sign` | 花費 1 遠古印記 |
| `hp` | 受到 N 點傷害 |
| `san` | 受到 N 點恐懼 |
| `discard_hand` | 棄 N 張手牌 |
| `discard_specific` | 棄掉 1 張 [條件] 的手牌 |
| `exhaust_self` | 消耗此卡 |
| `exhaust_other` | 消耗 1 張 [目標卡] |
| `uses` | 花費 N [使用次數名稱] |
| `clue` | 花費 1 線索 |
| `action_point` | 額外花費 1 行動點 |
| `doom` | 在此卡上放置 1 毀滅標記 |

## 三、條件句型

### 三 種條件句型的區分

| 句型 | 性質 | 用法 |
|---|---|---|
| **如果 X,Y** | 軟性條件 | 效果依條件分支,條件不滿足時效果為空 |
| **只要 X,Y** | 持續條件 | 被動效果,條件滿足時持續生效 |
| **只有在 X 時才能 Z** | 硬性限制 | 打出/觸發的前提,不滿足則不能執行 |

### 3.1 軟性條件:「如果 X,Y」

**標準格式**:
```
如果 [條件],[效果]。
```

**範例**:
- 如果你有至少 3 資源,抽 1 張牌。
- 如果此次攻擊擊敗一名敵人,治癒 1 點恐懼。
- 如果該檢定成功,發現 1 線索。

**失敗對稱句型**:
```
如果 [條件 A],[效果 A]。如果 [條件 B],[效果 B]。
```

**範例**:
- 進行智力檢定 (2)。如果成功,治癒 1 點傷害。如果失敗,對你造成 1 點傷害。

禁用「否則」、「反之」連接失敗分支,因為 `success` 與 `fail` 在程式層面是兩個獨立的觸發點。

### 3.2 持續條件:「只要 X,Y」

**標準格式**:
```
只要 [條件],[效果]。
```

**範例**:
- 只要你位於已揭示地點,你的感知 +1。
- 只要此卡在場上,所有武器的傷害 +1。
- 只要你有至少 5 張手牌,抽牌能力 +1。

「只要」表達**條件 → 效果** 的持續綁定,條件消失效果立即消失。

### 3.3 硬性限制:「只有在 X 時才能 Z」

**標準格式**:
```
只有在 [條件] 時才能 [動作]。
```

**範例**:
- 只有在你所在地點沒有敵人時才能打出。
- 只有在你的手牌有至少 3 張時才能觸發。

硬性限制放在**效果句之前**,作為獨立一行。

### 3.4 條件中的否定

| 正確 | 禁用 |
|---|---|
| 你沒有線索 | 你無線索、你不擁有線索 |
| 敵人不在你所在地點 | 敵人不與你同地點、無敵人在此 |
| 你未受到傷害 | 你沒被打、你沒扣血 |

### 3.5 數量條件的表達

| 條件 | 標準寫法 | 範例 |
|---|---|---|
| 下限 | 至少 N | 至少 3 資源 |
| 上限 | 至多 N | 至多 2 張手牌 |
| 等於 | 正好 N / 恰好 N | 正好 1 個線索 |
| 無下限 | 沒有 | 沒有線索 |
| 無上限 | 不超過 N | 手牌不超過 5 張 |

## 四、選擇句型

### 四 種選擇句型依複雜度區分

| 複雜度 | 句型 | 適用 |
|---|---|---|
| 簡單二擇 | 斜線 A/B | 兩個動作 |
| 中等選擇 | 選擇一項 —— A、B、C | 3 項以上擇 1 |
| 多選 | 選擇 N 項:◆ A ◆ B ◆ C | N 項擇多 |
| 條件分支 | 如果 X,A;如果 Y,B | 根據狀態決定 |

### 4.1 簡單二擇:斜線

**標準格式**:
```
[動作 A]/[動作 B]。
```

**範例**:
- 攻擊/閃避。
- 移動/調查。

僅用於**兩個動作選一個**,且動作名稱極短的情境。

### 4.2 多選一:破折號列舉

**標準格式**:
```
選擇一項 —— [選項 A]、[選項 B]、[選項 C]。
```

**範例**:
- 選擇一項 —— 治癒 1 點傷害、治癒 1 點恐懼、抽 1 張牌。
- 選擇一項 —— 移動到連接地點、發現該地點 1 線索、絆倒該地點一名敵人。

破折號使用**全形 em dash**(——,兩個長破折號),前後**不加空格**。

### 4.3 多選 N 項:項目符號

**標準格式**:
```
選擇 [N] 項:
◆ [選項 A]
◆ [選項 B]
◆ [選項 C]
```

**範例**:
```
選擇 2 項:
◆ 該卡獲得快速。
◆ 該卡費用 −1 資源。
◆ 此次結算中你所有檢定 +2。
```

項目符號使用 **◆**(黑色實心菱形),每個選項獨佔一行。

### 4.4 條件分支:等值觸發

**標準格式**:
```
如果 [狀態 A],[效果 A]。
如果 [狀態 B],[效果 B]。
如果 [狀態 C],[效果 C]。
```

**範例**:
```
如果此卡上正好有 1 充能,檢定 +3。
如果此卡上正好有 2 充能,你可以移動最多 2 次。
如果此卡上正好有 3 充能,你可以進行一次額外行動。
```

使用**互斥條件**(正好 N)時,效果不重疊。條件分支適用於**狀態驅動**的分支,不是玩家選擇。

## 五、時機句型

### 5.1 觸發時機代碼對應

規則書第三章 §5.1 的觸發時機代碼,卡面中文寫法:

| 代碼 | 卡面標準寫法 |
|---|---|
| `on_play` | 在此卡打出時 |
| `on_commit` | 在此卡加值投入時 |
| `on_consume` | 在此卡消費時 |
| `on_enter` | 在此卡進場時 |
| `on_leave` | 在此卡離場時 |
| `on_draw` | 在此卡被抽到時 |
| `on_success` | 在檢定成功時 |
| `on_fail` | 在檢定失敗時 |
| `on_take_damage` | 在你受到傷害時 |
| `on_take_horror` | 在你受到恐懼時 |
| `before_take_damage` | 在你將要受到傷害時 |
| `before_take_horror` | 在你將要受到恐懼時 |
| `before_downed` | 在你將要倒下時 |
| `round_start` | 在回合開始時 |
| `round_end` | 在回合結束時 |

### 5.2 「時」與「後」的精細區分

| 用詞 | 觸發點 | 範例 |
|---|---|---|
| 在 X 時 | 事件**發生的當下** | 在攻擊時 |
| 在 X 前 | 事件**即將發生** | 在將要受到傷害時(攔截窗口) |
| 在 X 後 | 事件**結算完成** | 在擊敗敵人後(戰利品結算) |

**範例對照**:
- 「在你受到傷害時」→ 傷害正在結算(可用防禦效果介入)
- 「在你受到傷害後」→ 傷害已結算完畢(結果已定,觸發後續效果)

### 5.3 時間範圍詞

| 詞彙 | 涵義 |
|---|---|
| **這次** | 當前單一動作範圍 |
| **下次** | 下一個同類動作 |
| **本回合** | 當前整個回合 |
| **本階段** | 當前遊戲階段 |
| **直到 X** | 至特定條件觸發為止 |
| **直到本回合結束** | 回合結束時效果解除 |

**範例**:
- 這次攻擊 +1 傷害。(僅當前攻擊)
- 下次技能檢定自動成功。(等待下個檢定才生效)
- 本回合你所有攻擊 +1 傷害。(整個回合持續)

## 六、結果句型

### 6.1 檢定結果的表達

規則書支援三層結果判定:**失敗 / 成功 / 超額成功**。

**二層判定(最常用)**:
```
如果成功,[效果 A]。如果失敗,[效果 B]。
```

**三層判定(階梯式)**:
```
如果失敗,[效果 A]。
如果成功且等於難度,[效果 B]。
你成功且每超過難度 1 點,[效果 C]。
```

**範例**:
```
進行意志檢定 (2)。如果失敗,對你造成 1 點恐懼。
你成功且每超過難度 1 點,你可以花費 1 充能治癒 1 點恐懼。
```

### 6.2 戰鬥結果

| 結果 | 標準寫法 |
|---|---|
| 命中 | 造成 [武器傷害] |
| 爆擊 | 造成雙倍傷害 |
| 大失敗 | 若有交戰隊友,打到隊友 |

### 6.3 擊敗後觸發

**標準格式**:
```
如果此次攻擊擊敗 [目標],[效果]。
```

**範例**:
- 如果此次攻擊擊敗一名敵人,治癒 1 點恐懼。
- 如果此次攻擊擊敗精英以上敵人,獲得 2 經驗值。

## 七、連接詞規範

### 7.1 連接詞的語意層級

| 連接詞 | 用途 | 範例 |
|---|---|---|
| **並** | 連接同類動作 / 複合成本 | 抽 1 張牌並獲得 1 資源 |
| **且** | 連接條件(邏輯 AND) | 如果你位於已揭示地點且有線索 |
| **或** | 連接選項(邏輯 OR) | 你所在地點或連接地點 |
| **然後** | 連接先後順序 | 抽 1 張牌。然後,棄 1 張牌 |
| **以** | 連接代價與目的 | 受到 1 點傷害,以治癒 3 點恐懼 |
| **若** | 文言的「如果」 | **禁用**,統一用「如果」 |
| **否則** | 反向結果 | **禁用**,統一用「如果失敗,...」 |
| **反之** | 相反情境 | **禁用**,統一用「如果 ...」另起一句 |

### 7.2 「和 / 與 / 及 / 跟」的處理

**統一使用「和」**,其他視情境:

| 用法 | 範例 |
|---|---|
| 名詞並列 → 用「和」 | 力量和敏捷 |
| 人物互動 → 可用「與」 | 與敵人交戰 |
| 條件並列 → 用「且」 | 有線索且沒有敵人 |
| **禁用** | 及、跟 |

### 7.3 「或」與「和」的互斥

- **或** → 玩家選擇其一(排他)
- **和** → 同時發生(並立)
- **和/或** → 選擇其一或兩者皆選(Arkham 式)

**範例**:
- 「治癒 1 點傷害或 1 點恐懼」(二擇一)
- 「治癒 1 點傷害和 1 點恐懼」(同時治兩種)
- 「治癒 1 點傷害和/或 1 點恐懼」(自選其一、或兩者皆治)

---

## 八、屬性修改句型

### 8.1 一次性修正

**標準格式**:
```
[時間範圍] [屬性] [+/−] N。
```

**範例**:
- 這次攻擊你 +1 力量。
- 本回合你的意志 +2。
- 下次技能檢定你 +3 智力。

### 8.2 持續性修正

**被動能力(無時間範圍)**:
```
你的 [屬性] +N。
```

**範例**:
- 你的力量 +1。
- 你的智力和感知 +1。
- 你的反應 +1。(v0.2 新增:閃避/中斷/應激類能力強化的標準寫法)

### 8.3 屬性替換

**標準格式**:
```
[動作] 中你不使用 [原屬性],改為使用 [新屬性]。
```

**範例**:
- 這次攻擊中你不使用力量,改為使用意志。
- 這次閃避中你不使用敏捷或體質,改為使用感知。

### 8.4 屬性相加

**標準格式**:
```
[動作] 中將你的 [屬性 A] 數值加入你的技能值。
```

**範例**:
- 這次檢定中將你的智力數值加入你的技能值。

區分三種機制:
- **加值**(+N):在原屬性值基礎上增加 N
- **替換**(不用 A 改用 B):用另一個屬性的值取代
- **相加**(A + B):兩個屬性值加總

---

## 九、使用次數操作句型

### 9.1 消耗次數

```
花費 N [使用次數名稱]:[效果]。
```

**範例**:
- 花費 1 彈藥:攻擊。
- 花費 2 充能:抽 1 張牌並獲得 1 資源。

### 9.2 補充次數

```
在此卡上放置 N [使用次數名稱]。
```

**範例**:
- 在此卡上放置 1 充能。
- 在本醫療包上放置 2 繃帶。

### 9.3 上限限制

```
在此卡上放置 N [使用次數名稱] (至多 M)。
```

**範例**:
- 在此卡上放置 1 證據 (至多 3)。

### 9.4 次數耗盡的自動處理

若此卡用完即離場,敘述為:
```
如果此卡沒有 [使用次數名稱],則棄掉此卡。
```

**範例**:
- 如果此卡沒有彈藥,則棄掉此卡。
- 如果此卡沒有充能,則將此卡移出遊戲。

## 一、標點符號規範

### 1.1 全形 vs 半形的使用分界

| 符號 | 全形 / 半形 | 使用情境 |
|---|---|---|
| 句號 | 全形 。 | 句末 |
| 逗號 | 全形 , | 句中 |
| 冒號 | 全形 : | 成本與效果的分隔符 |
| 分號 | 全形 ; | 並列的複雜子句分隔 |
| 頓號 | 全形 、 | 列舉項目分隔 |
| 問號 | 全形 ? | 疑問句(卡名或風味) |
| 驚嘆號 | 全形 ! | 強調(卡名或風味) |
| 括號(上限/補充) | **半形 ( )** | 使用次數、DC 值、補充說明 |
| 破折號 | 全形 —— | 多選一列舉、副標題分隔 |
| 引號 | 全形 「」 | 自訂術語、外部引用 |
| 直引號 | 半形 " " | 風味文字的角色台詞 |

### 1.2 括號的三種用途

| 用途 | 範例 |
|---|---|
| 使用次數聲明 | 使用 (4 彈藥) |
| 檢定 DC | 進行意志檢定 (3) |
| 上限補充 | 在此卡上放置 1 證據 (至多 3) |
| 規則補充說明 | (每次檢定限制 1 次) |

**括號使用規則**:
- 半形括號 `( )`,與內容間空一格
- 例外:`(N 資源類型)` 中數字與文字無空格

### 1.3 冒號的唯一用途

冒號**僅用於分隔成本與效果**,不用於其他語意:

- 正確:「花費 1 彈藥:攻擊。」
- 錯誤:「本次攻擊:+1 傷害。」(應寫成「本次攻擊 +1 傷害。」)

### 1.4 分號的使用時機

分號用於**並列的複雜子句**,避免連續逗號造成混亂:

**範例**:
- 「受到 1 點傷害,以治癒 3 點恐懼;或受到 1 點恐懼,以治癒 3 點傷害。」

若子句簡短,用逗號或句號即可,不需分號。

### 1.5 破折號的三種用途

| 用途 | 範例 |
|---|---|
| 多選一列舉 | 選擇一項 —— A、B、C |
| 副標題分隔 | 演武 —— 心神合一 |
| 規則豁免補充 | 強制 —— 在此卡離場時:... |

破折號一律使用**全形 em dash 兩個**(——),前後**不加空格**(除列舉格式外)。

### 1.6 引號的使用規則

**全形引號「」使用於**:
- 卡內自訂術語(如「座標點」、「符咒」)
- 引用其他卡片的能力代號(如「『行動』能力」)

**半形直引號 " " 使用於**:
- 風味文字中的角色台詞

**範例**:
- 卡內術語:「該地點為『座標點』,直到此卡離場為止。」
- 風味台詞:`"稍縱……"`

### 1.7 刪節號

使用**全形六點刪節號 ……**(不是三點 ... 也不是 . . .),表示:

- 語氣延續或中斷
- 列舉的省略

**範例**:
- 「稍縱……」
- 「你可以為所欲為……,只要你承擔後果。」

## 二、數字與單位規範

### 2.1 數字形式

- 一律使用**半形阿拉伯數字**(1, 2, 3)
- **禁用**中文數字(一、二、三)
- **禁用**全形阿拉伯數字(1、2、3)

**範例**:
- 正確:「造成 2 點傷害。」
- 錯誤:「造成兩點傷害。」、「造成 2 點傷害。」

### 2.2 單位的附加規則

| 情境 | 是否加「點」 | 範例 |
|---|---|---|
| 傷害 / 恐懼 | 加「點」 | 造成 2 點傷害 |
| 資源 | 不加 | 花費 1 資源 |
| 線索 | 不加 | 發現 1 線索 |
| 牌 | 加「張」 | 抽 1 張牌 |
| 地點 | 加「個」 | 移動 1 個地點 |
| 次數 | 加「次」 | 攻擊 2 次 |
| 標記 | 加「個」 | 放置 1 個毀滅標記 |
| 使用次數(自訂) | 不加 | 花費 1 彈藥、花費 2 充能 |
| 修正值 | 不加(+/− 符號表示) | +1 傷害、−2 難度 |

### 2.3 數字與符號的間距

- 修正值符號 **`+` / `−`** 與數字**不空格**:`+1`、`−2`
- 數字與後接詞**空一格**:`1 張牌`、`2 點傷害`
- 使用次數例外:`(4 彈藥)` 數字與詞之間空一格

### 2.4 減號的正確寫法

使用**全形減號「−」**(U+2212),**禁用**半形連字號「-」。

- 正確:「難度 −2」、「花費 −1 資源」
- 錯誤:「難度 -2」、「花費 -1 資源」

## 三、風味文字寫作原則

### 3.1 風味文字(`flavor_text`)的定位

風味文字是**非機制性的氛圍文字**,位於效果區下方。它的任務:

1. **強化主題感知** —— 讓卡片不只是機制
2. **延伸世界觀** —— 建構克蘇魯神話的質地
3. **呼應機制** —— 與能力文字形成意義互補

### 3.2 風味文字的五種合法類型

| 類型 | 說明 | 範例場合 |
|---|---|---|
| 景物描寫 | 環境、氛圍、物品外觀 | 儀式、道具、場景卡 |
| 旁白敘述 | 第三人稱的情境描述 | 調查線索、事件 |
| 角色台詞 | 人物直接引語(用半形直引號) | 盟友卡、技能卡 |
| 角色獨白 | 第一人稱的內心話 | 簽名卡、弱點卡 |
| 文學引用 | 引自虛構典籍的片段 | 書籍卡、遺跡卡 |

### 3.3 風味文字的長度

| 長度 | 適用 |
|---|---|
| 3–10 字 | 極簡氛圍(「稍縱……」) |
| 一句話 | 通用情境 |
| 兩句話 | 需要情境鋪陳時 |
| 三句以上 | **除書籍/遺跡外,禁用** |

### 3.4 風味文字的禁用事項

- **禁止重複機制文字** —— 風味不等於能力說明
- **禁止使用遊戲術語** —— 不寫「這張卡」、「觸發效果」
- **禁止破壞氛圍** —— 不使用網路流行語、現代俚語
- **禁止出戲玩笑** —— 克蘇魯主題不適合輕浮調性

### 3.5 文學引用的格式

引用**虛構典籍** 時,在引文後換行寫明來源:

```
"本書作者是路德維格·普林恩,他在巫術審判達到高潮時死於布魯塞爾的審訊樁上。"
—— 羅伯特·布洛克,《蹣跚者》
```

格式要點:
- 引文用半形直引號包起
- 作者與出處用破折號引入
- 出處若為書名,書名用全形《》包起

## 四、正反範例對照

### 4.1 主詞用詞

| 劣例 | 優例 | 說明 |
|---|---|---|
| 調查員獲得 1 資源 | 你獲得 1 資源 | 主詞用「你」 |
| 玩家可以選擇 | 你可以選擇 | 同上 |
| 我方的力量 +1 | 你的力量 +1 | 同上 |

### 4.2 效果動詞

| 劣例 | 優例 | 說明 |
|---|---|---|
| 打他 3 點傷害 | 造成 3 點傷害 | 使用 `deal_damage` 標準詞 |
| 扣掉 2 點血 | 造成 2 點傷害 | 同上 |
| 補血 1 點 | 治癒 1 點傷害 | 使用 `heal_hp` 標準詞 |
| 可以重新骰一次 | 重擲 1 次 | 使用 `reroll` 標準詞 |
| 讓他躺著 | 絆倒該敵人 | 使用 `stun_enemy` 標準詞 |

### 4.3 條件句

| 劣例 | 優例 | 說明 |
|---|---|---|
| 若汝有線索 | 如果你有線索 | 禁用「若」,統一用「如果」 |
| 有資源的話可以抽牌 | 如果你有至少 1 資源,抽 1 張牌 | 條件要明確 |
| 成功則治癒,否則受傷 | 如果成功,治癒 1 點傷害。如果失敗,受到 1 點傷害 | 禁用「否則」 |

### 4.4 數字與單位

| 劣例 | 優例 | 說明 |
|---|---|---|
| 造成兩點傷害 | 造成 2 點傷害 | 阿拉伯數字 |
| 造成 3 傷害 | 造成 3 點傷害 | 傷害/恐懼要加「點」 |
| 抽三張 | 抽 3 張牌 | 需明示「張」和「牌」 |
| -1 傷害 | −1 傷害 | 全形減號 |
| (1彈藥) | (1 彈藥) | 數字與詞要空格 |

### 4.5 選擇句

| 劣例 | 優例 | 說明 |
|---|---|---|
| 攻擊或者閃避 | 攻擊/閃避 | 簡單二擇用斜線 |
| 三選一:A、B、C | 選擇一項 —— A、B、C | 使用破折號列舉 |
| 二選二(從 A、B、C 中選二) | 選擇 2 項:◆ A ◆ B ◆ C | 項目符號列舉 |

### 4.6 自指

| 劣例 | 優例 | 說明 |
|---|---|---|
| 消耗急救箱並且對急救箱造成 1 點傷害 | 消耗此卡並對此卡造成 1 點傷害 | 使用「此卡」 |
| 本卡牌的使用次數 | 此卡的使用次數 | 統一用「此卡」 |
| 治癒急救箱上所有傷害(牌組中有多張急救箱時) | 治癒本急救箱的所有傷害 | 需區分同名卡時用「本 [卡名]」 |

### 4.7 取消 / 忽略 / 預防

| 劣例 | 優例 | 說明 |
|---|---|---|
| 你可以無視這次攻擊 | 取消該次攻擊 | Cancel 譯為「取消」 |
| 忽略該次傷害 | 預防該次傷害 | Prevent 譯為「預防」 |
| 取消藉機攻擊 | 忽略藉機攻擊 | Ignore 譯為「忽略」 |

### 4.8 時機詞

| 劣例 | 優例 | 說明 |
|---|---|---|
| 當你受傷害時 | 在你受到傷害時 | 統一用「在 X 時」格式 |
| 攻擊完畢後 | 在攻擊結算後 | 使用「結算」的標準結束時點 |
| 這場戰鬥 | 本回合 | 避免情境不明的用詞 |

## 六、AI Prompt 生成建議

### 6.1 Prompt 中應強制指定的規則

在 Gemini / AI 生成卡片時,系統 prompt 必須包含以下硬性約束:

1. **主詞一律使用「你」**
2. **效果動詞使用規則書第三章 §5.2 的唯一對應中文**(引用 Part 1 §5)
3. **數字一律半形阿拉伯數字,減號用全形「−」**
4. **句型結構依 Part 2 §1 的六要素順序**
5. **觸發時機使用中括號標示「【行動】【免費行動】【反應】【被動】【強制】」**
6. **取消/忽略/預防 三個動詞不得互換**
7. **禁用「若、否則、反之、及、跟」等非標準連接詞**

### 6.2 Prompt 結構建議

AI 生成卡片時,prompt 應依序提供:

```
1. 設計參數(類型、陣營、費用、等級、稀有度)
2. 效果意圖(設計者想做什麼)
3. 術語約束(Part 1 §3 術語表)
4. 句型模板(Part 2 §1 六要素)
5. 格式範例(Part 3 §5 完整卡片範例)
```

### 6.3 Prompt 驗證清單

AI 生成後,驗證輸出是否符合以下清單:

- [ ] 主詞統一為「你」
- [ ] 所有效果動詞使用標準中文
- [ ] 數字為半形、單位正確
- [ ] 標點符號符合 Part 3 §1
- [ ] 觸發類型用【】標示(【行動】/【免費行動】/【反應】/【被動】/【強制】)
- [ ] **三合一用途正確使用【加值】/【消費】標示,與行動類型並列而非衝突**(v0.2)
- [ ] **屬性名稱使用八屬性(含反應),不再使用七屬性表述**(v0.2)
- [ ] 冒號僅用於成本 / 效果分隔
- [ ] 自指使用「此卡」或「本 [卡名]」
- [ ] 連接詞只使用許可清單
- [ ] 條件句、選擇句符合 Part 2 模板
- [ ] 風味文字不重複機制、不破壞氛圍

### 6.4 驗證函式實作建議

在 MOD-01 卡片設計器的 `validateAndFixCardData` 函式中,新增:

1. **術語掃描器** —— 檢查敘述中是否出現禁用詞,發現即標紅提示
2. **句型檢查器** —— 檢查是否符合六要素順序
3. **格式檢查器** —— 檢查標點符號、數字格式、單位正確性
4. **一鍵修正** —— 對可自動修正的問題提供修正建議

## 七、既有卡片的批次修正方針

第四步驟執行時,依以下優先順序處理:

### 7.1 可自動替換的問題

| 問題類型 | 替換規則 | 風險 |
|---|---|---|
| 主詞錯誤 | 調查員/玩家/我 → 你 | 低 |
| 數字格式 | 中文數字 → 阿拉伯數字 | 低 |
| 半形減號 | - → − | 低 |
| 禁用連接詞 | 若 → 如果、及/跟 → 和 | 低 |
| 效果動詞非標準 | 使用術語表對應 | 中(需人工複核) |

### 7.2 需人工複核的問題

- 句型結構不符合六要素順序
- 條件句的軟硬性判斷(如果 / 只要 / 只有在)
- 選擇句的複雜度選擇
- 風味文字的風格判斷

### 7.3 批次修正的階段規劃

階段 1:術語替換(全自動,風險最低)
階段 2:格式標準化(全自動)
階段 3:句型重構(半自動 + 人工確認)
階段 4:風味文字打磨(純人工)
