## 第七部｜卡面資訊架構

### §7.1 資訊壓縮的設計哲學

#### 7.1.1 從 Netrunner 學到的五個手法

本草案的卡面架構借鑑 *Android: Netrunner* 的資訊壓縮智慧。該遊戲在有限卡面空間上承載了大量機制資訊，其核心手法有五：

**手法一：圖示取代名詞**
Credit、Click、Memory Unit 都用單一圖示表達，零字元成本。一個圖示 = 一個概念。

**手法二：固定句型結構**
所有能力都遵守 `[費用]: [效果]` 的格式——冒號前是費用、冒號後是效果。玩家學會句型後，任何新卡都能瞬間理解。

**手法三：類型標籤承擔背景規則**
「`PROGRAM: Icebreaker - Killer`」一行 7 字承載四個資訊：卡片類型、子類型、細分類、對應目標。類型本身就是規則，不需要在效果文字重複寫。

**手法四：強度與費用獨立角落**
強度寫在左下角、費用寫在左上角，獨立成框。玩家掃一眼角落就能評估，不必讀文字。

**手法五：限制條件獨立成行**
安裝限制、打出限制放在能力之外獨立一行，不打斷能力句型。

#### 7.1.2 本專案的對應策略

我們把這五個手法對應到本專案的既有體系：

| Netrunner 手法 | 本專案對應 | 實現方式 |
|---|---|---|
| 圖示取代名詞 | 屬性圖示系統（既有）| 延伸至資源類型、動作類型、充能類型 |
| 固定句型結構 | 《卡片敘述文法規範》（既有）| 擴充法器卡專用句型 |
| 類型標籤承擔規則 | 子類型系統（既有）| 新增法器類型行的標準格式 |
| 角落數值獨立 | 傷害、使用次數（既有）| 新增「破事強度」角落數值 |
| 限制條件獨立行 | 打出限制句（既有）| 沿用既有規範 |

### §7.2 法器卡的卡面結構

#### 7.2.1 卡面分區

法器卡的卡面分為以下區塊：

```
┌───────────────────────────────────────┐
│ [費用]              卡名               │   ← 上區：身份識別
│                                       │
│ [破事 N]                               │   ← 右上角：破事強度
│ [類型:類別]                            │   ← 左上輔助：適用遭遇卡類型
├───────────────────────────────────────┤
│                                       │
│            美術圖                      │   ← 中區：視覺敘事
│                                       │
├───────────────────────────────────────┤
│ 類型行（法器分類 + 子類型資訊）         │   ← 類型行：背景規則
├───────────────────────────────────────┤
│ 效果區（被動 / 主動 / 限制）            │   ← 效果區：機制文字
├───────────────────────────────────────┤
│ 風味文字（斜體）                       │   ← 風味區：世界觀
├───────────────────────────────────────┤
│ [傷害 N]                  [陣營圖示]   │   ← 下區：戰鬥資訊
└───────────────────────────────────────┘
```

#### 7.2.2 對角線數值的視覺語言

**左下角 = 殺敵軸（傷害）**
**右上角 = 破事軸（破事強度）**

這個對角設計讓玩家**一眼掃外形**就知道卡片類型：

| 卡片類型 | 左下角（傷害）| 右上角（破事強度）| 外型特徵 |
|---|---|---|---|
| 純武器 | 有數字 | 空白 | 只有左下有數字 |
| 純法器 | 空白 | 有數字 | 只有右上有數字 |
| 雙用途卡 | 有數字 | 有數字 | 對角線都有數字 |

**留白本身就是資訊**——玩家不用讀文字就能判斷這張卡是哪一型。

#### 7.2.3 法器類型行的標準格式

法器類型行用一行字承載四個資訊：

**純法器**：
```
法器: [遭遇卡類型] - [破除時機] ([充能名稱])
```

範例：
- `法器: 精神 - 即時 (神聖度)`
- `法器: 物質 - 檢定 (魅力)`
- `法器: 儀式 - 儲蓄 (預兆)`

**雙用途卡（武器 + 法器）**：
```
資產: 武器/法器 - [武器子類型]
[法器類型] - [破除時機] ([充能名稱])
```

範例：
- `資產: 武器/法器 - 隨身`
- `物質 - 即時 (純度)`

**特殊子類型雙用途（書籍/遺跡 + 法器）**：
```
法器/遺跡: [遭遇卡類型] - [破除時機] ([充能名稱])
```
或
```
法器/書籍: [遭遇卡類型] - [破除時機] ([充能名稱])
```

範例：
- `法器/遺跡: 精神 - 儲蓄 (預兆)`（預兆水晶）
- `法器/書籍: 儀式 - 儲蓄 (封印)`（古印之卷）

### §7.3 破除費用的標準效果句型

#### 7.3.1 三種時機的標準句型

**A 即時型**：
```
使用 (N [充能名稱])。
[行動] 花費 X [充能名稱]: 對應一張強度 X 的 [類型] 類遭遇卡，立即破除其中 X + Y 段效果（Y = 額外花費的充能）。
```

**範例（銀製護身符）**：
```
使用 (5 神聖度)。
[行動] 花費 X 神聖度: 對應一張強度 X 的精神侵蝕類遭遇卡，
立即破除其中 X 段效果。每額外花 1 神聖度，多破 1 段。
```

**B 檢定型**：
```
使用 (N [充能名稱])。
[行動] 花費 1 [充能名稱]: 對應一張 [類型] 類遭遇卡，
進行 [屬性] 檢定 (DC 為其強度)。成功則完全破除；
失敗則此卡照常觸發。
```

**範例（警徽）**：
```
使用 (3 共鳴)。
[行動] 花費 1 共鳴: 對應一張精神侵蝕類遭遇卡，
進行魅力檢定 (DC 為其強度)。成功則完全破除；
失敗則此卡照常觸發。
自然 20: 額外獲得 1 共鳴。
自然 1: 失去 2 共鳴並受 1 SAN 反噬。
```

**C 儲蓄型**：
```
在此卡上放置 1 [計量名稱] (至多 M)，於每 [累積觸發條件]。
[行動] 花費 X [計量名稱]: 對應一張強度 X 的 [類型] 類遭遇卡，立即破除。
```

**範例（預兆水晶）**：
```
在此卡上放置 1 預兆 (至多 6)，於每回合開始時。
[行動] 花費 X 預兆: 對應一張強度 X 的精神侵蝕類遭遇卡，
立即破除。
[免費行動] 花費 1 預兆: 觀看混沌袋頂 1 顆標記。
```

#### 7.3.2 雙用途卡的效果區順序

雙用途卡的效果區應該分兩段：武器面在前、法器面在後。中間用獨立一行分隔。

**範例（桃木劍）**：
```
使用 (6 木質)。

[攻擊] 對敵人造成 2 點傷害。

[行動] 花費 X 木質: 對應一張強度 X 的儀式詛咒類遭遇卡，
立即破除。

每次使用法器能力後，此卡傷害永久 -1。
```

這個順序符合玩家的閱讀習慣——先理解「這是一把武器」，再理解「它還能破除遭遇卡」。

### §7.6 卡面視覺識別的陣營延伸

除了角落數值的對角線設計，卡面可以進一步用**視覺元素**強化陣營與法器類型的識別：

#### 7.6.1 陣營色帶（既有系統延伸）

各陣營已有既定主題色，建議在法器卡的邊框或裝飾元素中體現：
- E 號令：金色 / 琥珀
- I 深淵：深靛藍
- S 鐵證：鏽銅
- N 天啟：紫羅蘭
- T 解析：冷鋼藍
- F 聖燼：暖紅
- J 鐵壁：石墨灰
- P 流影：翠綠

#### 7.6.2 法器類型標記（新增系統）

每種法器類型對應一個小圖示，放在類型行旁或卡面右上角：
- 精神侵蝕 → 漩渦 / 腦狀圖示（象徵心智）
- 物質異變 → 裂痕 / 破碎圖示（象徵物質）
- 儀式詛咒 → 符印 / 五芒星圖示（象徵儀式）

#### 7.6.3 破除時機標記（新增系統）

即時、檢定、儲蓄三種時機對應三個小圖示：
- 即時 → 閃電 / 瞬時圖示
- 檢定 → 骰子圖示
- 儲蓄 → 沙漏 / 容器圖示

這些視覺元素讓玩家在**不讀文字的情況下**就能判斷卡片的核心屬性。

## 第八部｜給創作者的設計指引

### §8.1 設計新法器的決策樹

創作者在 MOD-01 設計一張新法器時，應按以下順序做決策：

#### 8.1.1 第一層決策：兩軸比重

- **純法器**（只有破事軸）→ 簡單、專注、容易平衡
- **雙用途卡**（武器 + 法器）→ 複雜、彈性、V 值疊加
- **三合一**（武器 + 法器 + 遺跡/書籍）→ 極高複雜度、通常為簽名卡或傳奇稀有度

#### 8.1.2 第二層決策：破除時機

- **即時**（A）→ 確定性、容量有限 → 適合 S、J、F 陣營
- **檢定**（B）→ 主屬性依賴、無限嘗試 → 適合 E、T、F 陣營
- **儲蓄**（C）→ 累積型、時機性 → 適合 N、I、P 陣營

#### 8.1.3 第三層決策：適用遭遇卡類型

- **專一類型**（僅 Mental 或 Physical 或 Ritual）→ 標準設計
- **雙類型**（例如 Mental + Physical）→ 進階或稀有以上稀有度
- **全類型**（通用法器）→ 傳奇稀有度，費用極高

#### 8.1.4 第四層決策：軸向歸屬

- **陣營極軸**——這張卡是否指認某個陣營？（強度低，風味為主）
- **戰鬥風格軸**——這張卡是否同時是武器？（指認戰鬥風格）
- **戰鬥專精軸**——這張卡是否強化某種專精路線？
- **卡名軸**——這張卡是否屬於某個卡名家族？
- **法器類型軸**——這張卡的物質載體是什麼（桃木、銀製、符文、聖物）？

#### 8.1.5 第五層決策：特殊代價

- 無代價（即時型預設）
- 檢定失敗代價（擲骰型預設）
- 累積時間代價（儲蓄型預設）
- **卡片專屬代價**（選填）：
  - 每次使用 -1 武器傷害（桃木劍型）
  - 失敗累積裂痕、達上限毀壞（解咒儀刀型）
  - 大失敗觸發反噬（警徽型）
  - 一次性消耗（鹽圈瓶型）
  - 禁用某類行動（設計師自創）

### §8.2 設計新遭遇卡的決策樹

創作者在 MOD-06 或 MOD-07（場景編輯器）設計新遭遇卡時：

#### 8.2.1 第一層決策：類型

- **精神侵蝕**（Mental）→ 影響 SAN、手牌、神啟
- **物質異變**（Physical）→ 影響 HP、地點、物品
- **儀式詛咒**（Ritual）→ 影響詛咒標記、議程、長期效果

#### 8.2.2 第二層決策：強度

參考本草案 §4.6.2 的 DV 對照表：
- 章節 1-3：強度 1-2
- 章節 4-6：強度 3-4
- 章節 7-9：強度 5-6
- 最終章節：強度 7+

#### 8.2.3 第三層決策：子程式結構

參考本草案 §4.5.2：
- 強度 1-2：1 條子程式
- 強度 3-4：2 條子程式（主效果 + 副效果）
- 強度 5-6：3 條子程式（立即傷害 + 持續影響 + 敘事後果）
- 強度 7+：4 條子程式（災難級連鎖）

#### 8.2.4 第四層決策：不破除後果

不破除後果應與強度線性對應（紅線內）：
- 低強度：資源損失、小量 HP/SAN 損失
- 中強度：中量 HP/SAN、棄手牌、放置標記
- 高強度：大量 HP/SAN、地點變異、長期負面效果
- 極高強度：創傷上限 -1、神啟洗入、議程加速

**禁止違反的紅線**（§4.6.2）：
- 低強度卡不得造成角色死亡或創傷
- 高強度卡必須在議程推進後才能出現
- 單卡不得同時造成 HP 與 SAN 歸零

#### 8.2.5 第五層決策：軸向歸屬

- **陣營極軸**——這張遭遇卡屬於哪個派系？（邪教徒派、大學派、深潛者派）
- **家族軸**——這張卡屬於哪個家族？（印斯茅斯、敦威治、阿卡姆）
- **類型軸**——本章節的三種類型之一

### §8.3 變奏範例

給創作者三個「從九宮格範本變奏」的具體範例，展示設計空間：

#### 8.3.1 變奏範例一：從⑦預兆水晶出發

**原卡**：預兆水晶（N/I 陣營、儲蓄、精神類）

**變奏方向 A（改陣營氣質）**：
- 新卡：「禁忌筆記」（I 陣營專屬，儲蓄型，精神類）
- 累積條件改：每次研究遺跡 +1 洞察（不是每回合）
- 效果文字改：「花費 X 洞察：對應一張強度 X 的精神侵蝕類遭遇卡，立即破除。」
- 風味：「每一頁翻過的禁忌都讓你更接近真相，也更接近崩潰。」

**變奏方向 B（改類型）**：
- 新卡：「夢境殘片」（N 陣營，儲蓄型，儀式類）
- 類型改為儀式詛咒
- 風味：「你在夢中見過這個儀式。醒來時，你已經知道該怎麼阻止它。」

**變奏方向 C（改代價結構）**：
- 新卡：「血色水晶」（N 陣營，儲蓄型，精神類）
- 累積改為：每次受 1 SAN 傷害時 +1 預兆
- 設計意圖：讓「受傷」反而變成累積資源的方式——克蘇魯風味的扭曲成長

#### 8.3.2 變奏範例二：從③桃木劍出發

**原卡**：桃木劍（S/J/F 陣營、即時、儀式類、雙用途）

**變奏方向 A（改法器類型）**：
- 新卡：「銀製匕首」（S 陣營，即時型，物質類，雙用途）
- 同為隨身武器雙用途，但類型改為銀製、破物質異變類
- 效果文字結構完全相同，只改類型與風味

**變奏方向 B（改武器類別）**：
- 新卡：「桃木弓」（P 陣營，即時型，儀式類，雙用途）
- 從 sidearm 改為 archery，成為弓術雙用途
- 設計意圖：「桃木」軸跨多種戰鬥風格

**變奏方向 C（改代價結構）**：
- 新卡：「桃木印章」（S 陣營，即時型，儀式類，純法器）
- 去掉武器面，變成純法器
- 代價改為：每次破除後，將 1 個封印放在此卡上，封印達 3 時從遊戲中移除
- 設計意圖：純法器版本、更高破事強度、但有使用次數限制

#### 8.3.3 變奏範例三：填補九宮格空位

九宮格中某些「陣營 × 時機 × 類型」的組合目前沒有範本。創作者可以主動填補：

**新卡：「社交名片」**（E 陣營，檢定型，儀式類，純法器）
- 位置：九宮格中「檢定 × 儀式」位置（原為⑥解咒儀刀）
- 差異：⑥是 T/F 陣營的力量/敏捷檢定，本卡是 E 陣營的魅力檢定
- 效果：「花費 1 信譽：對應一張儀式詛咒類遭遇卡，進行魅力檢定。成功則說服儀式主持者中止儀式，完全破除遭遇卡。」
- 風味：「不是所有邪教徒都堅定。有時候，一張名片就足以讓他們重新考慮。」

這類變奏**填補了陣營覆蓋的空白**，讓 9 個位置變成 27 個位置（9 × 3 陣營主要對應）。

### §8.4 常見錯誤與避免方式

#### 8.4.1 錯誤一：軸向強度與廣度不匹配

**錯誤範例**：
```
[陣營極軸卡，低費用]
每次你打出一張深淵卡時，+2 傷害。
```
這張卡把強效果（+2 傷害）放在最廣的軸向（陣營極）上，違反強度與廣度反比合約。

**修正方向**：
- 改為陣營極軸 + 弱效果：「+1 傷害」
- 或改為卡名軸 + 強效果：「每次你打出一張『禁忌契約』卡時，+2 傷害」

#### 8.4.2 錯誤二：雙用途卡兩軸都達到極致

**錯誤範例**：
```
[雙用途卡，進階稀有度，費用 3]
傷害 5。充能 5 神聖度。
破事強度 5。
```
這張卡在兩軸都達到進階級極致，違反「雙用途卡總 V ≤ 1.5 倍單軸極致」合約。

**修正方向**：
- 降低一軸極致：「傷害 3、破事強度 5」或「傷害 5、破事強度 3」
- 或提高稀有度與費用：改為傳奇稀有度、費用 6

#### 8.4.3 錯誤三：遭遇卡無解

**錯誤範例**：
```
[強度 9 的精神侵蝕類遭遇卡]
強度 9、4 子程式
```
當前 9 張法器範本中，沒有任何一張能破強度 9 的遭遇卡（⑦預兆水晶上限 6、其他類似）。這張遭遇卡無解，違反紅線一。

**修正方向**：
- 降低強度至 7-8
- 或設計配套的「大型法器」（傳奇稀有度、破事強度 9+）

#### 8.4.4 錯誤四：強度與後果脫鉤

**錯誤範例**：
```
[強度 2 的儀式詛咒類遭遇卡]
強度 2、1 子程式：議程牌堆放置 3 毀滅標記
```
強度 2 的遭遇卡後果卻是「3 毀滅標記」（通常對應強度 5+ 的後果），違反設計合約。

**修正方向**：
- 降低後果：改為「1 毀滅標記」
- 或提高強度至 5 以上
- 或加入「特殊額外條件」（例如「只有在玩家有 2+ 毀滅標記時才能抽到」），合法化強度與後果的脫鉤

#### 8.4.5 錯誤五：文字冗長違反精簡規範

**錯誤範例**：
```
[某法器卡，效果文字 200+ 字元]
這張卡是一張可以對抗精神侵蝕類遭遇卡的法器。當你撞到這類
遭遇卡時，你可以宣告使用此卡的破除能力，然後消耗 X 神聖度
（其中 X 是遭遇卡的強度），這樣就可以破除該遭遇卡的一段效果。
如果你想破除更多段，你可以每多花 1 神聖度來破除多 1 段效果……
```

**修正方向**：
```
使用 (5 神聖度)。
[行動] 花費 X 神聖度: 對應一張強度 X 的精神侵蝕類遭遇卡，
立即破除其中 X 段效果。每額外花 1 神聖度，多破 1 段。
```

精簡原則：類型行承擔規則、句型承擔結構、角落承擔數值——三者分工，文字極簡。

### §8.5 創作者自檢清單

每張新法器卡或遭遇卡完成設計後，建議自檢以下項目：

#### 8.5.1 法器卡自檢清單

- [ ] 類型行格式正確（法器: 類型 - 時機 (充能名稱)）
- [ ] 右上角標示「破事 N」或「破事:檢定」或「破事:儲蓄」
- [ ] 左下角標示「傷害 N」（雙用途卡）或空白（純法器）
- [ ] 效果文字符合該時機的標準句型
- [ ] 總 V 值計算正確、費用落在 0-6 範圍
- [ ] 軸向歸屬明確（陣營、戰鬥風格、卡名、法器類型）
- [ ] 卡名符合 2-4 字原則（除專有名詞）
- [ ] 風味文字符合克蘇魯氛圍
- [ ] 不違反強度與廣度反比合約
- [ ] 不違反雙用途卡總 V 上限合約

#### 8.5.2 遭遇卡自檢清單

- [ ] 類型明確（精神侵蝕 / 物質異變 / 儀式詛咒）
- [ ] 強度與子程式數對應（§4.5.2）
- [ ] 強度與後果線性對應（或有合法的非線性條件）
- [ ] 不違反紅線四條（§4.6.2）
- [ ] 至少存在一種法器組合可破除
- [ ] 不破除路線可承受（不會直接導致角色死亡）
- [ ] 所屬章節的 DV 平均合理
- [ ] 軸向歸屬明確（陣營、家族、類型）
- [ ] 敘事與克蘇魯世界觀咬合

### §9.1 MOD-01 卡片設計器的擴充

#### 9.1.1 法器卡編輯介面的新增欄位

現有 MOD-01 卡片設計器需新增以下欄位，以支援法器卡設計：

**基本欄位**：
- `is_talisman` (BOOLEAN) — 是否為法器
- `talisman_type` (VARCHAR) — 法器類型（桃木、銀製、符文、聖物、水晶、骨製、鹽製等，可擴充）
- `target_encounter_types` (JSONB) — 適用遭遇卡類型陣列（mental / physical / ritual）
- `break_timing` (ENUM) — 破除時機（instant / test / stockpile）
- `break_strength_max` (INTEGER) — 破事強度上限（即時型與儲蓄型適用）
- `break_charge_label` (VARCHAR) — 充能顯示名稱（神聖度 / 共鳴 / 預兆等）
- `break_charge_max` (INTEGER) — 充能上限
- `break_test_attribute` (VARCHAR) — 檢定屬性（檢定型適用）
- `stockpile_accumulation_rule` (TEXT) — 儲蓄累積規則（儲蓄型適用）

**V 值相關欄位**：
- `break_axis_value` (DECIMAL) — 破事軸 V 值（系統計算）
- `kill_axis_value` (DECIMAL) — 殺敵軸 V 值（系統計算，雙用途卡才有）
- `leverage_modifier` (DECIMAL) — 槓桿修正值（雙用途卡才有）

#### 9.1.2 法器卡編輯 UI 建議

MOD-01 的卡片設計介面，法器卡部分建議分為三個頁籤：

**頁籤一：基本資訊**
- 卡名、費用、陣營、稀有度
- 美術圖上傳區
- 風味文字

**頁籤二：破事軸設定**
- 破事時機（下拉選單：即時 / 檢定 / 儲蓄）
- 適用類型（多選：精神 / 物質 / 儀式）
- 破事強度上限
- 充能類型名稱
- 充能上限
- **過路費函數預覽**（系統自動生成）

**頁籤三：殺敵軸設定**（僅雙用途卡）
- 傷害值
- 戰鬥風格
- 屬性修正
- 使用次數（若適用）

**頁籤四：V 值計算與驗證**
- 破事軸 V 值（系統計算）
- 殺敵軸 V 值（系統計算）
- 槓桿修正（系統計算）
- 總 V 值
- 稀有度反推
- 費用建議
- **違反合約警告**（若有）

#### 9.1.3 過路費函數自動預覽

創作者填完破事軸設定後，系統應自動生成過路費函數對照表，供創作者預覽：

```
過路費函數預覽：f(S, N) = ⌈S / 2⌉ + N

對抗範例：
- 對強度 2、1 子程式的遭遇卡：2 神聖度
- 對強度 4、2 子程式的遭遇卡：4 神聖度
- 對強度 6、3 子程式的遭遇卡：6 神聖度
- 對強度 8、4 子程式的遭遇卡：8 神聖度（超過上限 5，無法破除）

可破除範圍：強度 0 ~ 5 的指定類型遭遇卡
```

這個預覽讓創作者立刻看到「這張卡在什麼情境下有用、什麼情境下無用」。

#### 9.1.4 軸向指認句型範本下拉選單

卡片效果文字編輯區旁，建議提供一個「**軸向指認句型範本**」的下拉選單，讓創作者一鍵插入標準句型：

範本列表：
- 在你打出另一張 [陣營] 卡時，...
- 你打出的 [戰鬥風格] 卡費用 -1 資源。
- 只要你場上有 N 張 『[卡名字串]』 卡，...
- 只有在你有 [法器類型] 卡在場上時，才能 ...

創作者選擇範本後，系統插入標準格式，創作者只需填入變數。

### §9.2 MOD-06 / MOD-07 場景編輯器的擴充

> **給未來 Claude 實例的備註｜Note to Future Claude Instances**
>
> 本草案是針對「武器同質化問題」的系統性解法，核心設計哲學有以下幾點，未來修訂時應嚴格遵守：
>
> 1. **軸向系統不新增 schema 欄位** — 軸向只是對既有資訊的指認用法，不是新分類
> 2. **V 值是過路費函數，不是單一數字** — 玩家視角評估法器，而非設計者視角
> 3. **遭遇卡是城主端工具，保留設計自由度** — 不把玩家側的線性合約強加其上
> 4. **雙用途卡的槓桿修正保守** — +1 至 +2V，不能讓雙用途卡碾壓單用途卡
> 5. **九宮格是設計起點，不是設計天花板** — 創作者應變奏與擴展，而非被框限
> 6. **強度與廣度反比合約是隱性的** — 不寫硬性程式檢查，靠創作者腦中校準
> 7. **卡面極簡 — 類型行承擔規則、句型承擔結構、角落承擔數值**
> 8. **所有類型的遭遇卡必須有通用解法** — 不允許「某陣營專屬無解」的設計
> 9. **三種破除時機對應陣營氣質** — 這是陣營風味差異化的具體機制
> 10. **本草案屬加法設計** — 不追溯既有卡片，舊卡繼續運作、新空間被打開
