## Card Upgrade System Refactor v1 · Part 1: Core Mechanics

- **版本**：v1.0
- **日期**：2026-04-25

- **權威順序**：本規範 > 規則書 v1.0 第三章 §8 > 卡片價值計算規範 v1.1 > 規則書 v1.0 第四章 §1.1 第一條

## 一、本規範解決的問題

舊系統將「卡片升級路徑」設計為**設計師預先決定 LV0–LV5 各等級長什麼樣**，導致兩個問題：

1. 設計師工作量過重（一張卡 = 設計六個版本）
2. 玩家對卡片成長路徑無控制權，設計師決定的升級方向可能讓原本好用的卡變爛

本規範將升級系統重構為「**玩家自主投資成長配額**」模式：每張卡有 5 點成長空間，由設計師起始投入與玩家後續投入共同填滿。

---

## 二、核心概念定義

### 2.1 成長空間配額

| 概念 | 定義 |
|---|---|
| 成長空間配額 | 每張卡片固定 5 點，是該卡從基礎到滿配額的總投入上限 |
| 設計師起始投入 | 設計師在卡片設計時決定的初始投入點數，範圍 0–5，每點對應 1V 效果預算 |
| 玩家後續投入 | 玩家在整備期透過花費 XP 購買強化選項，累積投入到該卡上的點數 |
| 配額守恆 | 設計師起始投入 + 玩家後續投入 ≤ 5 |

### 2.2 兌換軌道

> **1 點投入 = 1V 效果預算**（無論是設計師預付或玩家投資）

設計師起始投入與玩家後續投入走同一條兌換軌道，匯率一致。

### 2.3 用詞決策（請 Claude Code 與 Uria 確認）

新系統下原 `level` 欄位的語意已改變，請與 Uria 確認以下三個候選方案：

| 候選 | 欄位名稱 | 卡面顯示 | 規則書用詞 | 風險 |
|---|---|---|---|---|
| A | 保留 `level` | LV0–LV5 改 ★0–★5 | 起始投入點數 | 欄位名與語意不一致，未來協作者會誤解 |
| B | 改名 `starting_xp` | ★0–★5 | 起始投入點數 | 需 schema migration，欄位語意最直白 |
| C | 改名 `init_points` | ★0–★5 | 起始投入點數 | 需 schema migration，名稱中性 |

本規範以下文字統一以「起始投入點數」為規則書用詞，欄位名稱保留 `level` 直到 Uria 確認後再做替換。

### 2.4 卡片庫（概念草稿）

> **卡片庫**是玩家在冒險中累積的、所有曾經屬於他的卡片總集合。整備期玩家可以從卡片庫中調整下一關的牌組組成。

本次規範**只釘下這個概念存在**。卡片庫的具體容器規格、UI 呈現、資料結構由後續迭代定義。Claude Code 在本次實作中，**不需要建立卡片庫資料表**，僅需在文件層面登記此概念已被引入。

## 三、設計師端：起始投入機制

### 3.1 起始投入點數欄位

| 規格 | 內容 |
|---|---|
| 欄位語意 | 設計師決定這張卡出廠時已預先投入幾點成長空間 |
| 範圍 | 0–5 整數 |
| 預設值 | 0（純基本卡，玩家可投滿 5 點） |
| 對 V 值的貢獻 | 起始投入點數 × 1V，計入「起始投入抵扣」（見 Part 2） |

### 3.2 設計師可在卡上開放的強化欄位（菜單）

設計師在卡片設計時，於卡面的可強化欄位旁掛上「**強化標記**」，表達該欄位開放玩家用 XP 購買強化。

| 規格 | 內容 |
|---|---|
| 標記方式 | 卡面欄位旁附加可選的強化標籤（資料結構詳見 Part 3） |
| 設計師決策 | 哪些欄位開放、各欄位的強化幅度上限 |
| 售價計算 | **本次規範不定義售價推算規則**，售價計算公式為待確認項目 |
| 範例 | 武器卡的「攻擊力」欄位可掛強化標記、「使用次數」欄位可掛強化標記 |

### 3.3 設計師端 LV0–LV5 六頁籤的廢除

舊 MOD-01 卡片設計器的「LV0–LV5 六個獨立頁籤」整個拆除。具體拆除指令見 Part 3。

## 四、玩家端：強化購買機制

### 4.1 玩家投入 XP 的兩個用途

| 用途 | 時機 | 對象 | 收費依據 |
|---|---|---|---|
| 購買卡片 | 整備期，從卡片庫加入牌組 | 起始投入點數 > 0 的非獨特卡 | 起始投入點數 × 1 XP（標準）或 ×2 XP（Exceptional） |
| 購買強化 | 整備期，對牌組內已有卡片投入 XP | 該卡開放的強化標記 | 待後續訂定 |

### 4.2 「購買卡片」與「購買強化」的範圍邊界

| 動作 | 收費 | 受天賦鎖 | 受 5 點配額限制 |
|---|---|---|---|
| 冒險中獲得卡片 | 不收費 | 探索獎勵卡受鎖；劇情固定卡不受鎖 | 不適用（卡片直接進牌組或場上） |
| 整備期購買卡片副本 | 收費 | 受鎖 | 該副本帶有設計師起始投入點數，玩家可在剩餘配額內再投資 |
| 整備期購買強化 | 收費（待訂定） | 不受鎖（強化過程不受天賦鎖） | 受 5 點配額限制 |

### 4.3 整備期購買的 XP 公式

> **購買一張卡片副本的 XP 成本 = 該卡起始投入點數 × Exceptional 倍率**

| Exceptional 標記 | 倍率 |
|---|---|
| 無 | ×1 |
| 有 | ×2 |

範例：
- 起始投入 0 點的純基本卡 → 0 XP（無償）
- 起始投入 3 點的標準卡 → 3 XP
- 起始投入 5 點的滿配額卡 → 5 XP
- 起始投入 3 點且 Exceptional 的強卡 → 6 XP

### 4.4 重複購買的處理

> 整備期再購買同名卡的另一張，**每張副本獨立計費**，與該玩家先前是否擁有過此卡無關。

獨特卡（`is_unique = true`）不可購買第二張，無論玩家是否曾擁有過。

## 五、Exceptional 關鍵字

### 5.1 雙重特性

Exceptional 是卡片屬性層級的關鍵字，標記後同時觸發兩個獨立特性：

| 特性 | 規格 |
|---|---|
| 機制面 | 玩家投入 XP 時成本 ×2（包含購買卡片入場費與後續強化購買） |
| 費用面 | V 值費用公式中額外抵扣 -2V（沿用 v1.1 §3.5 的歸納值，不變） |

### 5.2 Exceptional 卡片的卡面顯示

依卡片敘述文法規範 Part 1 §8.3 的關鍵字格式，Exceptional 標記顯示於效果區最上方。中文卡面寫法請與 Uria 確認（候選：「卓越」「Exceptional」「特例」）。

### 5.3 Exceptional 不適用的卡片類型

| 卡片類型 | Exceptional 是否適用 |
|---|---|
| 資產卡（含書籍、遺跡的本體） | 適用 |
| 事件卡 | 適用 |
| 盟友卡 | 適用 |
| 技能卡 | 適用 |
| 升級版卡片（書籍/遺跡 transform 後產生的卡） | **不適用**，原因見第七節 |

## 六、天賦等級鎖

### 6.1 鎖的範圍

> **天賦鎖只管獲取資格，不管強化過程。**

| 範圍 | 受天賦鎖限制 |
|---|---|
| 玩家可獲取的該陣營卡片 | ✓（見 6.2 對照表） |
| 探索中撿起該陣營卡片 | ✓ |
| 整備期購買該陣營卡片副本 | ✓ |
| 對已在牌組中的卡片投入 XP 強化 | **✗（不受鎖）** |
| 劇情固定發放的該陣營卡片 | **✗（不受鎖）** |

### 6.2 對照表（每 2 級解鎖 1 點起始投入額度）

| 該陣營天賦等級 | 玩家可獲取的該陣營卡，其起始投入點數上限 |
|---|---|
| 0（未投資） | 不可獲取該陣營卡片 |
| 1 | 起始投入 0 點 |
| 2–3 | 起始投入 0–1 點 |
| 4–5 | 起始投入 0–2 點 |
| 6–7 | 起始投入 0–3 點 |
| 8–9 | 起始投入 0–4 點 |
| 10–12 | 起始投入 0–5 點 |

### 6.3 中立卡的鎖

> 中立卡上限取調查員所有已投資陣營中**最高**的天賦等級，依上表對應。

### 6.4 與支柱五 §2.4 的關係

本對照表**取代**支柱五 v0.1 §2.4 的舊對照表（每 3 級解鎖一級的版本）。Part 3 列出具體文字替換指令。

## 七、書籍卡與遺跡卡的特殊處理

### 7.1 transform 系統獨立、不收編

書籍卡與遺跡卡透過 `transform` 動詞完成「研究 → 升級版卡片」的軸線，**獨立於本次規範定義的「起始投入 + 玩家強化」軸線**。兩條軸不互相覆蓋。

### 7.2 升級版卡片的特殊規格

| 屬性 | 規格 |
|---|---|
| 起始投入點數 | 5（全配額預付） |
| Exceptional 標記 | **不適用**（升級版不能被玩家強化，倍率機制無意義） |
| 玩家後續強化 | **不可** |
| 費用 | 0（沿用 v1.1 §4.6 規定，研究投入已是隱性成本） |
| 取得管道 | 唯一管道為 transform 觸發，**不可由整備期購買** |

### 7.3 升級版的 UI 反向關聯

升級版卡片在 MOD-01 列表中需明確標示「**此卡為書籍/遺跡升級版，由 [原書名] 研究完成後 transform 取得，不可獨立購買**」。

## 八、卡片獲取的三條路徑（語意校準）

### 8.1 對應到 `deck_cards.acquired_method` 既有欄位

| acquired_method | 說明 | 收費 | 受天賦鎖 |
|---|---|---|---|
| `starting` | 創角時的起始牌組成員 | 無償 | 不適用（創角階段） |
| `exploration` | 探索場景中撿到的隨機卡 | 無償 | **受鎖** |
| `reward` | 劇情固定發放或關卡獎勵 | 無償 | **不受鎖**（劇情固定卡） |
| `craft` | 製作系統產出 | 走製作系統規格 | 走製作系統規格 |
| `trade` | 交易（待設計） | 待設計 | 待設計 |
| `purchase` | 整備期從卡片庫購買的副本（**新增值**） | 收費（XP = 起始投入 × Exceptional 倍率） | **受鎖** |

### 8.2 三條路徑的職責分工

| 路徑 | 職責 | 對應 acquired_method |
|---|---|---|
| 劇情固定發放 | 劇情驅動，發放敘事核心物件（如死靈之書） | `reward`（須額外標記為劇情固定，見 Part 3） |
| 探索隨機獲得 | 探索驅動，玩家主動搜索場景的回報 | `exploration` |
| 整備期購買 | 構築驅動，玩家在卡片庫中花 XP 購入副本 | `purchase` |

### 8.3 兩個既有旗標的協同

| 旗標 | 控制 |
|---|---|
| `is_unique` | 是否可擁有多份。獨特卡同名最多一張，非獨特卡可有多份 |
| `is_temporary` | 冒險結束後是否消失。臨時卡冒險結束消失，永久卡進入卡片庫 |

兩旗標自由組合，涵蓋四種發放情境，本次規範不變動其語意。

## 九、設計紅線

以下原則為本次重構的紅線，Claude Code 實作時不得違反：

1. **5 點配額硬上限**：任何卡片的「設計師起始投入 + 玩家後續投入」不得超過 5 點。
2. **天賦鎖不管強化過程**：玩家對已在牌組中的卡片投入 XP 強化時，不檢查天賦等級。
3. **Exceptional 倍率覆蓋整條 XP 軌道**：購買卡片入場費和後續強化購買都 ×2，不可只在其中一處生效。
4. **書籍/遺跡升級版獨立**：不收編進新系統的配額體系，不可被購買、不可被強化、費用恆為 0。
5. **獨特卡單張上限**：`is_unique = true` 的卡片不可購買第二張，整備期 UI 必須擋下。
6. **冒險中獲得不收費**：探索獲得、劇情固定發放的卡片，無論起始投入點數多少，玩家都不需付 XP 入手。
7. **舊資料相容**：舊版 LV0–LV5 各等級的卡片資料在遷移時，視為「設計師起始投入 = 該卡舊等級」，玩家後續配額 = 5 - 該值。

## 十、與既有規則的衝突解決

### 10.1 等級抵扣階梯

本規範的「起始投入抵扣 = 起始投入點數 × 1V」**取代以下三處的舊版階梯**：

| 來源 | 舊版階梯 |
|---|---|
| 規則書 v1.0 第三章 §8.5 | LV1 -1V 至 LV5 -5V |
| 卡片價值計算規範 v1.1 §6 | LV1 -1V 至 LV5 -5V（標記為「不變」） |
| 規則書索引 v01 §4 覆蓋規則 | LV1 -0.5V 至 LV5 -4V |

新系統下統一為線性 1:1 對應，廢除非線性階梯。

### 10.2 支柱五 §5.2 卡片數值強化費用表

支柱五 v0.1 §5.2 的「武器升 1 級 = 目標等級 × 3 XP」「盟友升 1 級 = 目標等級 × 2 XP」等費用公式**全部廢除**。新系統下玩家投入 XP 的成本由本規範第四節定義（購買卡片副本）和待後續訂定（購買強化菜單）兩處計算。

### 10.3 規則書第四章 §1.1 第一條

第四章 §1.1 五條成長路徑表的第一條「卡片升級｜經驗值｜升級版卡片（LV0→LV5）」需重寫為：「**卡片升級｜經驗值｜起始投入 + 玩家強化（5 點配額）**」。具體文字替換在 Part 3。

### 10.4 規則書第二章 §1.6 整備模式

整備模式中「花費**經驗值** → 購買升級版卡片」這條動作的語意擴大為「**花費經驗值 → 購買卡片副本或對牌組內卡片購買強化**」，書籍/遺跡的升級版維持原 transform 路徑。具體文字替換在 Part 3。
