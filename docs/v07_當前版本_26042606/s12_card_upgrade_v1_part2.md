## 一、本篇對 V 值體系的調整總覽

| 項目 | 舊狀態 | 新狀態 |
|---|---|---|
| 等級抵扣階梯 | LV1 -0.5V 至 LV5 -4V（非線性） | **起始投入抵扣 = 起始投入點數 × 1V**（線性） |
| 等級抵扣的對象 | 設計師指定的 LV0–LV5 | 設計師起始投入點數（0–5） |
| Exceptional -2V | 沿用 | **沿用**（不變，v1.1 §3.5 歸納值保留） |
| 資產卡比例 | 1:1（v1.1 已修正） | **1:1**（不變） |
| 留場修正 | -1V | **-1V**（不變） |
| 行動點成本 | -1V（除盟友卡與技能卡外） | **-1V**（不變） |
| 稀有度抵扣 | 由效果價值反推（v1.1 §5） | **由效果價值反推**（不變） |

**核心改動只有一條**：等級抵扣階梯被「起始投入抵扣 = 起始投入點數 × 1V」取代，其他全部不變。

## 二、新版費用公式

### 2.1 資產卡（含書籍卡、遺跡卡的本體）

> **費用 = 效果價值 - 1V（行動點）- 1V（留場）- 起始投入抵扣 - 稀有度抵扣 - Exceptional 抵扣 - 負面副作用**

| 項目 | 計算 |
|---|---|
| 效果價值 | 依 v1.1 §4 規則計算（單次效果價值 × 預期使用次數，含成長型逐層累加、無消耗引擎加權等） |
| 行動點抵扣 | 固定 -1V |
| 留場抵扣 | 固定 -1V |
| 起始投入抵扣 | 起始投入點數 × 1V（範圍 0–5） |
| 稀有度抵扣 | 依 v1.1 §5 稀有度反推區間表（不變） |
| Exceptional 抵扣 | 有 Exceptional 標記時 -2V，無則 0 |
| 負面副作用 | 依 v1.1 §7 負面副作用參考表（不變） |

費用上下限：0–6（不變）。封頂規則不變。

### 2.2 事件卡

> **費用 = 效果價值 - 起始投入抵扣 - 稀有度抵扣 - Exceptional 抵扣 - 負面副作用**

事件卡無留場抵扣（打出後進棄牌堆），無行動點抵扣（行動點成本本身不計入打出費用）。

### 2.3 盟友卡

> **費用 = HP 價值 + SAN 價值 + 留場價值 + 被動價值 + 能力價值 - 起始投入抵扣 - 稀有度抵扣 - Exceptional 抵扣**

盟友卡的「總價值」依 v1.1 §3.2 拆解為四項，公式中起始投入抵扣替換原等級抵扣，其他不變。

| 項目 | 價值 |
|---|---|
| 盟友 1 HP | 0.5V |
| 盟友 1 SAN | 0.5V |
| 留場價值 | 2V |
| 無代價被動（單屬性 +1） | 0.5V |
| 有代價能力 | 效果正常計價 × 預期使用次數（v1.1 廢除「有代價 = 0V」規則） |

### 2.4 技能卡

> **費用固定為 0**（不變）

技能卡費用恆為 0，價值體現在加值區間與消費上限。技能卡仍可有起始投入點數，影響的是「玩家能否購買此技能卡進牌組」的入場費與成長空間。

### 2.5 升級版卡片（書籍/遺跡 transform 後產出）

> **費用恆為 0**（沿用 v1.1 §4.6 規定）

升級版卡片不走標準費用公式。其資料庫表現為：起始投入點數固定 5、Exceptional 標記為 false、`acquired_method` 不可為 `purchase`、玩家不可投入後續強化。

## 三、起始投入抵扣的計算範例

### 3.1 純基本卡（起始投入 0 點）

| 項目 | 數值 |
|---|---|
| 卡片性質 | LV0 等級的標準武器，效果價值 6V |
| 起始投入點數 | 0 |
| 起始投入抵扣 | 0 × 1V = 0V |
| 稀有度反推 | 6V → 標準（-2V） |
| 費用 | 6 - 1 - 1 - 0 - 2 = **2** |

### 3.2 中階強卡（起始投入 3 點）

| 項目 | 數值 |
|---|---|
| 卡片性質 | 中階武器，效果價值 13V（含使用次數加成等） |
| 起始投入點數 | 3 |
| 起始投入抵扣 | 3 × 1V = 3V |
| 稀有度反推 | 13V → 進階（-3V） |
| 費用 | 13 - 1 - 1 - 3 - 3 = **5** |

### 3.3 滿配額神卡（起始投入 5 點，無 Exceptional）

| 項目 | 數值 |
|---|---|
| 卡片性質 | 高階傳奇武器，效果價值 22V |
| 起始投入點數 | 5 |
| 起始投入抵扣 | 5 × 1V = 5V |
| 稀有度反推 | 22V → 傳奇（-5V） |
| 費用 | 22 - 1 - 1 - 5 - 5 = **10**，封頂為 **6** |

### 3.4 滿配額 Exceptional 神卡（起始投入 5 點 + Exceptional）

| 項目 | 數值 |
|---|---|
| 卡片性質 | 高階傳奇武器 + Exceptional，效果價值 22V |
| 起始投入點數 | 5 |
| 起始投入抵扣 | 5V |
| Exceptional 抵扣 | 2V |
| 稀有度反推 | 22V → 傳奇（-5V） |
| 費用 | 22 - 1 - 1 - 5 - 5 - 2 = **8**，封頂為 **6** |

對玩家而言，這張卡的入場費為 5 × 2 = **10 XP**，後續無剩餘配額可投入。

### 3.5 起始 3 點 + Exceptional（玩家還能再投資的強卡）

| 項目 | 數值 |
|---|---|
| 卡片性質 | 中階強卡 + Exceptional，效果價值 13V |
| 起始投入點數 | 3 |
| 起始投入抵扣 | 3V |
| Exceptional 抵扣 | 2V |
| 稀有度反推 | 13V → 進階（-3V） |
| 費用 | 13 - 1 - 1 - 3 - 3 - 2 = **3** |

對玩家而言，入場費為 3 × 2 = **6 XP**，後續可再投資 2 點配額，每點 2 XP（共需 4 XP），總投入 10 XP 才能滿配。

## 四、與 v1.1 既有驗證案例的相容性

### 4.1 The Red Clock（v1.1 §8.6）

| 項目 | v1.1 計算 | 新規範計算 |
|---|---|---|
| 效果價值 | 15V | 15V（不變） |
| 等級/起始投入 | LV5（-5V，舊階梯） | 起始投入 5 點（-5V，新公式） |
| Exceptional | -2V | -2V（不變） |
| 稀有度 | 稀有（-4V） | 稀有（-4V，不變） |
| 費用 | 15 - 1 - 1 - 5 - 4 - 2 = 2 | 15 - 1 - 1 - 5 - 4 - 2 = **2** ✓ |

結果一致。新公式對既有案例的計算結果不變（因為 LV5 = 起始投入 5 點，抵扣值都是 5V）。

### 4.2 Gold Pocket Watch（v1.1 §8.13）

| 項目 | v1.1 計算 | 新規範計算 |
|---|---|---|
| 效果價值 | 14V | 14V |
| 等級/起始投入 | LV5（-5V） | 起始投入 5 點（-5V） |
| Exceptional | -2V | -2V |
| 稀有度 | 進階（-3V） | 進階（-3V） |
| 費用 | 14 - 1 - 1 - 5 - 3 - 2 = 2 | 14 - 1 - 1 - 5 - 3 - 2 = **2** ✓ |

結果一致。

### 4.3 Antikythera（v1.1 §8.4）

| 項目 | v1.1 計算 | 新規範計算 |
|---|---|---|
| 效果價值 | 13V | 13V |
| 等級/起始投入 | LV5（-5V） | 起始投入 5 點（-5V） |
| Exceptional | 無 | 無 |
| 稀有度 | 進階（-3V） | 進階（-3V） |
| 費用 | 13 - 1 - 1 - 5 - 3 = 3 | 13 - 1 - 1 - 5 - 3 = **3** ✓ |

結果一致。

### 4.4 結論

> **新公式對所有 v1.1 既有驗證案例的計算結果完全一致**，因為既有案例都假設「LV5 = 抵扣 5V」，新公式下「起始投入 5 點 = 抵扣 5V」數值相同。

新公式真正帶來改變的是 **LV1–LV4 區間的計算**（舊階梯非線性 -0.5V/-1V/-2V/-3V，新公式線性 -1V/-2V/-3V/-4V）。Part 3 提供 LV1–LV4 既有卡片的重算清單。

## 五、稀有度反推區間表（沿用 v1.1 §5，不變）

> **稀有度由效果價值決定，不由起始投入點數決定。**

| 稀有度 | 代碼 | 效果價值區間 | 抵扣 |
|---|---|---|---|
| 隨身 | `pocket` | 0 – 3V | 0 |
| 基礎 | `basic` | 3 – 6V | -1V |
| 標準 | `standard` | 6 – 10V | -2V |
| 進階 | `advanced` | 10 – 15V | -3V |
| 稀有 | `rare` | 15 – 22V | -4V |
| 傳奇 | `legendary` | 22V 以上 | -5V |

> 起始投入 5 點的卡片不自動等於傳奇。稀有度永遠由打出效果價值反推。

## 六、消耗類型修正的處理

舊規則書第三章 §8.6 的「消耗類型修正（暫定）」（stay -1V、discard 0、short_rest -1V、long_rest -2V、removed -3V）在 v1.1 修正案中被「預期使用次數」吸收。本規範**不重新引入消耗類型修正**，繼續沿用 v1.1 的處理：

| 消耗類型 | 處理方式 |
|---|---|
| `stay`（留場） | 已包含在標準資產卡公式的 -1V 留場抵扣 |
| `discard`（進棄牌堆） | 預期使用次數通常為 1 次 |
| `short_rest`（短休息回復） | 預期使用次數視回復次數計算 |
| `long_rest`（長休息回復） | 預期使用次數視回復次數計算 |
| `removed`（移除出遊戲） | 預期使用次數固定 1 次 |

## 七、能力價值計算原則（沿用 v1.1 §4，不變）

| 能力類型 | 計算 |
|---|---|
| 有代價的能力 | 效果正常計價，代價透過預期使用次數間接反映 |
| 無代價的被動 | 依效果價值表計算（不乘次數） |
| 二選一效果 | 取較高價值的效果 |
| 條件觸發效果 | 依效果價值表計算（條件不打折） |
| 成長型被動 | 逐層累加每一層的效果價值（v1.1 §4.4） |
| 無消耗引擎 | 加權單次效果 × 預期觸發次數（v1.1 §4.5） |
| 多模式使用次數 | 列 3–4 種典型分配組合取期望值（v1.1 §4.3） |

## 八、cast_spell / transform 等特殊動詞的處理

### 8.1 transform 動詞（沿用 v1.1 §4.6，不變）

> **transform 效果價值 = 2V（直接進場）+（新卡效果價值 - 原卡效果價值）**

書籍/遺跡的研究升級透過 `transform` 動詞觸發，新卡片費用恆為 0。本規範不變動 transform 的計算規則。

### 8.2 升級版卡片不重複計算起始投入

升級版卡片的起始投入點數雖固定為 5，但**不在費用公式中扣減**——升級版的費用直接寫死為 0。這是因為升級版的「對價」已經由研究投入（累積閱讀次數、檢定門檻）支付，不再走 XP 軌道。

## 九、AI 生成 Prompt 的更新指令

MOD-01 卡片設計器的 AI 生成功能（`buildGeminiPrompt(userDescription)`，admin-card-designer.html L2329）內嵌了 300+ 行的全規則 prompt，本次規範更新後該 prompt 必須同步調整：

### 9.1 必須加入 prompt 的新規則

1. **卡片有「起始投入點數」欄位**（範圍 0–5），對應 V 值費用公式的「起始投入抵扣」項。
2. **5 點為配額硬上限**，AI 生成時不可建議超過 5 點。
3. **Exceptional 標記在費用公式中額外抵扣 -2V**（與 v1.1 §3.5 一致）。
4. **稀有度由效果價值反推**，不由起始投入點數決定。

### 9.2 必須從 prompt 移除的舊規則

1. **「LV0–LV5 各等級獨立設計效果」的引導敘述**全部刪除。
2. **舊版等級抵扣階梯**（任一版本）刪除，替換為線性 1:1。
3. **「升級時為每個等級各設計一份效果」的引導**刪除。

### 9.3 prompt 結構調整指令

請 Claude Code 在 `buildGeminiPrompt()` 函式中：
- 在「卡片設計步驟」段落，刪除「為 LV0–LV5 各設計一份效果」相關引導文字
- 在「費用計算」段落，將等級抵扣替換為起始投入抵扣的線性公式
- 在「稀有度反推」段落，明確聲明稀有度與起始投入點數無關

## 十、計算函式的程式碼層級替換

MOD-01 卡片設計器（admin-card-designer.html）的 V 值計算函式群（L841–L962）需依以下指令調整：

### 10.1 calcEffectValue(eff)（L841）

不需更動。單一效果的 V 值計算規則沒變。

### 10.2 calcAllEffectsValue()（L847）

不需更動。

### 10.3 calcWeaponOutputValue() / calcAllyBaseValue()（L852/L859）

不需更動。

### 10.4 calcBalanceInfo()（L865）

**需更動**：將原讀取 `card.level` 並查表計算等級抵扣的邏輯，改為「起始投入抵扣 = card.level × 1V」（在 `level` 欄位語意改寫前繼續使用此欄位，改名後相應替換）。

### 10.5 rarityFromEffectValue(effectValue)（L952）

不需更動。稀有度反推規則沒變。

### 10.6 updateBalancePanel()（L962）

**需更動**：平衡面板顯示時，將「等級抵扣」標籤改為「起始投入抵扣」，數值改用線性公式。
