## 一、專案身份｜Project Identity

**專案類型：** 克蘇魯神話世界觀・卡牌驅動・牌組構築・回合制・合作冒險・網頁遊戲

**核心精神一句話：**
你帶著一把手槍、六發子彈和一瓶威士忌走進迷霧，然後在黑暗裡一點一點找到活下去的方法。

| 項目 | 規格 |
|------|------|
| 目標平台 | 網頁瀏覽器（響應式，行動裝置相容） |
| 玩家人數 | 1–4 人合作（從第一天起納入多人架構設計） |
| 戰鬥節奏 | 慢節奏桌遊級策略（每場戰鬥 15+ 分鐘） |
| 美術方向 | 暗黑哥德奇幻，1920–30 年代偵探黑色電影 × 宇宙恐怖 |

## 二、八大設計支柱｜Eight Design Pillars

以下八根支柱是本專案的結構基礎。任何設計決策若與支柱矛盾，支柱優先。

### 支柱 1：陣營與構築｜Faction & Deckbuilding

- 調查員創建時綁定一個**主陣營**，完全存取該陣營卡池。
- **副陣營**在冒險開始時鎖定，需透過劇情觸發、資源累積或代價交換在冒險中解鎖。
- 中立卡可供所有調查員使用。

> **詳細設計：** 見《支柱一詳細設計：陣營與構築》。

### 支柱 2：雙軌檢定｜Dual-Track Skill Checks

- **主要檢定：擲骰 + 修正值。** 玩家感受到成長帶來的數值提升，具備掌控感。
- **特殊檢定：混沌袋抽取。** 保留給宇宙恐怖時刻 — 直面舊日支配者、翻開禁忌典籍、神啟事件。無論多強都無法保證安全。
- **戰鬥中的應用：** 物理攻擊使用擲骰（掌控感），施法類攻擊使用混沌袋（玩火感）。檢定使用的屬性由**戰鬥風格卡**動態決定，而非由武器固定指定。

> **完整規則：** 見第二章第一節（檢定系統）及第二章第五節（混沌袋與法術系統）。

### 支柱 3：章節制冒險｜Campaign Chapters

- 每個戰役由約 **10 個章節**組成，劇情連貫。
- 章節間保持完整狀態持續（牌組、生命、理智、劇情旗標、資源、NPC 關係、混沌袋修改）。
- 通關後調查員可存檔並帶入其他戰役。

> **層級結構：** 戰役（Campaign）→ 章節（Chapter）→ 關卡（Stage）→ 場景（Scenario）。詳見第五章。

### 支柱 4：三層死亡｜Three-Layer Death

- **第一層 — 創傷侵蝕：** 被擊倒（HP 或 SAN 歸零）獲得創傷，永久降低上限。上限歸零 = 永久死亡。節奏：緩慢磨損。
- **第二層 — 神啟詛咒：** 特殊詛咒卡被洗入牌組，抽到即觸發，無法正常移除。特定高階詛咒可直接刪除角色。節奏：定時炸彈。
- **第三層 — 倒地瀕死：** HP 或 SAN 歸零時進入瀕死狀態。瀕死檢定沿用 D&D 規則（d20, DC 10）。節奏：突發危機。
- **永久死亡是真實的。角色死亡 = 資料刪除。**

> **完整規則：** 見第二章第四節（瀕死與救援系統）。

### 支柱 5：成長子系統｜Growth Subsystems

- 包含：卡片強化、裝備鍛造、道具製作、角色技能樹、陣營天賦。
- **所有系統的產出都是卡片或對卡片的修改** — 卡片是通用單位。
- 成長必須在世界中贏得，不能透過抽象選單獲取。
- 部分成長功能（鍛造、製作）鎖在團隊精神系統後面，需投資凝聚力解鎖。

> **完整規則：** 見第四章（成長系統總綱）。

### 支柱 6：動態牌組｜Dynamic Deckbuilding

- 冒險中持續獲得新卡，玩家即時決定是否納入牌組。
- 每一張加入的卡都是有重量的決策。

### 支柱 7：模組化架構｜Modular Architecture

- **調查員與場景是兩個獨立模組**，可各自獨立設計與擴充。
- 擴充路線三軌：新調查員包 / 新冒險劇本 / 大型擴充（兩者皆有）。
- 提供**三大工具模組**：**創角器**（玩家 / 社群）、**關卡編輯器**（玩家 / 社群）、**系統管理員後台**（開發者 / 管理員，含數值模擬器群組）。

### 支柱 8：探險驅動構築｜Exploration-Driven Building

- **起始牌組精簡**（約 15–20 張），這不是完整裝備。
- **消耗品真實消耗** — 子彈打完就沒了，藥用了就少一張，牌組會「變薄」。
- **新卡透過探索場景獲得** — 搜索書櫃、翻查屍體、與 NPC 交易。不是戰鬥後的三選一獎勵畫面。
- **成長曲線由場景設計與玩家選擇共同塑造。**

## 三、設計紅線｜Design Red Lines

以下是絕對不可違反的設計禁區。若任何設計方案觸及紅線，必須立即停止並重新思考。

1. **不得出現無限循環構築。** 行動經濟有上限、重複效果有遞減、資源費用防止免費循環。
2. **不得讓成長消滅挑戰 — 但威脅有層級。**
   - **不可戰勝的存在：** 外神與大部分舊日支配者不是用來被「打敗」的。玩家能做的是絆倒祂、拖慢祂、封印裂隙 — 通常需要有人犧牲。
   - **可被武力擊敗的存在：** 較低階的神話生物、邪教徒、人類敵人可以被正面擊敗。
   - **力量曲線的天花板由敵人的存在層級決定，而非由數值無限攀升決定。**
3. **不得讓卡片獲取脫離敘事。** 沒有抽象的「獎勵畫面」。每張卡的獲得都有故事脈絡。
4. **不得弱化死亡的重量。** 永久死亡是真實的。不設「付費復活」「花金幣取消死亡」等機制。
5. **不得讓多人模式成為事後補丁。** 1–4 人合作從架構層開始設計。
6. **不得破壞模組獨立性。** 調查員與場景的耦合僅限於明確定義的介面。

## 四、核心情感設計｜Emotional Design Targets

克蘇魯遊戲的靈魂不在機制，在情感。以下是每個系統設計時應追求的情感體驗：

| 情境 | 目標情感 |
|------|----------|
| 探索未知地點 | 好奇與不安並存 — 「我想知道裡面有什麼，但我怕打開之後無法承受」 |
| 獲得新卡片 | 如獲至寶的珍惜 — 每一把新武器都意味著多一次活命的機會 |
| 消耗品用盡 | 真實的焦慮 — 「我只剩兩發子彈，但還有三個房間沒搜」 |
| 面對混沌袋 | 無助與臣服 — 「無論我準備得多好，命運不由我」 |
| 隊友倒地 | 急迫的抉擇 — 「我救他就會暴露自己，不救他他就死了」 |
| 累積創傷 | 溫水煮青蛙的恐懼 — 「我還活著，但我的上限越來越低」 |
| 角色永久死亡 | 真實的失落與敬意 — 這個角色的故事結束了，但他的犧牲有意義 |
| 副陣營解鎖 | 禁忌的誘惑 — 「我不應該碰這個力量，但我需要它」 |
| 團隊精神解鎖里程碑 | 戰友的羈絆 — 「我們一起走過地獄，這份默契是真實的」 |
| 重返已知關卡 | 熟悉中的陌生 — 「我記得這個地方，但有些東西變了……而且更危險了」 |
| 休息時刻 | 暴風雨前的寧靜 — 「趁現在還能喘口氣，整理好自己」 |
| 分支路線抉擇 | 不可逆的重量 — 「走這條路就回不了頭了」 |
| 用熟練武器戰鬥 | 職人的自信 — 「這把獵槍在我手上，每一發都有意義」 |
| 用不熟練的方式掙扎 | 絕望的即興 — 「我不會用劍，但我沒有別的選擇了」 |
| 施放神秘法術 | 玩火的代價 — 「力量從我體內湧出，但我不確定我還能控制多久」 |
| 閱讀戰鬥敘事描述 | 身歷其境 — 不是「你造成 3 點傷害」，而是一段有畫面的描述 |
| 觀察敵人狀態描述 | 獵人的直覺 — 「牠的動作變慢了……再一擊應該就夠了，但我不確定」 |
| 發現敵人弱點 | 破局的興奮 — 「火焰有用！快，把火油瓶傳給我！」 |
| 戰鬥風格卡抽到不擅長的屬性 | 命運的玩笑 — 「該死，偏偏這時候要靠意志力……」 |
| 瀕死檢定 | 命懸一線 — 「求你了，10 以上就好……」 |
| 閱讀神話典籍 | 禁忌的渴望 — 「我知道繼續讀下去會付出代價，但我需要知道真相」 |
| 研究遠古遺跡 | 考古者的敬畏 — 「這東西比人類文明還古老……它在對我說什麼？」 |

## 五、技術設計原則｜Technical Design Principles

1. **卡片是通用單位。** 武器是卡片、道具是卡片、技能是卡片、詛咒是卡片。所有系統都透過卡片介面互動。
2. **狀態持續是核心。** 章節間、戰役間的狀態保持必須可靠且完整。這不是 roguelike。
3. **模組邊界清晰。** 調查員模組與場景模組透過定義明確的 API 介面溝通，不得直接耦合內部狀態。
4. **人數縮放內建。** 敵人數值、線索需求、遭遇效果都必須包含人數縮放參數。
5. **反作弊考量。** 永久死亡機制意味著資料完整性至關重要。關鍵狀態必須在伺服器端驗證。
6. **編輯器優先設計。** 創角器和關卡編輯器不是事後功能。內部開發工具應以最終成為玩家工具為目標設計。
7. **系統管理員模組。** 需建置系統管理員後台，內含數值模擬器群組（骰子機率分析器、Boss 戰鬥模擬器、創角點數分配模擬器、敵人位階設計器、傷害曲線與 HP/SAN 推算器等）。
8. **圖示優先。** 遊戲中的屬性、戰鬥風格、元素、狀態效果、動作類型等資訊使用圖案表示（非文字）。開發測試階段以有顏色的文字代替，正式圖案設計為後續美術任務。

## 六、設計差異化定位｜Competitive Differentiation

| 面向 | Arkham Horror LCG | Slay the Spire | 本專案 |
|------|-------------------|-----------------|--------|
| 起始牌組 | 厚（30+ 張預構築） | 無（從零開始） | 精簡（15–20 張，帶著最低限度出門） |
| 卡片獲取 | 花經驗值購買 | 戰鬥後隨機獎勵 | 探索場景中有機發現 |
| 消耗品 | 部分消耗 | 無真實消耗 | 完全真實消耗（子彈打完就沒了） |
| 構築時機 | 關卡之間 | 戰鬥之間 | 冒險進行中即時 |
| 死亡後果 | 獲得創傷但可繼續 | 重新開始一局 | 角色資料永久刪除 |
| 檢定機制 | 混沌袋 | 無（純卡牌） | 雙軌（擲骰 + 混沌袋） |
| 戰鬥解析 | 固定屬性檢定 | 無（卡牌效果） | 戰鬥風格卡動態決定檢定屬性 |
| 手牌決策 | 單一用途 | 單一用途 | 三合一（打出 / 加值 / 消費） |
| 平台 | 實體卡牌 | PC / 主機 | 網頁瀏覽器 |

## 七、核心層級結構｜Core Hierarchy

### 7.1 遊戲進程層級

```
戰役（Campaign）
  └ 章節（Chapter）— 約 10 章，劇情連貫
      └ 關卡（Stage）— 通常 1 個，可多個
          └ 場景（Scenario）— 關卡內的獨立遊玩空間
```

- **戰役**：一組完整的冒險劇情，包含約 10 個章節。
- **章節**：戰役中的一個段落，結束後進入長休息。章節之間保持狀態持續。
- **關卡**：章節內的一個獨立遊玩單元，包含地圖（地點）、目標牌堆（ACT）、議程牌堆（Agenda）。一般來說一個章節只有一個關卡。
- **場景**：關卡內的獨立空間，可有不同的環境配置和事件。

**支線結構（不隸屬章節）：**

```
關卡（Stage）
  └ 場景（Scenario）
```

### 7.2 敵人位階層級

| 位階 | 代碼 | 中文 | 定位 |
|------|------|------|------|
| 第一階 | `minion` | 雜兵 | 數量壓制，單體弱 |
| 第二階 | `threat` | 威脅 | 需認真對付的個體 |
| 第三階 | `elite` | 精英 | 小型 Boss，有特殊能力 |
| 第四階 | `boss` | 頭目 | 關卡核心敵人 |
| 第五階 | `titan` | 巨頭 | 戰役級終極威脅 |

### 7.3 卡片類型

| 類型 | 代碼 | 打出後去向 | 說明 |
|------|------|-----------|------|
| 資產 | `asset` | 留在場上 | 武器、裝備、道具、法術 |
| 事件 | `event` | 進入棄牌堆 | 一次性效果 |
| 技能 | `skill` | 進入棄牌堆 | 專門用於加值（Commit） |
| 盟友 | `ally` | 留在場上 | NPC 夥伴，有獨立 HP/SAN |

**特殊卡片：**

| 類型 | 說明 |
|------|------|
| 簽名卡 | 調查員獨有，無法移除，2–3 張 |
| 個人弱點 | 調查員獨有的負面卡，強制納入牌組 |
| 神啟卡 | 詛咒卡，被洗入牌組，抽到即觸發 |
| 書籍 | 資產卡的一種，附帶「閱讀」長期投資行動 |
| 遺跡 | 資產卡的一種，附帶「研究」長期投資行動 |

### 7.4 卡片稀有度

| 稀有度 | 代碼 | 價值抵扣 | 說明 |
|--------|------|----------|------|
| 隨身 | `pocket` | 0 | 最基礎的工具 |
| 基礎 | `basic` | -1V | 基礎工具和能力 |
| 標準 | `standard` | -2V | 有特色的裝備和能力 |
| 進階 | `advanced` | -3V | 強力且有主題性的卡片 |
| 稀有 | `rare` | -4V | 稀有強力卡片 |
| 傳奇 | `legendary` | -5V | 改變遊戲規則的卡片 |

### 7.5 元素屬性

| 元素 | 代碼 | 說明 |
|------|------|------|
| 物理 | `physical` | 基礎傷害類型 |
| 火 | `fire` | 與燃燒狀態聯動 |
| 冰 | `ice` | 與冷凍狀態聯動 |
| 雷 | `electric` | 與潮濕狀態聯動 |
| 神秘 | `arcane` | 遊戲最強元素，無任何怪物有抗性 |

### 7.6 七大屬性

| 屬性 | 代碼 | 影響範圍 | 衍生值 |
|------|------|---------|--------|
| 力量 | `strength` | 近戰傷害、負重、破壞、體力檢定 | — |
| 敏捷 | `agility` | 遠程攻擊、閃避、先攻、手巧檢定 | — |
| 體質 | `constitution` | 毒素/疾病抵抗、體力持久 | HP 上限 |
| 智力 | `intellect` | 調查、研究、解謎、神秘學知識 | — |
| 意志 | `willpower` | 恐懼抵抗、法術施放、精神防禦 | SAN 上限 |
| 感知 | `perception` | 搜索、陷阱偵測、敵人觀察、伏擊抵抗 | — |
| 魅力 | `charisma` | NPC 互動、說服、威嚇、交易價格 | — |

**屬性修正值：** 屬性值 = 修正值（1:1 對應）。屬性 3 = 修正值 +3。

## 八、核心術語表｜Master Glossary

> **重要原則：** 克蘇魯神話相關的專有名詞（舊日支配者、外神、地名、書名、組織名等），本專案**嚴禁自創翻譯**，一律採用台灣桌遊 / TRPG 社群的既有慣用譯名。詳見《克蘇魯神話專有名詞對照表》。

>
> 本章是規則書 v1.0 第一章，與後續章節共同構成完整的遊戲設計文件。
>
> **關於專案原創者 Uria：** Uria 不具備資工相關背景，對技術實作完全不介入。Uria 的角色是設計決策者與體驗把關者。所有技術層面的決策由開發者自行判斷。
>
> 在你協助本專案時，請：
> 1. 先完整閱讀本章，理解設計意圖而非僅記住規則。
> 2. 任何設計提案都應對照八大支柱與六條紅線。
> 3. 若發現潛在衝突，主動提出而非默默妥協。
> 4. 保持克蘇魯的情感基調 — 這不是一個讓玩家感覺無敵的遊戲。
> 5. 尊重「探險驅動構築」的核心哲學。
> 6. 與 Uria 溝通時，用遊戲設計語言描述方案，避免技術術語。
> 7. 戰鬥風格卡是核心機制 — 任何涉及攻擊、檢定、武器的設計都必須考慮。
> 8. 術語表中的程式碼代碼是規範性的，所有開發必須遵循。
