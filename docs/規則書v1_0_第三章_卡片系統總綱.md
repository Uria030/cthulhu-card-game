## 一、基礎單位與價值錨點｜Base Unit

> **1V = 1 行動點 = 1 資源 = 抽 1 張牌 = 造成 1 點傷害**

此為價值計算的基準單位，所有效果價值皆以此為錨點推導。

---

## 二、三合一卡片用途系統｜Triple-Use Card System

### 2.1 設計哲學

每張手牌同時承載三種用途，讓玩家手中的每一張牌都有多重選擇，減少「死牌」，壓縮牌庫厚度。

### 2.2 三種用途互斥

| 用途 | 代碼 | 動作 | 行動點 | 結果 |
|------|------|------|--------|------|
| 打出 | `play` | 花費資源，將卡片放到場上或發動效果 | 1 AP | 依卡片類型留在場上或進入棄牌堆 |
| 加值 | `commit` | 檢定前從手牌投入，提供屬性圖示加值 | 不花費 | 自動進入棄牌堆 |
| 消費 | `consume` | 從手牌直接棄掉，觸發輔助效果 | 1 AP | 自動進入棄牌堆 |

選擇其中一種，就放棄了其他兩種。這是手牌管理的核心決策。

### 2.3 價值獨立原則

三種用途各自獨立計算價值，互不影響：

- 打出效果的價值 → 決定費用與稀有度
- 加值的價值 → 屬性圖示價值 + 附贈效果價值（技能卡用等級區間檢驗）
- 消費效果的價值 → 獨立計算，受稀有度或等級上限約束

一張卡片上配了很強的屬性圖示，不會讓它的打出費用變貴；反之亦然。

## 三、加值系統（Commit）｜Commit System

### 3.1 屬性圖示規格

| 項目 | 規格 |
|------|------|
| 可標註的屬性 | 七大屬性（力量、敏捷、體質、智力、意志、感知、魅力） |
| 每個屬性的加值範圍 | +0 至 +5（極端可至 +6） |
| 一張卡片可標註的屬性數量 | 1 個以上，無硬性上限 |
| 萬能加值 | 可設計為「任意屬性 +X」的萬能圖示 |
| JSON 欄位 | `commit_icons` |

> **高風險提醒：** `commit_icons`（手牌加值時的屬性圖示）與 `attribute_modifiers`（場上武器的檢定修正）是兩個完全不同的概念，必須在資料結構中完全分開，共用會導致價值計算錯誤。

### 3.2 技能卡的加值特權

| 項目 | 技能卡 | 非技能卡 |
|------|--------|---------|
| 屬性圖示 | ✓ | ✓ |
| 附贈效果（成功時） | ✓ | ✗ |
| 附贈效果（失敗時） | ✓ | ✗ |

### 3.3 加值不影響打出費用

屬性圖示的價值不計入打出效果的費用計算。使用加值時，玩家已放棄打出機會，這本身就是代價。

## 四、消費系統（Consume）｜Consume System

### 4.1 固定成本

| 成本項目 | 價值 |
|----------|------|
| 1 行動點 | 1V |
| 棄掉 1 張手牌 | 0.5V |
| **固定成本合計** | **1.5V** |

消費效果必須 > 1.5V 才值得設計。

### 4.2 合法效果範圍

消費效果**不能是基本動作**（移動、攻擊、調查），限定為輔助性效果：

獲得資源（1V/資源）、補充使用次數（0.5V/次）、回復 HP（1.5V/點）、回復 SAN（1.5V/點）、抽牌（1V/張）、獲得正面狀態（依狀態價值表）、移除負面狀態（依狀態價值表）、取消傷害/恐懼（0.5V/點）

### 4.3 消費效果價值上限

**非技能卡 — 由稀有度決定：**

| 稀有度 | 消費效果價值上限 |
|--------|----------------|
| 隨身 / 基礎 | 2.5V – 3V |
| 標準 | 5V – 6V |
| 進階 | 7V – 8V |
| 稀有 | 8V – 9V |
| 傳奇 | 9V – 10V |

**技能卡 — 由等級決定：** 同上表，LV0–1 對應隨身/基礎，LV2 對應標準，以此類推。

### 4.4 消費效果不影響稀有度反推

稀有度反推只看打出效果。消費效果不能單獨把稀有度拉高。

## 五、卡片效果語言｜Card Effect Language

每個卡片效果由六大要素組成：

```
觸發時機（When）→ 條件限制（If）→ 費用（Cost）→ 目標（Target）→ 效果（Effect）→ 持續時間（Duration）
```

### 5.1 觸發時機

| 代碼 | 中文 | 說明 |
|------|------|------|
| `on_play` | 打出時 | 從手牌打出到場上時 |
| `on_commit` | 加值投入時 | 投入到檢定時（技能卡專用） |
| `on_consume` | 消費時 | 從手牌消費棄掉時 |
| `on_enter` | 進場時 | 卡片進入場上時 |
| `on_leave` | 離場時 | 卡片離開場上時 |
| `on_draw` | 抽到時 | 從牌庫抽到手牌時 |
| `on_success` | 檢定成功時 | 擲骰結果 ≥ DC |
| `on_fail` | 檢定失敗時 | 擲骰結果 < DC |
| `on_take_damage` | 受到傷害時 | 受到物理傷害後 |
| `on_take_horror` | 受到恐懼時 | 受到恐懼傷害後 |
| `before_take_damage` | 將要受到傷害時 | 傷害結算前的攔截窗口 |
| `before_take_horror` | 將要受到恐懼時 | 恐懼結算前的攔截窗口 |
| `before_downed` | 將要倒地時 | HP/SAN 即將歸零的瞬間 |
| `reaction` | 反應 | 不佔行動點，條件觸發 |
| `passive` | 被動 | 持續生效 |
| `free_action` | 免費行動 | 不佔行動點的主動觸發 |
| `action` | 行動 | 佔 1 行動點的主動觸發 |
| `round_start` | 回合開始時 | 每回合開始觸發 |
| `round_end` | 回合結束時 | 每回合結束觸發 |

## 六、狀態效果系統｜Status Effect System

### 6.1 核心規則

- **所有狀態可堆疊** — 同一狀態可累積多層。
- **經歷完整回合後減 1 層** — 除非狀態自身有特殊減層規則。
- 狀態分為**數值型**（層數 = 效果強度）和**開關型**（效果固定，層數 = 持續時間）。

### 6.2 負面狀態

#### 數值型負面狀態（層數 = 效果強度）

| 狀態 | 代碼 | 效果 | 價值/層 |
|------|------|------|---------|
| 中毒 | `poison` | 受到的攻擊傷害 +X | 3V |
| 流血 | `bleed` | 回合結束時扣 X 點 HP | 2V |
| 燃燒 | `burning` | 回合開始扣 X HP，火攻增傷 +X，移除潮濕 | 3V |
| 冷凍 | `frozen` | 移動花費 +1（開關），冰攻增傷 +X（數值） | 3V |
| 潮濕 | `wet` | 雷屬性傷害 +X | 1V |
| 發瘋 | `madness` | 受到的恐懼傷害 +X | 6V |
| 標記 | `marked` | 受到的傷害和恐懼都 +X，與隱蔽互抵 | 6V |
| 脆弱 | `vulnerable` | 受到的物理傷害 +X | 2V |
| 無力 | `weakness_status` | 造成的近戰傷害 -X | 2V |

## 八、費用公式｜Cost Formula

### 8.1 事件卡

> **費用 = 總價值 - 等級抵扣 - 稀有度抵扣 - 負面副作用**

換算比例：**1:1**。打牌成本 = 1 行動點 = 1V（卡片效果需 > 1V 才值得打出）。

### 8.2 資產卡

> **費用 = （總價值 - 留場修正 - 消耗類型修正 - 等級抵扣 - 稀有度抵扣）÷ 2**

換算比例：**2:1**。總價值 = 單次效果價值 × 預期使用次數。

### 8.3 盟友卡

> **費用 = 總價值 - 等級抵扣 - 稀有度抵扣**

換算比例：**1:1**。

總價值 = HP 價值 + SAN 價值 + 留場價值 + 能力價值。

| 項目 | 價值 |
|------|------|
| 盟友 1 HP | 0.5V |
| 盟友 1 SAN | 0.5V |
| 留場價值 | 2V |
| 持續被動（單屬性 +1） | 0.5V |
| 有代價的能力 | 0V |

### 8.4 技能卡

費用固定為 **0**。價值體現在等級，用經驗值購買。

### 8.5 等級抵扣

| 等級 | 抵扣 |
|------|------|
| LV0 | 0 |
| LV1 | -1V |
| LV2 | -2V |
| LV3 | -3V |
| LV4 | -4V |
| LV5 | -5V |

升級時卡片可能同時獲得能力提升（價值增加），最終費用不一定變便宜。

### 8.6 消耗類型修正（暫定）

| 消耗類型 | 代碼 | 修正 | 說明 |
|----------|------|------|------|
| 留在場上 | `stay` | -1V | 打出當下無即時價值 |
| 進入棄牌堆 | `discard` | 0 | 基準 |
| 短休息回復 | `short_rest` | -1V | 受限回復 |
| 長休息回復 | `long_rest` | -2V | 更受限 |
| 移除出遊戲 | `removed` | -3V | 只能用一次 |

### 8.7 能力價值計算原則

| 能力類型 | 價值計算 |
|----------|----------|
| 有代價的能力 | 0V（代價抵消價值） |
| 無代價的被動 | 依效果價值表計算（不乘次數） |
| 無代價的主動/反應 | 依效果價值表計算 |
| 二選一效果 | 取較高價值的效果 |
| 條件觸發效果 | 依效果價值表計算（條件不打折） |

## 十二、卡片設計流程｜Design Workflow

### 12.1 非技能卡設計流程

```
步驟 1：設計打出效果 → 決定類型、效果、等級、費用
步驟 2：系統反推稀有度 → 純看打出效果價值
步驟 3：設計屬性圖示（commit_icons） → 系統提示比對（不強制）
步驟 4：設計消費效果（consume_effects） → 檢驗 ≤ 稀有度上限且 > 1.5V
```

### 12.2 技能卡設計流程

```
步驟 1：選擇等級
步驟 2：設計屬性圖示 + 附贈效果 → 檢驗落在等級價值區間
步驟 3：設計消費效果 → 檢驗 ≤ 等級上限且 > 1.5V
```

### 12.3 設計檢查清單

**通用：** 費用 0–6？公式計算正確？等級抵扣正確？稀有度抵扣正確？陣營歸屬與機制關鍵字吻合？效果語言使用合法代碼？

**武器卡：** 傷害符合稀有度階層？使用次數合理？標註 `combat_style`？標註 `attribute_modifiers`？使用 2:1 比例？

**法術卡：** 標註 `spell_type` 和 `spell_casting`？元素與主題匹配？使用次數低於同級物理武器？

**盟友卡：** HP+SAN 合理？正確計算各項價值？有代價能力 = 0V？使用 1:1 比例？

**技能卡：** 總價值落在等級區間？費用為 0？

> 3. **稀有度反推只看打出效果** — 消費效果不拉高稀有度。
> 4. **`commit_icons` 和 `attribute_modifiers` 必須分開** — 前者是手牌加值，後者是場上武器修正。
> 5. **使用次數是通用概念** — `uses` 欄位 + `uses_label` 顯示名稱（彈藥、充能、證據等）。
> 6. **資產卡 2:1、事件/盟友卡 1:1** — 不同換算比例。
> 7. **有代價能力 = 0V** — 代價抵消價值。
> 8. **條件限制不影響價值** — 不打折。
> 9. **正面狀態刻意稀少** — 這是克蘇魯遊戲。
> 10. **隱蔽是特殊狀態** — 不遵循標準減層規則，移動或攻擊後全部移除。
> 11. **書籍和遺跡共用相同的研究機制欄位** — 只差 `is_book` vs `is_relic`。
