# 規則書 v1.0
# 第五章：場景系統總綱
## Chapter 5: Scenario System

> **文件用途｜Purpose**
> 本章定義場景系統的完整結構，包含分支路線與非線性敘事、劇情旗標系統、
> 地點卡系統、目標牌堆與議程牌堆結構、間章事件、支線關卡與重返機制、隨機地城。
>
> 本章合併了原《補充 07：分支路線與非線性敘事》與《補充 08：支線關卡與重返機制》為統一文件。

## 一、場景層級結構｜Scenario Hierarchy

### 1.1 主線結構

```
戰役（Campaign）
  └ 章節（Chapter）— 約 10 章，劇情連貫，章節間保持狀態持續
      └ 關卡（Stage）— 通常 1 個，可多個
          └ 場景（Scenario）— 關卡內的獨立遊玩空間
```

| 層級 | 代碼 | 說明 |
|------|------|------|
| 戰役 | `campaign` | 一組完整的冒險劇情 |
| 章節 | `chapter` | 戰役中的一個段落，結束後進入長休息 |
| 關卡 | `stage` | 章節內的獨立遊玩單元，包含地圖（地點）、目標牌堆、議程牌堆 |
| 場景 | `scenario` | 關卡內的獨立遊玩空間 |

- 一個章節通常只有一個關卡，但可以有多個。
- 戰役固定十章，分支影響章節內部配置，不增減章節數。

### 1.2 支線結構

支線不隸屬章節，直接從遊戲大廳進入：

```
關卡（Stage）
  └ 場景（Scenario）
```

### 1.3 場景類型

| 類型 | 代碼 | 說明 |
|------|------|------|
| 主線 | `main` | 主線章節的關卡 |
| 預設小關卡 | `side` | 預先設計的支線 |
| 預設小關卡重返 | `side_return` | 支線的重玩版本 |
| 隨機地城 | `side_random` | 每次進入重新生成 |

## 二、關卡結構｜Stage Structure

### 2.1 關卡組成

每個關卡包含以下元素：

| 元素 | 說明 |
|------|------|
| 地圖（地點集合） | 由多個地點組成的探索空間 |
| 目標牌堆（ACT） | 推進玩家端故事進度的卡片序列 |
| 議程牌堆（Agenda） | 推進敵人端計畫的計時卡片序列 |
| 遭遇牌堆 | 每回合抽取的隨機事件卡片池（部分關卡可能不設） |
| 混沌袋配置 | 此關卡的混沌袋標記組成（見第二章第五節） |

### 2.2 目標牌堆與議程牌堆的賽跑

目標牌堆（玩家推進）和議程牌堆（敵人推進）構成時間壓力的核心機制：

- **目標牌堆先完成** → 玩家達成目標
- **議程牌堆先跑完** → 敵人計畫得逞
- 兩者的進度組合決定章節結果

### 2.3 目標牌堆（ACT）卡結構

每張目標牌堆卡有正面和背面：

| 面 | 內容 |
|----|------|
| 正面 | 任務目標描述、推進條件（花費線索、完成特定行動等） |
| 背面 | 完成結算 — 旗標設定、獎勵（經驗值、天賦點等）、敘事指引、結算指引代碼 |

### 2.4 議程牌堆（Agenda）卡結構

每張議程牌堆卡有正面和背面：

| 面 | 內容 |
|----|------|
| 正面 | 毀滅門檻 — 毀滅標記累積到此數值時翻面 |
| 背面 | 後果結算 — 旗標設定、懲罰、敘事後果 |

## 三、地點系統｜Location System

### 3.1 地點卡設計

地點卡有正面和背面（背面是隱藏資訊）：

| 面 | 內容 |
|----|------|
| 正面 | 地點名稱、調查難度 DC、線索數量、連接關係、進入效果 |
| 背面 | 隱藏描述、翻面觸發效果、翻面條件（旗標、感知門檻等） |

### 3.2 地點尺度

地點的尺度由關卡設計決定，可以是：

| 尺度 | 範例 | 移動成本傾向 |
|------|------|-------------|
| 房間級 | 宅邸的各個房間 | 行動點 |
| 街區級 | 城鎮的各個區域 | 行動點 |
| 城市級 | 不同城鎮之間 | 時間 |
| 跨國級 | 不同國家之間 | 時間 |

### 3.3 移動成本

| 類型 | 代碼 | 說明 |
|------|------|------|
| 行動點移動 | `action_point` | 花費行動點移動到相鄰地點（預設） |
| 時間移動 | `time` | 花費遊戲內時間移動（大尺度場景） |

行動點管「地點內行動」，時間管「地點間移動」— 兩者是獨立資源。

### 3.4 地點資料欄位

| 欄位 | 代碼 | 說明 |
|------|------|------|
| 調查難度 | `shroud` | 調查檢定的 DC |
| 基礎線索數 | `clues_base` | 此地點的基礎線索數量 |
| 按人數縮放 | `clues_per_player` | 線索數量是否乘以玩家人數 |
| 連接地點 | `connections` | 相鄰地點列表 |
| 移動費用 | `travel_cost` | 移動到此地點的行動點花費 |
| 移動費用類型 | `travel_cost_type` | `action_point` 或 `time` |
| 可發現的卡片 | `discoverable_cards` | 場景中可探索到的卡片資源 |
| 進入效果 | `enter_effects` | 進入此地點時觸發的效果 |
| 背面翻面條件 | `back_reveal_condition` | 旗標、感知門檻等 |

## 四、劇情旗標系統｜Story Flag System

### 4.1 旗標定義

劇情旗標是記錄遊戲狀態的標記，用於驅動分支路線、間章事件、條件判定。

### 4.2 九大類別

| 類別 | 代碼 | 預設可見性 | 說明 |
|------|------|-----------|------|
| 行動進度 | `act` | 可見 | 目標牌堆的推進狀態 |
| 議程進度 | `agenda` | 可見 | 議程牌堆的推進狀態 |
| NPC 狀態 | `npc` | 視設計決定 | NPC 的存活、態度、位置 |
| 物品取得 | `item` | 可見 | 是否取得特定物品 |
| 地點探索 | `location` | 視設計決定 | 是否探索過特定地點 |
| 玩家選擇 | `choice` | 可見 | 劇情分歧點的選擇記錄 |
| 章節結果 | `outcome` | 可見 | 章節結束時的結果（A/B/C 等） |
| 時間狀態 | `time` | 可見 | 時間相關的進度或觸發 |
| 隱藏旗標 | `hidden` | **永遠隱藏** | 系統內部判定用，玩家看不到 |

### 4.3 命名規則

```
[類別].[章節縮寫]_[描述]
```

範例：`act.ch3_sealed_gate`、`npc.ch5_henry_alive`、`hidden.ch7_ritual_progress`

### 4.4 旗標與支線的關係

> **支線不使用旗標系統。** 支線結果只分通過/失敗，不產生任何主線旗標。

## 五、分支路線與非線性敘事｜Branching Paths

### 5.1 章節結果判定

- 每章最少 2 種，最多 5 種分歧結果。
- 由目標牌堆進度 × 議程牌堆進度 × 旗標組合判定。
- 結果影響下一章的版本，不影響是否能繼續遊戲。

### 5.2 分支影響範圍

分支影響章節內部配置，不增減章節數：

| 受影響的內容 | 說明 |
|-------------|------|
| 地圖配置 | 不同結果可能導向不同的地點組合 |
| 目標牌堆內容 | 不同版本的任務目標 |
| 議程牌堆內容 | 不同版本的敵人計畫 |
| 遭遇牌堆內容 | 不同的隨機事件 |
| 混沌袋配置 | 不同的混沌袋標記組成 |
| 敘事文字 | 不同的劇情描述 |

### 5.3 戰役中的混沌袋演變

混沌袋在戰役推進過程中會逐漸變差：

- 每章結束後，依據結果可能永久新增負面標記。
- 結果越差（例如 C 結果），混沌袋惡化越嚴重。
- 呼應支柱 4（三層死亡）的溫水煮青蛙設計。

## 六、間章事件｜Interlude Events

### 6.1 定義

間章事件是插入在章節之間的特殊事件，根據旗標條件決定是否觸發。

### 6.2 兩個插入點

| 插入點 | 代碼 | 時機 |
|--------|------|------|
| 章末事件 | `epilogue` | 章節結束後、長休息前 |
| 章首事件 | `prologue` | 長休息後、新章節開始前 |

### 6.3 間章事件可執行的操作

- 損耗物資（扣除資源、使用次數、HP、SAN 等）
- 設定旗標（可見或隱藏）
- 觸發檢定（成功/失敗導向不同結果）
- 給予選擇（玩家做出影響後續的決定）
- 給予獎勵或懲罰

### 6.4 觸發條件

間章事件可設定觸發條件（旗標組合），條件為 NULL 時無條件觸發。

## 七、支線關卡系統｜Side Stage System

### 7.1 兩種支線

| 類型 | 代碼 | 說明 |
|------|------|------|
| 預設小關卡 | `side` | 預先設計，有劇情敘事 |
| 隨機地城 | `side_random` | 每次進入重新生成 |

### 7.2 進入條件

玩家在**遊戲大廳**即可進入支線，包含：

- 首次進入遊戲，進入遊戲大廳後
- 每個章節結束，回到遊戲大廳後

玩家從大廳自由選擇：主線章節、預設小關卡、隨機地城。

### 7.3 預設小關卡（A）規則

| 項目 | 規格 |
|------|------|
| 場景結構 | 使用目標牌堆 + 議程牌堆結構 |
| 結果判定 | 目標牌堆先完成 = 通過，議程牌堆先跑完 = 失敗 |
| 旗標系統 | **不使用**，不產生任何主線旗標 |
| 關卡規模 | 可單關，也可多關成組（3+ 關），各關以「通過/失敗」串接 |
| 解鎖條件 | 彈性：無條件開放 / 主線進度門檻 / 特定條件 |

### 7.4 隨機地城（B）規則

| 項目 | 規格 |
|------|------|
| 地圖結構 | 待定，場景編輯器階段設計 |
| 地點數量 | 待定 |
| 勝利條件 | 待定，不限定單一種 |
| 創建規則 | 待定 |

### 7.5 支線與主線的差異

| 面向 | 主線 | 支線 |
|------|------|------|
| 劇情影響 | 產生旗標，影響後續章節 | 不產生任何旗標 |
| 結果判定 | 多分支（A/B/C 等） | 二元（通過/失敗） |
| 角色風險 | 瀕死/死亡照常 | 瀕死/死亡照常 |
| 關卡失敗懲罰 | 影響旗標與後續劇情 | 無懲罰，白跑一趟 |

## 八、獎勵結構｜Reward Structure

### 8.1 通關獎勵

支線通關獎勵與主線相同，是主線獎勵的加速版本：

| 獎勵類型 | 說明 |
|---------|------|
| 經驗值 | 調查員成長資源 |
| 天賦點 | 天賦樹投資資源 |
| 素材 | 鍛造與製作用素材 |
| 凝聚力 | 團隊精神投資資源 |

### 8.2 支線專屬簽名卡

每個預設小關卡可設計支線專屬簽名卡 — 僅**首次通關**時獲得。重返時不再給予。

### 8.3 隨機地城獎勵

每次都是新局，獎勵**不遞減**。無專屬簽名卡。

## 九、重返機制｜Return Mechanic

### 9.1 適用範圍

| 類型 | 重返 |
|------|------|
| 預設小關卡 | ✅ 可重返 |
| 隨機地城 | ❌ 不適用（每次重新生成） |

### 9.2 重返規則

| 項目 | 規格 |
|------|------|
| 難度 | 每次重返難度遞增 |
| 內容 | 重返時內容可能不同（不同遭遇、敘事、地圖配置） |
| 獎勵 | 遞減 — 簽名卡僅首次，後續只給經驗值、天賦點、素材、凝聚力 |
| 次數 | 無上限，難度持續遞增自然形成停止點 |
| 難度遞增機制 | 具體機制待數值規格定義 |

---

## 十、失敗規則｜Failure Rules

### 10.1 關卡層級

| 情境 | 後果 |
|------|------|
| 主線失敗（議程牌堆先跑完） | 依結果判定影響旗標和後續劇情，但仍可繼續 |
| 支線失敗 | **無懲罰**，不獲得通關獎勵，可重新挑戰 |

### 10.2 角色層級

| 情境 | 後果 |
|------|------|
| HP 或 SAN 歸零 | 進入瀕死狀態，照常執行瀕死檢定 |
| 瀕死檢定三次失敗 | 死亡，歸零項上限 -1 |
| 上限歸零 | **永久死亡**，角色資料刪除 |

> **永久死亡不因支線而豁免。** 支線「輕鬆」是指獎勵好拿、不影響主線，但角色的命是真的。

## 十一、場景編輯器需求摘要｜Scenario Editor Requirements

場景編輯器需支援以下功能：

| 功能類別 | 具體功能 |
|---------|---------|
| 場景類型 | 選擇 `main` / `side` / `side_random` |
| 地點設計 | 地點正反面、連接關係、調查難度、線索數量、可發現卡片 |
| 目標牌堆 | 多張目標卡的正反面設計、推進條件、結算效果 |
| 議程牌堆 | 多張議程卡的正反面設計、毀滅門檻、後果效果 |
| 混沌袋配置 | 四難度 × 五大類標記的數量、數值、場景效果 |
| 旗標管理 | 旗標定義、設定、讀取、條件判定 |
| 間章事件 | 章末/章首事件設計、觸發條件、操作定義 |
| 分支路線 | 結果判定條件、下一章版本指向 |
| 支線設定 | 解鎖條件、獎勵、簽名卡、重返版本 |
| 人數縮放 | 敵人數值、線索需求、遭遇效果的人數參數 |
| 怪物配置 | 場景中的怪物種類、位階、生成位置、偏好設定 |

>
> 1. **戰役固定十章** — 分支不增減章節數。

> 10. **隱藏旗標玩家看不到** — 用於系統內部判定。
> 11. **地點尺度由關卡設計決定** — 房間級到跨國級皆可。
> 12. **隨機地城的詳細設計留待場景編輯器階段。**
