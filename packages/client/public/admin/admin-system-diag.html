系統診斷面板

用於驗證遠端裝置（iPad / 手機 / 其他電腦）能否連線到 Admin 與 bridge。

用途：在隔離環境複製 MOD-01 的 Gemini 呼叫流程，完整 log 每個階段（prompt 建構、API 請求、HTTP status、原始回應、JSON 解析、法器欄位檢查），用來診斷 Gemini「跑不出結果」的實際原因。

此按鈕與 MOD-01 卡片設計器共用同一份 admin-card-prompt.js,確保測試的 prompt 與實際生成一致(不會漂移)。

用途：當 MOD-12 批次寫卡或 MOD-01 儲存時跳「column X does not exist」這類 DB 42703 錯誤,代表 Railway 遠端資料庫的 schema 落後於程式碼。按「列出欄位」先確認缺哪些,再按「補跑」強制重跑 MIGRATION_021-025(使用 IF NOT EXISTS / ON CONFLICT,重跑安全)。補跑完成後回到 MOD-12 再試一次即可。

用途：建立新使用者帳號或重設既有密碼。Role 決定可見模組:

  • viewer / editor:可看 MOD-01~11 + SIM-01~06(一般卡片設計者用)
  • admin / owner:全部模組可見(含 MOD-12 / MOD-14 / AXIS / DIAG)
Log
在 iPad 上：長按選取 → 複製。或按「複製全部 log」把內容貼給 Claude。
