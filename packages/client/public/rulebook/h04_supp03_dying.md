## 一、瀕死與救援系統｜Downed & Rescue System

### 1.1 觸發條件

HP 歸零或 SAN 歸零時，調查員進入**瀕死狀態**。HP 和 SAN 是獨立的，可能只有一項歸零，也可能兩項同時歸零。

### 1.2 瀕死狀態

- 瀕死的調查員**留在板塊上**，算躺在地上。
- 與其交戰中的怪物**維持交戰關係**。
- 瀕死調查員**不能執行任何行動**。
- 回合開始時以**瀕死檢定**取代正常行動。

### 1.3 瀕死檢定

沿用 D&D 瀕死判定規則：

| 項目 | 規則 |
|------|------|
| 檢定方式 | 擲 d20，DC 10 |
| 成功累積 3 次 | 自行穩定，站起來，歸零的數值恢復至 **1 點** |
| 失敗累積 3 次 | **死亡**，且該項歸零的數值**上限永久 -1**（創傷侵蝕） |
| 自然 20 | **直接站起來**，歸零值恢復至 1 |
| 自然 1 | 算 **2 次失敗** |
| 被怪物攻擊 | 每次命中算 **1 次瀕死檢定失敗** |

### 1.4 HP 與 SAN 同時歸零

- 只做 **1 組**瀕死檢定（不分別做兩組）。
- 站起來時 HP 和 SAN **都恢復 1 點**。
- 死亡時 HP 和 SAN **上限都 -1**。

### 1.5 隊友救援方式

隊友可透過以下兩種方式救援瀕死的調查員（二擇一）：

| 方式 | 效果 | 是否需要檢定 |
|------|------|-------------|
| **治療** | 對瀕死調查員使用任何量的治療效果（卡片或行動），立即脫離瀕死 | 否，自動生效 |
| **穩定** | 花費 1 行動點，讓瀕死調查員的檢定計數增加 1 次成功 | 否，自動成功 |

### 1.6 永久死亡的銜接

- 瀕死檢定失敗 3 次 → 死亡 → 歸零項上限 -1。
- 若上限 -1 後變為 **0** → **永久死亡**，角色資料刪除。
- 例如：調查員 HP 上限僅剩 1，HP 歸零後瀕死檢定三次失敗，HP 上限 -1 變為 0 = 永久死亡。
- 永久死亡後的處理流程參照《資料庫結構設計》的 memorial 紀念表。

## 二、敵人 AI 行為規則｜Enemy AI Behavior

### 2.1 設計前提

所有檢定都由玩家擲骰。城主是 AI，不會擲骰。怪物的行為由系統根據規則自動執行。

### 2.2 行動順序

敵人階段中，多隻怪物的行動順序依**位階從高到低**排列：

```
巨頭（Titan）→ 頭目（Boss）→ 精英（Elite）→ 威脅（Threat）→ 雜兵（Minion）
```

同位階的怪物依**生成順序**行動（先出現的先動）。

### 2.3 基礎行為模式

每隻怪物在敵人階段執行以下流程：

```
已與調查員交戰？
  ├ 是 → 攻擊交戰對象
  └ 否 → 朝目標移動 1 格
           └ 移動後若進入有調查員的板塊 → 進入交戰
```

### 2.4 攻擊次數

| 位階 | 基礎攻擊次數 |
|------|-------------|
| 雜兵、威脅 | 1 次 |
| 精英 | 1 次（可透過詞綴增加） |
| 頭目 | 可透過設計增加至 2–3 次 |
| 巨頭 | 可透過設計增加至 2–3 次 |

高階敵人的壓迫感透過攻擊次數（以及傳奇行動）體現。

### 2.5 偏好系統（追誰？）

每隻怪物可配置一個**偏好類型**，決定牠在未交戰時追蹤哪位調查員：

| 偏好代碼 | 中文 | 說明 |
|---------|------|------|
| `nearest` | 最近 | 追蹤最近的調查員 |
| `lowest_hp` | 血量最低 | 追蹤當前 HP 最低的調查員 |
| `lowest_san` | 理智最低 | 追蹤當前 SAN 最低的調查員 |
| `most_clues` | 線索最多 | 追蹤持有最多線索的調查員 |
| `last_attacker` | 仇恨 | 追蹤上回合攻擊過牠的調查員 |
| `lowest_attr` | 屬性最低 | 追蹤指定屬性最低的調查員（需搭配屬性參數） |
| `random` | 隨機 | 隨機選擇目標 |

偏好類型由**關卡設計師**在怪物定義中配置，不寫死在系統規則中。

### 2.6 移動尋路

- 計算到目標的**最短路徑**。
- 每回合沿最短路徑前進 **1 格**（快速詞綴 2 格）。
- 多條等距路徑時**隨機選一條**。

### 2.7 怪物攻擊的解析方式

怪物攻擊時不由城主擲骰，而是：

1. 怪物從自己的**攻擊招式池**中抽取一張**怪物戰鬥風格卡**。
2. 卡面呈現攻擊敘事、指定調查員的**防禦檢定屬性**和 **DC**，以及**傷害值**。
3. 調查員擲骰做指定屬性的防禦檢定。
4. 成功 → 避免或減輕傷害（依卡面定義）。
5. 失敗 → 承受卡面傷害。

詳細的戰鬥風格卡系統見第三章。

## 三、戰鬥風格卡系統｜Combat Style Card System

### 3.1 核心概念

戰鬥風格卡系統是敘事式戰鬥（E-2）的機制實作。每一次攻擊不是單純的「擲骰比數字」，而是抽出一張風格卡，呈現一段敘事，並決定本次檢定的屬性。

**核心規則：誰發起攻擊，誰抽戰鬥風格卡。** 被攻擊方根據卡面指示做對應的檢定。

### 3.2 調查員攻擊流程

```
調查員選擇攻擊方式（由武器決定，例如「射擊」）
  ↓
系統從該攻擊方式的風格卡池中隨機抽 1 張
  ↓
呈現敘事描述（例如「你沉穩地握住槍的握把，扣下板機」）
  ↓
卡面指定本次檢定屬性（例如「力量檢定」）
  ↓
調查員擲 d20 + 屬性修正 + 熟練/專精修正 + 武器修正（僅對應屬性生效）
  ↓
結果 ≥ 怪物 DC → 命中，造成武器卡面固定傷害
結果 < 怪物 DC → 未命中
自然 20 → 爆擊，2 倍傷害
自然 1 → 大失敗（見《補充 01》5.5 攻擊結算）
```

### 3.3 怪物攻擊流程

```
怪物從自己的招式池中隨機抽 1 張怪物戰鬥風格卡
  ↓
呈現攻擊敘事（例如「怪物的觸手朝你捲過來，嘗試把你困住」）
  ↓
卡面指定：防禦檢定屬性 + DC + 傷害值
  ↓
調查員擲 d20 + 屬性修正 + 相關卡片修正
  ↓
結果 ≥ DC → 防禦成功，依卡面定義處理（免傷或減傷）
結果 < DC → 防禦失敗，承受卡面傷害
```

### 3.4 神秘攻擊例外

當攻擊方式屬於**施法**類型時，不擲骰，改為**抽混沌袋**決定結果。對應核心設計原則的雙軌檢定（支柱 2）：物理攻擊用骰子（掌控感），神秘攻擊用混沌袋（玩火感）。

### 3.5 調查員風格卡結構

調查員的戰鬥風格卡記錄以下資訊：

| 欄位 | 說明 |
|------|------|
| 歸屬攻擊方式 | 屬於哪個第一層戰鬥風格（射擊、近戰格鬥等） |
| 檢定屬性 | 本次檢定使用的屬性（力量/敏捷/意志等） |
| 攻擊敘事 | 攻擊前的描述文字 |
| 成功敘事 | 命中時的描述文字 |
| 失敗敘事 | 未命中時的描述文字 |

- **DC 不在風格卡上** — DC 來自目標怪物。
- **傷害不在風格卡上** — 傷害來自調查員的武器卡。
- 風格卡**只決定「用什麼屬性檢定」和「敘事怎麼說」**。

### 3.6 怪物風格卡結構

怪物的戰鬥風格卡記錄以下資訊：

| 欄位 | 說明 |
|------|------|
| 歸屬怪物 | 屬於哪隻怪物或哪類怪物 |
| 攻擊方式名稱 | 招式名稱（撕咬、觸手纏繞、精神侵蝕等） |
| 防禦檢定屬性 | 調查員需要用哪個屬性做防禦檢定 |
| DC | 防禦檢定的目標值 |
| 傷害值 | 防禦失敗時造成的傷害 |
| 傷害類型 | 物理傷害 / 恐懼傷害 / 兩者皆有 |
| 元素屬性 | 物理 / 火 / 冰 / 電 / 神秘 |
| 攻擊敘事 | 怪物發動攻擊的描述文字 |
| 成功敘事 | 調查員防禦成功時的描述文字 |
| 失敗敘事 | 調查員防禦失敗時的描述文字 |

- 怪物風格卡與調查員風格卡是**完全獨立的資料結構**。
- 同一隻怪物的不同招式可以有不同的 DC、不同的傷害、不同的防禦屬性。

### 3.7 武器修正值與風格卡的關係

武器卡上的屬性修正**只在對應屬性的風格卡被抽到時生效**。

- 基礎武器（等級 0）：通常只有**一個屬性**的修正。
  - 例如：.45 手槍（等級 0）→ 力量 +1。
  - 抽到力量檢定時 +1 生效，抽到敏捷或意志檢定時無加成。

- 升級後的武器：可透過玩家選擇的強化路線獲得更多修正。
  - **廣度路線**：所有攻擊檢定 +1（不論抽到哪個屬性都有加成）。
  - **深度路線**：特定屬性 +2（例如力量 +2，只在力量檢定時生效但更強）。
  - **代價路線**：某屬性 +2 但另一屬性 -1（玩家自行決定特化方向）。

- 負面修正**不出現在基礎卡片**上，只在玩家強化卡片時作為取捨出現。

### 3.8 戰鬥熟練與風格卡池的關係

- 風格卡池是**公用的** — 不管調查員有沒有對應的戰鬥熟練，使用該類武器攻擊時都從同一個風格卡池抽卡。
- 差別在於有**熟練**的調查員在檢定時額外加上熟練修正值。
- 有**專精**（第二層）的調查員獲得更高的修正值或其他專精特有效果。

### 3.9 武器卡與戰鬥風格的對應

每張武器卡需標註 `combat_style` 欄位，指定它屬於哪個第一層戰鬥風格。攻擊時依此欄位決定從哪個風格卡池抽卡。

| combat_style 值 | 對應戰鬥風格 |
|-----------------|-------------|
| `shooting` | 槍枝射擊 |
| `archery` | 弓術 |
| `sidearm` | 隨身武器 |
| `military` | 軍用武器 |
| `brawl` | 搏擊 |
| `arcane` | 施法 |
| `engineer` | 工兵 |
| `assassin` | 暗殺 |

### 3.10 卡面顯示的動態標籤

武器卡上顯示的戰鬥風格標籤是**動態的**，反映調查員當前的熟練/專精狀態：

- 調查員只有第一層熟練 → 卡面顯示第一層名稱（例如「搏擊」）。
- 調查員已專精到第二層 → 卡面顯示對應的第二層專精名稱（例如「酒館鬥毆者」）。
- 卡片定義本身只記錄 `combat_style`（第一層），動態標籤由前端渲染時查詢調查員狀態決定。

- **第一層：戰鬥熟練（Proficiency）** — 對該類攻擊方式的基礎掌握，提供熟練修正。
- **第二層：戰鬥專精（Specialization）** — 從熟練中進一步特化，提供更高修正或專精特有效果。
- 共 **8 個第一層**，**30 個第二層**。

## 五、對卡片效果語言的補充｜Effect Language Supplement

### 5.1 新增觸發時機

以下觸發時機需追加至《補充 02》的觸發時機列表（7.3 傷害相關）：

| 代碼 | 中文 | 說明 |
|------|------|------|
| `before_take_damage` | 將要受到傷害時 | 傷害結算前的介入窗口，可觸發減傷、抵消、轉移等效果 |
| `before_take_horror` | 將要受到恐懼時 | 恐懼傷害結算前的介入窗口 |
| `before_downed` | 將要倒地時 | HP 或 SAN 即將歸零的瞬間，可觸發保命效果 |

**設計意圖：** `before_` 系列與 `on_` 系列是不同的時機窗口。`before_` 是傷害落地前的攔截機會，`on_` 是傷害落地後的反應。例如護身符的「抵消 1 點恐懼」應掛在 `before_take_horror`，因為它要在傷害結算前攔截。

### 5.2 新增卡片欄位

以下欄位需追加至 `card_definitions` 資料表：

| 欄位 | 類型 | 說明 |
|------|------|------|
| `combat_style` | VARCHAR(32) | 武器卡歸屬的第一層戰鬥風格，值為 shooting / archery / sidearm / military / brawl / arcane / engineer / assassin |

### 5.3 武器修正值結構重構

原有的 `check_attribute`（單一屬性）+ `check_modifier`（單一修正值）結構不再適用。

武器卡的檢定屬性不再由武器決定，而是由戰鬥風格卡在攻擊時動態決定。武器上只記錄「對各屬性提供的修正值」。

新結構：

| 原欄位 | 處理方式 |
|--------|---------|
| `check_attribute` | 移除 — 檢定屬性由風格卡決定 |
| `check_modifier` | 移除 — 改為下方的多屬性修正結構 |
| `check_method` | 移除 — 檢定方式由風格卡（擲骰）或攻擊方式類型（施法 → 混沌袋）決定 |

新增欄位：

| 欄位 | 類型 | 說明 |
|------|------|------|
| `attribute_modifiers` | JSONB | 武器對各屬性的修正值，格式見下方 |

```json
// 基礎武器（等級 0）— 單一屬性修正
{
  "strength": 1
}

// 升級後 — 廣度路線
{
  "all": 1
}

// 升級後 — 深度路線
{
  "strength": 2
}

// 升級後 — 代價路線
{
  "strength": 2,
  "willpower": -1
}
```

未列出的屬性預設為 +0。`"all"` 表示對所有屬性檢定都提供該修正值。

### 5.4 圖示系統備註

遊戲中以下類別的資訊都將使用**圖案**表示（非文字）：

- 七大能力屬性
- 戰鬥風格 / 專精
- 五種元素屬性
- 正面狀態
- 負面狀態
- 動作類型

開發測試階段以**有顏色的文字**代替圖案。正式圖案設計為後續美術任務。

## 六、對卡片設計器（MOD-01）的影響摘要｜Impact on Card Designer

本文件確認的規則對卡片設計器有以下具體影響：

1. **新增 `combat_style` 欄位** — 武器卡需選擇歸屬的戰鬥風格（8 選 1）。
2. **移除 `check_attribute`、`check_modifier`、`check_method`** — 這三個欄位的功能被戰鬥風格卡系統取代。
3. **新增 `attribute_modifiers`** — 取代舊的單一修正值，支援多屬性修正的 JSON 結構。
4. **新增 3 個觸發時機** — `before_take_damage`、`before_take_horror`、`before_downed`。
5. **戰鬥風格卡的設計工具** — 在 MOD-05（戰鬥熟練管理）中建立調查員風格卡池的編輯功能。
6. **怪物風格卡的設計工具** — 在 MOD-03（敵人設計器）中建立怪物招式池的編輯功能。

> **給未來 Claude 實例的備註｜Note to Future Claude Instances**
>
> 本文件的所有規則均經過與專案原創者 Uria 的逐項討論確認。
> 特別注意以下設計精神：
>
> 1. **所有檢定都由玩家擲骰** — 城主是 AI，不會擲骰。怪物攻擊時由調查員做防禦檢定。
> 2. **戰鬥風格卡創造不確定性** — 同一把武器每次攻擊可能要求不同屬性，「萬能角色」不存在。
> 3. **武器修正只在對應屬性生效** — 這讓抽到什麼風格卡變成真正有影響的隨機事件。
> 4. **負面修正不在基礎卡上** — 玩家在強化時自己做取捨，不是被系統強加。
> 5. **風格卡池是公用的** — 有無熟練的差異體現在修正值，不是卡池內容。
> 6. **卡面標籤是動態的** — 顯示調查員當前的熟練/專精等級名稱，不是靜態文字。
> 7. **怪物風格卡獨立設計** — 跟調查員風格卡是完全不同的資料結構。
> 8. **瀕死檢定沿用 D&D 規則** — DC 10、自然 20 直接站起來、自然 1 算兩次失敗。
> 9. **穩定行動自動成功** — 隊友花 1 行動點就能幫忙，不存在「救援失敗」。
